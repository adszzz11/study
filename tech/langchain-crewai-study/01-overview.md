# LangChain & CrewAI 개요

## LangChain이란?

LangChain은 LLM(Large Language Model) 기반 애플리케이션을 쉽게 개발할 수 있게 해주는 프레임워크이다.

### 핵심 개념

```
┌─────────────────────────────────────────────────┐
│                  LangChain 아키텍처               │
├─────────────────────────────────────────────────┤
│  ┌─────────┐   ┌─────────┐   ┌─────────┐       │
│  │ Prompt  │ → │   LLM   │ → │ Output  │       │
│  │Template │   │         │   │ Parser  │       │
│  └─────────┘   └─────────┘   └─────────┘       │
│       ↑                           ↓            │
│  ┌─────────┐                ┌─────────┐        │
│  │ Memory  │ ←───────────── │  Chain  │        │
│  └─────────┘                └─────────┘        │
│                                  ↓             │
│                            ┌─────────┐         │
│                            │  Agent  │ → Tools │
│                            └─────────┘         │
└─────────────────────────────────────────────────┘
```

| 개념 | 설명 | 예시 |
|------|------|------|
| **LLM** | 언어 모델 래퍼 | ChatOpenAI, Claude, Ollama |
| **Prompt Template** | 재사용 가능한 프롬프트 | "다음을 {language}로 번역해줘: {text}" |
| **Chain** | 컴포넌트 연결 | prompt → llm → parser |
| **Memory** | 대화 기록 저장 | ConversationBufferMemory |
| **Agent** | 도구 사용 결정 | 검색, 계산, API 호출 |
| **Tools** | 에이전트가 사용하는 기능 | WebSearch, Calculator |

### LangChain 장점

- **모듈화**: 컴포넌트 조합으로 복잡한 파이프라인 구성
- **추상화**: 다양한 LLM 제공자를 동일한 인터페이스로 사용
- **생태계**: 풍부한 통합 (벡터 DB, 검색 엔진 등)
- **LCEL**: 직관적인 파이프 문법으로 체인 구성
- **모니터링**: LangSmith를 통한 추적, 디버깅, 평가

### LangChain 단점

- **학습 곡선**: 개념이 많아 초반 학습 필요
- **버전 변경**: 빠른 업데이트로 인한 Breaking Changes
- **오버헤드**: 간단한 작업에는 과할 수 있음
- **디버깅**: 체인이 복잡해지면 문제 추적이 어려움

## CrewAI란?

CrewAI는 역할 기반의 다중 AI 에이전트가 협업하여 복잡한 작업을 수행하는 프레임워크이다.

### 핵심 개념

```
┌─────────────────────────────────────────────────┐
│                   CrewAI 아키텍처                │
├─────────────────────────────────────────────────┤
│                    ┌────────┐                   │
│                    │  Crew  │                   │
│                    └────┬───┘                   │
│           ┌────────────┼────────────┐          │
│           ↓            ↓            ↓          │
│     ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│     │ Agent 1 │  │ Agent 2 │  │ Agent 3 │     │
│     │연구원   │  │작성자   │  │편집자   │     │
│     └────┬────┘  └────┬────┘  └────┬────┘     │
│          ↓            ↓            ↓          │
│     ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│     │ Task 1  │  │ Task 2  │  │ Task 3  │     │
│     │조사하기 │→ │초안작성 │→ │검토하기 │     │
│     └─────────┘  └─────────┘  └─────────┘     │
│          │            │            │          │
│          └────────────┼────────────┘          │
│                       ↓                       │
│                 ┌──────────┐                  │
│                 │  Tools   │                  │
│                 │검색,파일 │                  │
│                 └──────────┘                  │
└─────────────────────────────────────────────────┘
```

| 개념 | 설명 | 예시 |
|------|------|------|
| **Agent** | 특정 역할을 가진 AI | 연구원, 작성자, 분석가 |
| **Task** | 에이전트가 수행할 작업 | "시장 조사 리포트 작성" |
| **Crew** | 에이전트와 태스크의 팀 | 콘텐츠 제작 크루 |
| **Process** | 작업 실행 방식 | sequential, hierarchical |
| **Tools** | 에이전트가 사용하는 도구 | 웹 검색, 파일 읽기 |
| **Memory** | 에이전트 간 정보 공유 | 단기/장기 메모리 |

### CrewAI 장점

- **직관적**: 사람 팀처럼 역할 기반 설계
- **협업**: 에이전트 간 자연스러운 작업 분담
- **유연성**: 다양한 프로세스 타입 지원
- **통합**: LangChain 도구 활용 가능
- **확장성**: 커스텀 에이전트, 도구 쉽게 추가

### CrewAI 단점

- **비용**: 다중 에이전트로 인한 API 호출 증가
- **예측 어려움**: 에이전트 간 상호작용 결과 예측 힘듦
- **디버깅**: 복잡한 크루에서 문제 추적 어려움
- **오버엔지니어링**: 단순 작업에 과도할 수 있음

## 사용 사례

### LangChain 적합 케이스

| 사용 사례 | 설명 |
|-----------|------|
| **RAG 시스템** | 문서 검색 + LLM 답변 생성 |
| **챗봇** | 대화 기록 관리, 맥락 유지 |
| **문서 처리** | 요약, 번역, 정보 추출 |
| **데이터 분석** | SQL 쿼리 생성, 결과 해석 |
| **API 통합** | 외부 서비스 연동 |

### CrewAI 적합 케이스

| 사용 사례 | 설명 |
|-----------|------|
| **콘텐츠 제작** | 조사 → 작성 → 편집 파이프라인 |
| **코드 리뷰** | 분석가 + 리뷰어 협업 |
| **시장 조사** | 데이터 수집 → 분석 → 리포트 |
| **고객 지원** | 분류 → 응답 → 검토 워크플로우 |
| **자동화 워크플로우** | 복잡한 다단계 업무 처리 |

### 언제 무엇을 선택할까?

```
단일 작업, 선형 파이프라인 → LangChain
        ↓
복잡한 다단계, 협업 필요 → CrewAI
        ↓
둘 다 필요 → LangChain + CrewAI 통합
```

| 상황 | 추천 |
|------|------|
| 간단한 LLM 호출 | LangChain만 |
| RAG, 검색 시스템 | LangChain |
| 대화형 챗봇 | LangChain |
| 다단계 복잡한 작업 | CrewAI |
| 역할 분담 필요 | CrewAI |
| 복잡한 RAG + 에이전트 협업 | LangChain + CrewAI |

## LangChain vs CrewAI 비교

| 항목 | LangChain | CrewAI |
|------|-----------|--------|
| **목적** | LLM 앱 개발 프레임워크 | 다중 에이전트 협업 |
| **추상화 수준** | 컴포넌트 레벨 | 역할/팀 레벨 |
| **학습 곡선** | 중간 | 낮음 |
| **유연성** | 매우 높음 | 중간 |
| **복잡도 관리** | 개발자 책임 | 프레임워크가 관리 |
| **주요 사용처** | RAG, 챗봇, 파이프라인 | 콘텐츠, 리서치, 자동화 |
| **LLM 지원** | 매우 다양 | OpenAI 중심 |
| **커뮤니티** | 매우 큼 | 성장 중 |

```
│   - 자연스러운 협업 워크플로우                   │

└─────────────────────────────────────────────────┘
```
