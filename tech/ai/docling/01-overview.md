# Part 1: 개요

## 1.1 정의 및 핵심 개념

**Docling**은 PDF, DOCX 등 비정형 문서를 AI/LLM이 활용할 수 있는 구조화된 형식으로 변환하는 IBM의 오픈소스 라이브러리입니다.

### 해결하는 문제

```
Before Docling:
┌─────────────────┐     ┌─────────────────┐
│   PDF 문서      │ ──▶ │  텍스트만 추출   │ ──▶ 테이블 깨짐, 구조 손실
│ (테이블, 이미지) │     │  (PyPDF2 등)    │
└─────────────────┘     └─────────────────┘

After Docling:
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   PDF 문서      │ ──▶ │    Docling      │ ──▶ │ 구조화된 출력    │
│ (테이블, 이미지) │     │  (AI 기반 분석)  │     │ (MD, JSON, HTML) │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 지원 포맷

| 입력 | 출력 |
|------|------|
| PDF, DOCX, PPTX, XLSX | Markdown |
| HTML, 이미지 (PNG, JPG) | JSON (DoclingDocument) |
| 오디오 (실험적) | HTML, DocTags |

---

## 1.2 왜 Docling인가?

### 장점

| 장점 | 설명 |
|------|------|
| **높은 정확도** | DocLayNet(레이아웃), TableFormer(테이블) 등 특화된 AI 모델 |
| **다양한 포맷** | PDF, DOCX, PPTX, XLSX, HTML, 이미지까지 |
| **LLM 친화적** | Markdown, JSON 등 RAG에 최적화 |
| **MIT 라이선스** | 상업적 사용 가능 |
| **경량 실행** | GPU 없이도 일반 하드웨어에서 실행 |

### 단점

| 단점 | 설명 |
|------|------|
| CPU 속도 | GPU 권장 |
| 복잡한 레이아웃 | 100% 완벽하지 않음 |
| Python 버전 | 3.10+ 필수 |

---

## 1.3 주요 사용 사례

- **RAG 파이프라인**: 문서 전처리
- **학술 논문 분석**: 자동 파싱
- **기업 문서 디지털화**: PDF → 구조화 데이터
- **지식 베이스 구축**: 대규모 문서 처리
