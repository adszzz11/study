### 신규 기능

#### `/code-review --fix` (+ `/simplify` 통합)
- `/code-review --fix`: 리뷰 결과를 **working tree에 바로 적용**
  - 재사용·단순화·효율화 제안을 코드에 직접 반영
- `/simplify` 명령이 이제 내부적으로 `/code-review --fix`를 호출

```bash
/code-review --fix    # 리뷰 후 자동 수정 적용
/simplify             # 동일 (→ /code-review --fix 호출)
```

#### 스킬 `disallowed-tools` frontmatter
- 스킬이 활성 상태일 때 특정 도구를 **모델에서 제거** 가능
- 위험 도구를 스킬 범위 내에서 차단할 때 활용

```yaml
---
# SKILL.md frontmatter 예시
disallowed-tools:
  - Bash
  - Write
---
```

#### `/reload-skills` 명령
- 세션 재시작 없이 **스킬 디렉터리를 즉시 재스캔**
- 스킬 개발 중 변경사항을 빠르게 반영할 때 사용

#### `SessionStart` 훅 개선
- `reloadSkills: true` 반환으로 훅이 설치한 스킬을 **같은 세션에서 바로 사용 가능**
- `hookSpecificOutput.sessionTitle`로 **세션 타이틀 설정** (시작/재개 시)

```json
{
  "reloadSkills": true,
  "hookSpecificOutput": {
    "sessionTitle": "🚀 프로젝트명 - 작업 유형"
  }
}
```

#### `MessageDisplay` 훅 이벤트 (신규)
- 어시스턴트 메시지가 **표시되는 시점에** 훅이 개입 가능
- 메시지 텍스트를 **변환하거나 숨기는** 용도로 사용

> → [[../06-hooks|06-hooks]] 훅 참조 문서 보강 필요

#### `pluginSuggestionMarketplaces` 관리 설정
- 관리자가 허용한 **org 마켓플레이스만** context-aware 팁으로 플러그인 제안 가능
- 기업 환경에서 비승인 플러그인 노출 방지

#### `--fallback-model` 지원 강화
- 기본 모델이 존재하지 않을 때 **매 요청 실패 대신 fallback 모델로 자동 전환**
- 세션 전체에 걸쳐 적용됨

```bash
claude --fallback-model claude-sonnet-4-6
```

#### Auto 모드 — opt-in 동의 불필요
- 이전에는 Auto 모드 최초 사용 시 동의 절차 필요
- 이제 즉시 사용 가능

#### Vim 모드 `/` — 히스토리 역방향 검색
- NORMAL 모드에서 `/`를 누르면 `Ctrl+R`과 동일하게 **역방향 히스토리 검색** 열림
- bash/zsh vi-mode와 동작 일치

#### `/usage` 대형 세션 파일 포함 + 스트리밍 읽기
- 대형 세션 파일도 `/usage` 집계에 포함
- 스트리밍 읽기 방식으로 메모리 사용량 일정 유지

#### 사고 요약(Thinking Summaries) 개선
- 접힌 그룹에서 최소 **3초 이상 가독 상태 유지**
- 요약이 **마크다운으로 렌더링**
- 최대 **10줄로 제한** (전체 보기: `Ctrl+O`)

### 사용 팁

#### `/code-review --fix` 워크플로
```bash
# 1. 리뷰 후 직접 수정 적용
/code-review --fix

# 2. 또는 단순화에 집중
/simplify

# 3. 특정 강도로 리뷰 후 수정
/code-review high --fix
```

#### 스킬 개발 사이클 가속 (v2.1.152+)
```bash
# 스킬 파일 수정 후
/reload-skills   # 재시작 없이 즉시 반영
```
