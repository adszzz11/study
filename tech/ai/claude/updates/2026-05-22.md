# Claude 업데이트 - 2026-05-22

## Claude Code v2.1.147 (2026-05-21) ⭐

> 출처: https://code.claude.com/docs/en/changelog

### 주요 신기능

#### Workflow Tool (기본 비활성)
- **결정론적 멀티에이전트 오케스트레이션** 도구 추가
- 환경변수 `CLAUDE_CODE_WORKFLOWS=1` 로 활성화
- LLM 비결정성 없이 순서가 정해진 에이전트 파이프라인 구성 가능

#### 고정 백그라운드 세션 (Pinned Background Sessions)
- `claude agents`에서 **`Ctrl+T`** 로 세션을 "고정(pinned)" 상태로 전환
- 고정 세션은 idle 상태에서도 살아있음 (기존에는 자동 종료)
- 업데이트 적용 시 제자리(in-place) 재시작 지원

### 주목할 변경사항

| 항목 | 변경 내용 |
|------|-----------|
| `/simplify` → `/code-review` | 리네이밍 + effort 레벨 지원 (`/code-review high`) |
| `--comment` 플래그 | GitHub PR에 인라인 댓글로 리뷰 결과 게시 |
| 샌드박스 보안 강화 | REPL·Workflow 샌드박스를 prototype-pollution 및 thenable-based 탈출 공격에 대해 하드닝 |
| 자동 업데이터 개선 | 재시도 로직, 구체적인 오류 리포팅, 실패 시 버전 표시 |
| Diff 렌더링 성능 | 대용량 파일 편집 시 diff 렌더링 성능 향상 |
| 프롬프트 히스토리 | 연속 중복 입력은 히스토리에 기록하지 않음 |

### 버그 수정

- Enterprise 로그인 제한이 적용되지 않던 문제 수정
- 명령 출력에서 `&`가 `&amp;`로 표시되던 문제 수정 (URL 복사/붙여넣기 오동작)
- headless/SDK 모드에서 알 수 없는 슬래시 명령이 조용히 실패하던 문제 수정
- 작은 터미널에서 `/help` 렌더링 오류 수정
- 밑줄(`_`)로 시작하는 사용자 함수가 shell snapshot에서 누락되던 문제 수정
- 플러그인 에이전트가 frontmatter에 여러 `Agent(...)` 타입을 선언할 때 발생하던 문제 수정
- 다수의 PowerShell / Windows / 터미널 렌더링 문제 수정

## Claude Platform API - Managed Agents & MCP 업데이트 (2026-05-19)

> 출처: https://platform.claude.com/docs/en/release-notes/overview

### MCP Tunnels Research Preview

- **프라이빗 네트워크의 MCP 서버에 연결** 가능한 [MCP tunnels](https://platform.claude.com/docs/en/agents-and-tools/mcp-tunnels/overview) Research Preview 공개
- VPN이나 직접 노출 없이 내부망 MCP 서버와 통신 가능

### Claude Managed Agents 업데이트

#### 셀프호스팅 샌드박스 (Self-hosted Sandboxes)
- Anthropic 인프라 대신 **자체 인프라**에서 툴 실행 환경을 운영하는 옵션 추가
- 보안·컴플라이언스 요구사항이 엄격한 환경에서 Managed Agents 도입 가능
- 공식 문서: [Self-hosted sandboxes](https://platform.claude.com/docs/en/managed-agents/self-hosted-sandboxes)

#### 활성 세션 MCP 설정 실시간 업데이트
- 이미 실행 중인 세션에서 **MCP 서버·툴 설정 변경** 가능
- 세션 재시작 없이 MCP 도구 추가/제거/수정

#### 대용량 출력 자동 파일 스필
- `agent_toolset` 및 MCP 툴의 출력이 **100K 토큰 초과 시 자동으로 파일에 저장**
- 모델은 잘린 미리보기 + 파일 경로를 받고, 필요 시 파일에서 전체 내용 읽기 가능

## 사용 팁

### `/code-review` 활용 (v2.1.147+)
```bash
# 기본 리뷰
/code-review

# 높은 수준 리뷰
/code-review high

# GitHub PR에 인라인 댓글로 게시
/code-review --comment
```
- 이전 `/simplify` 명령이 완전히 대체됨

### Workflow Tool 활성화
```bash
# 환경변수로 활성화
export CLAUDE_CODE_WORKFLOWS=1
claude

# 또는 세션마다
CLAUDE_CODE_WORKFLOWS=1 claude
```
- 결정론적 멀티에이전트 파이프라인 구성 시 활용
- 비활성 기본값이므로 기존 워크플로우에 영향 없음

### MCP Tunnels 활용
- 내부망(VPN, 사내 서버) MCP 서버를 외부 노출 없이 연결
- 현재 Research Preview 단계
