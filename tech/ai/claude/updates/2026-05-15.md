# Claude 업데이트 - 2026-05-15

## Claude Code v2.1.143 (2026-05-15)

> 출처: https://code.claude.com/docs/en/changelog

### 플러그인 관리 개선

- **플러그인 의존성 강제**: `claude plugin disable` 명령 시 다른 활성화된 플러그인이 해당 플러그인에 의존하면 비활성화 거부 (비활성화 체인 힌트 복사 가능)
- **`claude plugin enable` 전이 의존성 자동 활성화**: 의존 플러그인도 함께 강제 활성화
- **`/plugin` 마켓플레이스 브라우즈 패널에 예상 컨텍스트 비용 표시**: 턴별·호출별 토큰 추정치 제공

### Worktree & 백그라운드 세션

- **`worktree.bgIsolation: "none"` 설정 추가**: Worktree가 비실용적인 레포에서 백그라운드 세션이 `EnterWorktree` 없이 워킹 카피를 직접 편집 가능
- **백그라운드 세션 모델·Effort 레벨 보존**: idle 후 재개 시 설정한 모델과 Effort 레벨 유지
- **Worktree 정리 시 `rm -rf` 폴백 제거**: `git worktree remove` 실패해도 gitignored·진행 중 파일 손실 방지

### PowerShell & CLI 개선

- **PowerShell 툴이 `-ExecutionPolicy Bypass` 옵션으로 실행**: 환경변수 `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1`로 옵트아웃 가능
- **Windows Bedrock/Vertex/Foundry 사용자에게 PowerShell 툴 기본 활성화**: `CLAUDE_CODE_USE_POWERSHELL_TOOL=0`으로 옵트아웃
- **`claude agents`에 `--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir` 옵션 추가**: 대시보드 및 백그라운드 세션에 적용
- **`claude agents`에 `--permission-mode`, `--model`, `--effort`, `--dangerously-skip-permissions` 옵션 추가**: 기본값 설정 가능

### 버그 수정

- `.credentials.json`의 `scopes` 값이 배열이 아닌 경우 CLI가 무한 대기하던 문제 수정
- Windows Terminal 및 WSL의 `claude agents`에서 우클릭 붙여넣기 수정
- Stop 훅 반복 차단으로 8회 연속 차단 후 경고와 함께 턴 종료되던 문제 수정
- 백그라운드 셸 실행 중 `/goal` 평가자가 조기 실행되던 문제 수정
- 백그라운드 세션 보존 및 MCP 서버 핸들링 관련 다수 수정
