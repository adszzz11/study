# Part 5: 실전 적용 프로젝트

## 프로젝트 개요

Hashimoto의 6단계 여정에서 배운 원칙들을 **실전에 적용**하기 위한 프로젝트 아이디어 모음이다. 난이도와 소요 시간 순으로 정리한다.

---

## 프로젝트 1: AGENTS.md 구축 프로젝트

**난이도**: 초급 | **소요**: 1~2주 | **Step**: 2, 5

### 목표

현재 진행 중인 프로젝트에 AGENTS.md를 추가하고, **에이전트 실수 기반**으로 규칙을 축적한다.

### 단계

```
Day 1-2: 기본 AGENTS.md 생성
  - 빌드/테스트/린트 명령어
  - 프로젝트 구조 설명
  - 기본 코드 스타일 규칙

Day 3-7: 에이전트 실수 기반 규칙 추가
  - 에이전트로 작은 작업 수행
  - 실수 발생 시 즉시 AGENTS.md에 규칙 추가
  - 목표: 20줄 이상

Day 8-14: 검증 스크립트 작성
  - 테스트 실행 단축 스크립트
  - 린트 + 타입체크 원커맨드 스크립트
  - AGENTS.md에 스크립트 사용법 추가
```

### 완료 기준

- [ ] AGENTS.md에 20줄 이상의 규칙
- [ ] 검증 스크립트 2개 이상
- [ ] 에이전트 실수 빈도 감소 체감

## 프로젝트 2: "두 번 일하기" 훈련 프로젝트

**난이도**: 초급 | **소요**: 2주 | **Step**: 2

### 목표

동일한 작업을 수동/에이전트로 수행하여 **에이전트의 강점과 약점을 체험**한다.

### 단계

```
매일 1개 작업을 선택:
  1. 수동으로 완료 → git commit
  2. 새 브랜치에서 에이전트로 동일 작업 시도
  3. 비교 기록:
     - 소요 시간 (수동 vs 에이전트)
     - 코드 품질 (어느 쪽이 더 나은가?)
     - 에이전트 프롬프트 (무엇이 효과적이었나?)
```

### 추천 작업 유형

| 유형 | 예시 | 예상 결과 |
|------|------|----------|
| 버그 수정 | 간단한 로직 에러 | 에이전트가 빠를 가능성 높음 |
| 유틸 함수 추가 | 날짜 포맷터, 문자열 처리 | 에이전트가 잘함 |
| API 엔드포인트 추가 | CRUD 엔드포인트 | 에이전트가 보일러플레이트에 강함 |
| 리팩토링 | 함수 분리, 네이밍 변경 | 에이전트가 scope 외 파괴 주의 |
| 복잡한 알고리즘 | 성능 최적화, 커스텀 자료구조 | 수동이 나을 가능성 높음 |

### 완료 기준

- [ ] 10개 이상의 비교 작업 완료
- [ ] 에이전트 강점 3가지 발견
- [ ] 에이전트 약점 3가지 발견
- [ ] 효과적인 프롬프트 패턴 3개 발견

## 프로젝트 3: End-of-Day 에이전트 루틴 구축

**난이도**: 중급 | **소요**: 2~3주 | **Step**: 3

### 목표

퇴근 전 30분 루틴을 정착시키고, **아침 warm start**의 효과를 측정한다.

### 단계

```
Week 1: 리서치 에이전트 루틴
  - 매일 퇴근 30분 전 리서치 작업 1개 시작
  - 예: "Next.js 15의 새로운 기능 정리", "Drizzle vs Prisma 비교"
  - 다음 날 아침 결과 리뷰

Week 2: 트리아지 에이전트 추가
  - GitHub 이슈 분류 에이전트 추가
  - gh CLI 활용: "최근 이슈를 분류하고 보고해줘"
  - 이슈 중 slam dunk 식별

Week 3: 탐색 에이전트 추가
  - 막연한 아이디어를 코드로 시도
  - 결과 자체보다 "unknown unknowns 발견"에 집중
```

### 완료 기준

- [ ] 2주 연속 퇴근 전 루틴 유지
- [ ] 아침 warm start로 시작한 날 10일 이상
- [ ] slam dunk 이슈 5개 이상 식별 및 위임

## 프로젝트 4: Scaffolding 패턴 실습

**난이도**: 중급 | **소요**: 1~2주 | **Step**: 4, 5

### 목표

새로운 기능 구현 시 **뼈대를 직접 작성하고 에이전트가 구현을 채우는** 패턴을 연습한다.

### 단계

```
1. 기능 설계 (수동):
   - 파일 구조 결정
   - 인터페이스/타입 정의
   - 함수 시그니처 + TODO 주석 작성

2. 구현 위임 (에이전트):
   - "TODO를 구현해줘. 각 함수 구현 후 테스트를 실행해."
   - Slop Zone 감지 시 pivot

3. 리뷰 + Anti-Slop (수동):
   - 생성된 코드 라인별 리뷰
   - 네이밍, 스타일, 에러 핸들링 수정
   - 이해하지 못하는 코드 재작성
```

### 완료 기준

- [ ] Scaffolding 패턴으로 기능 3개 이상 구현
- [ ] Anti-Slop 세션 5회 이상 수행
- [ ] "이해 못하는 코드" 0줄 달성

## 프로젝트 5: 모델 경쟁 벤치마크

**난이도**: 중급 | **소요**: 1주 | **Step**: 4

### 목표

자신의 프로젝트에서 **어떤 모델/에이전트가 가장 효과적인지** 실험한다.

### 단계

```
1. 프로젝트를 3개 복사 (동일 상태)
2. 동일 작업 + 동일 프롬프트로 3개 모델에 실행
   - Model A: Claude Code
   - Model B: Cursor/Windsurf
   - Model C: Amp/Codex
3. 결과 비교:
   - 완료 시간
   - 코드 품질 (린트 에러, 테스트 통과율)
   - 필요한 수동 수정량
```

### 완료 기준

- [ ] 3가지 작업 유형에서 3개 모델 비교 완료
- [ ] 작업 유형별 최적 모델 식별
- [ ] 비용 대비 효과 분석

## 프로젝트 6: 비간섭적 Ghostty Feature 클론

**난이도**: 고급 | **소요**: 2~3주 | **Step**: 전체

### 목표

Hashimoto의 Ghostty 자동 업데이트 사례처럼, **비간섭적 알림 기능**을 자신의 프로젝트에 구현한다. 16세션 패턴을 따른다.

### 단계

```
Phase 1: Planning (세션 1-2)
  - Oracle/Deep Mode로 아키텍처 설계
  - spec.md 작성
  - 접근법 3가지 비교

Phase 2: Prototyping (세션 3-5)
  - UI만 먼저 구현 (전체 기능 X)
  - Scaffolding 패턴 사용

Phase 3: Implementation (세션 6-10)
  - TODO 구현, 에러 핸들링
  - 검증 스크립트로 자기 검증

Phase 4: Review (세션 11-16)
  - Slop Zone 감지 시 pivot
  - Anti-Slop 세션
  - "What else am I missing?" 리뷰
```

### 완료 기준

- [ ] 전체 세션 기록 (비용, 시간, 성공/실패)
- [ ] Slop Zone 감지 및 pivot 1회 이상
- [ ] AGENTS.md에 새 규칙 10줄 이상 추가
- [ ] 최종 코드에 "이해 못하는 부분" 0
