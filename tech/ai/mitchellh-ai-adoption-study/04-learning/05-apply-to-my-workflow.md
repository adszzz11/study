# 05. 나의 워크플로우에 적용하기

## 📌 핵심 개념

Hashimoto의 6단계 여정을 **나의 현재 상황에 맞게 커스터마이징**하는 실습 노트다. 각 섹션의 질문에 답하면서 자신만의 AI 도입 로드맵을 설계한다.

---

## 자가 진단: 나는 어디에 있는가?

### 현재 단계 파악

```
□ Step 0: AI를 아직 사용하지 않음
□ Step 1: 챗봇을 사용하지만 에이전트는 아직
□ Step 2: 에이전트를 사용하지만 전문성 부족
□ Step 3: 비동기 에이전트를 활용하기 시작
□ Step 4: 확실한 작업을 위임하고 있음
□ Step 5: Harness Engineering을 실천 중
□ Step 6: 항상 에이전트가 돌아가고 있음
```

### 현재 사용 중인 도구

```
나의 AI 도구:
  □ ChatGPT / Claude (챗봇)
  □ GitHub Copilot (자동완성)
  □ Claude Code (터미널 에이전트)
  □ Cursor / Windsurf (IDE 에이전트)
  □ Amp / Codex (에이전트 플랫폼)
  □ 기타: ___________
```

---

## Phase 1: 에이전트 전환 (Step 0-1 → Step 2)

> 아직 챗봇만 사용 중이라면

### Action Items

```
Week 1: 에이전트 설치 및 첫 경험
  □ Claude Code 설치: npm install -g @anthropic-ai/claude-code
  □ 프로젝트 디렉토리에서 실행: claude
  □ 간단한 작업 시도: "이 프로젝트의 구조를 설명해줘"

Week 2: "두 번 일하기" 시작
  □ 작은 작업(버그 수정, 유틸 함수 추가)을 수동 → 에이전트 순서로
  □ 차이점 기록 (에이전트 강점/약점)
  □ 프롬프팅 기법 발견 (좁은 범위, 검증 가능한 요청)

Week 3-4: AGENTS.md/CLAUDE.md 작성 시작
  □ 프로젝트 루트에 파일 생성
  □ 에이전트 실수마다 한 줄 추가
  □ 빌드/테스트 명령어, 코드 스타일, 금지 행동 정리
```

### 성공 기준

- 에이전트가 잘하는 작업 3가지를 스스로 발견
- 에이전트가 못하는 작업 3가지를 스스로 발견
- AGENTS.md에 10줄 이상의 규칙이 축적

---

## Phase 2: 비동기 활용 (Step 2 → Step 3-4)

> 에이전트를 사용하지만 효율 개선을 느끼지 못한다면

### Action Items

```
Week 1: 퇴근 전 에이전트 루틴 시작
  □ 매일 퇴근 30분 전 알람 설정
  □ 다음 날을 위한 리서치 작업 1개 시작
  □ 결과를 아침에 리뷰하는 습관 형성

Week 2: Slam Dunk 식별 훈련
  □ 이슈 리스트에서 "에이전트가 해결할 수 있는 것" 태그
  □ 태그한 이슈 중 1개를 에이전트에 위임
  □ 결과를 리뷰하고 성공/실패 분석

Week 3-4: 알림 끄기 + 자연스러운 전환점 찾기
  □ 에이전트 알림 OFF
  □ 자신의 집중 사이클(포모도로 등)에 맞춰 확인 타이밍 설정
  □ 인터럽트 빈도 기록 (줄어드는지 확인)
```

### 성공 기준

- 아침에 warm start로 시작하는 날이 50% 이상
- 에이전트가 처리한 이슈의 성공률 70% 이상
- 컨텍스트 스위칭 횟수 감소 체감

---

## Phase 3: Harness Engineering (Step 4 → Step 5-6)

> 에이전트를 일상적으로 사용하고 있다면

### Action Items

```
Week 1-2: 실수 추적 시작
  □ 에이전트 실수를 분류: 빌드/스타일/로직/범위초과
  □ 각 실수에 대한 AGENTS.md 규칙 추가
  □ 반복되는 실수 패턴에 대한 스크립트 작성

Week 3-4: 도구 생태계 구축
  □ 스크린샷 스크립트 (UI 변경 검증)
  □ 필터된 테스트 실행 스크립트
  □ 린트/타입체크 원커맨드 스크립트
  □ AGENTS.md에 스크립트 사용법 추가

Ongoing: 지속적 개선
  □ 주 1회 AGENTS.md 리뷰 → 불필요한 규칙 제거, 새 규칙 추가
  □ 에이전트 성공률 추적 (감각적으로라도)
  □ Anti-Slop 세션 주 2~3회
```

### 성공 기준

- AGENTS.md 규칙 50줄 이상
- 검증 스크립트 3개 이상
- 에이전트 first-try 성공률 체감적으로 향상

### 프로젝트별 AI 사용 기준

```
나의 프로젝트 분류:

프로젝트 A (장기/중요):
  - 이름: ___________
  - 리뷰 수준: □ 모든 라인 □ 변경 파일만 □ 테스트 통과만
  - AGENTS.md: □ 상세 □ 기본 □ 없음
  - AI 공개: □ 필수 □ 선택 □ 불필요

프로젝트 B (단기/실험):
  - 이름: ___________
  - 리뷰 수준: □ 모든 라인 □ 변경 파일만 □ 테스트 통과만
  - AGENTS.md: □ 상세 □ 기본 □ 없음
  - AI 공개: □ 필수 □ 선택 □ 불필요
```
