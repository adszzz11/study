# oh-my-claudecode - 개요

## 1. What - OMC란?

> **한 줄 정의**: Claude Code에 32개 전문 에이전트와 스마트 모델 라우팅을 추가하는 오픈소스 확장 레이어

### 핵심 개념

OMC는 Claude Code CLI를 "더 똑똑하게" 사용하게 해주는 레이어다. 작업의 복잡도를 자동으로 판단하여 간단한 작업은 Haiku(저비용)로, 복잡한 작업은 Opus(고성능)로 라우팅한다. 32개의 역할별 전문 에이전트가 작업을 분담하며, 최대 5개까지 병렬 실행할 수 있다.

### 주요 용어

| 용어 | 설명 |
|------|------|
| Smart Model Routing | 작업 복잡도에 따라 Haiku/Sonnet/Opus 자동 선택 |
| Specialized Agents | 32개 역할별 전문 에이전트 (코딩, 리뷰, 테스트 등) |
| Parallel Autopilot | 최대 5개 워커 동시 실행 |
| Socratic Mode | 소크라테스식 질문으로 요구사항 정교화 |
| Skills | 40+ 내장 스킬 (TDD, 디버깅, 리팩토링 등) |
| Token Optimization | 30-50% 토큰 사용량 절감 |

### 동작 방식

```
사용자 입력
    ↓
[요구사항 분석] → Socratic Mode (필요시 질문)
    ↓
[복잡도 판단] → Model Routing
    │
    ├── 간단 → Haiku (빠르고 저렴)
    ├── 중간 → Sonnet (균형)
    └── 복잡 → Opus (고성능)
    ↓
[에이전트 선택] → 32개 중 적합한 에이전트
    ↓
[병렬 실행] → 최대 5 workers
    ↓
결과 통합
```

## 2. Why - 왜 OMC인가?

### 해결하려는 문제

- Claude Code를 Opus로만 돌리면 토큰 비용이 과다
- 모든 작업을 하나의 세션에서 순차 처리하면 느림
- 작업 유형에 맞는 프롬프트를 매번 작성하기 번거로움

### 기존 방식의 한계

| 문제 | 기존 방식 (Claude Code) | OMC |
|------|----------------------|-----|
| 모델 선택 | 수동으로 --model 지정 | 자동 라우팅 |
| 비용 | 항상 동일 모델 사용 | 복잡도별 최적 모델 |
| 프롬프트 | 매번 직접 작성 | 40+ 스킬 자동 적용 |
| 병렬 처리 | 단일 세션 | 최대 5 워커 |
| 요구사항 | 직접 구체화 | Socratic Mode |

## 3. 핵심 특징

### 장점

- **토큰 절감 30-50%**: 모델 라우팅으로 실질적 비용 절감
- **제로 러닝커브**: Claude Code CLI를 알면 바로 사용 가능
- **32개 전문 에이전트**: 코딩, 리뷰, 테스트, 문서화 등 역할 분담
- **Socratic Mode**: 요구사항이 모호할 때 질문으로 구체화
- **무료 오픈소스**: 커뮤니티 주도 발전

### 단점

- **Claude Code 전용**: Gemini CLI, Codex와 함께 사용 불가
- **격리 기능 없음**: git worktree 같은 작업 격리 미제공
- **Anthropic 종속**: Claude 모델만 사용 (다른 LLM 미지원)
- **커뮤니티 규모**: 아직 초기 단계 커뮤니티

## 4. 사용 사례

### 적합한 경우

| 사용 사례 | 설명 |
|----------|------|
| 비용 최적화 | Claude Max/API 비용을 줄이고 싶을 때 |
| 다양한 작업 혼합 | 간단한 수정 + 복잡한 리팩토링이 섞여 있을 때 |
| 워크플로우 자동화 | TDD, 코드리뷰 등 반복 패턴 자동화 |
| 팀 표준화 | 에이전트 사용 방식을 팀 전체에 통일 |
