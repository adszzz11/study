---

## 2. 대안 도구 비교

| 비교 항목 | OMC | Superpowers (기본 Skills) | 직접 프롬프트 |
|-----------|-----|--------------------------|-------------|
| 모델 라우팅 | ✓ 자동 | ✗ | ✗ 수동 |
| 전문 에이전트 | 32개 | 제한적 | 없음 |
| 토큰 절감 | 30-50% | - | - |
| 병렬 실행 | 5 워커 | 서브에이전트 | 수동 |
| 러닝커브 | 낮음 | 낮음 | 높음 (숙달 필요) |
| 커스터마이즈 | 높음 | 중간 | 무한 |

### OMC만의 강점

- **비용 효율성**: 유일하게 모델 라우팅을 자동화하는 도구
- **스킬 라이브러리**: 40+ 내장 스킬로 즉시 생산성 향상
- **Socratic Mode**: 다른 도구에 없는 요구사항 정교화 기능

### OMC의 약점

- Claude Code 전용 (다른 에이전트 미지원)
- git worktree 격리 없음 (Conductor, Squad 대비)

---

## 3. 함께 사용하면 좋은 도구

| 도구 | 역할 | 연동 방식 |
|------|------|----------|
| Claude Code | 기반 CLI | OMC가 래핑하여 실행 |
| cmux | 터미널 관리 | cmux 탭에서 OMC 실행 |
| Claude Squad | 세션 격리 | Squad worktree에서 OMC 사용 |
| CLAUDE.md | 프로젝트 컨텍스트 | OMC가 자동으로 참조 |

---

## 4. 트렌드

- **모델 라우팅**이 향후 에이전트 도구의 기본 기능이 될 전망
- Claude Code 자체도 모델 선택 기능을 강화하는 추세
- 커뮤니티 주도 스킬/에이전트 생태계 확장 중
