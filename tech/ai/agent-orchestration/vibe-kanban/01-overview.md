# Vibe Kanban - 개요

---

## 1. What - Vibe Kanban이란?

> **한 줄 정의**: 칸반 보드 UI로 AI 에이전트를 관리하고 내장 diff 리뷰어로 결과를 리뷰하는 오케스트레이션 플랫폼

### 핵심 개념

Vibe Kanban은 프로젝트 관리(칸반) + 코드 리뷰(diff viewer) + 에이전트 오케스트레이션을 하나로 합친 도구다. 각 에이전트 작업을 칸반 카드로 시각화하고, 작업이 완료되면 GitHub PR처럼 diff를 리뷰한 후 머지한다. BloopAI(YC 졸업)가 개발했으며, 크로스 플랫폼을 지원한다.

### 주요 용어

| 용어 | 설명 |
|------|------|
| Board | 칸반 보드 — 모든 에이전트 작업을 카드로 시각화 |
| Card | 개별 에이전트 작업 단위 |
| Diff Viewer | GitHub PR 스타일 코드 변경 리뷰 인터페이스 |
| Worktree | git worktree로 격리된 에이전트 작업 환경 |
| Web UI | 브라우저에서 접근하는 관리 인터페이스 |

### 동작 방식

```
Vibe Kanban (Web UI + CLI)
│
├── 칸반 보드
│   ├── [To Do] 카드 1: "API 엔드포인트 추가"
│   ├── [In Progress] 카드 2: "테스트 작성" → Agent (worktree-2)
│   ├── [In Progress] 카드 3: "리팩토링" → Agent (worktree-3)
│   └── [Review] 카드 4: "버그 수정" → Diff Viewer로 리뷰
│
└── Diff Viewer
    ├── 파일별 변경사항 (GitHub PR 스타일)
    ├── 승인 → 메인 브랜치 머지
    └── 수정 요청 → 에이전트 재작업
```

## 2. Why - 왜 Vibe Kanban인가?

### 해결하려는 문제

- 에이전트가 만든 코드를 체계적으로 리뷰할 방법 필요
- 여러 에이전트 작업의 진행 상황을 시각적으로 파악하고 싶음
- 프로젝트 관리와 에이전트 관리를 통합하고 싶음

### 기존 방식의 한계

| 문제 | 기존 방식 | Vibe Kanban |
|------|----------|-------------|
| 작업 시각화 | 터미널 목록 | 칸반 보드 |
| 코드 리뷰 | git diff 수동 | 내장 diff 리뷰어 |
| 프로젝트 관리 | 별도 도구 (Jira 등) | 칸반 통합 |
| 머지 워크플로우 | 수동 git merge | 리뷰 → 원클릭 머지 |

---

## 3. 핵심 특징

### 장점

- **칸반 보드**: 에이전트 작업을 프로젝트 관리 관점으로 시각화
- **내장 diff 리뷰어**: GitHub PR처럼 변경사항을 리뷰하고 승인/거절
- **크로스 플랫폼**: macOS, Linux, Windows 모두 지원
- **Web UI**: 브라우저에서 접근 — 원격 관리 가능
- **git worktree 격리**: 에이전트 간 충돌 방지
- **무료 오픈소스**: Apache 2.0

### 단점

- **상대적 복잡성**: 칸반 + 리뷰어 + 에이전트를 모두 이해해야 함
- **Web UI 의존**: 브라우저 기반이라 터미널 순수주의자에게는 오버헤드
- **상대적으로 새로움**: 커뮤니티 규모가 아직 작음
- **모델 라우팅 없음**: OMC 같은 비용 최적화 미제공

## 4. 사용 사례

### 적합한 경우

| 사용 사례 | 설명 |
|----------|------|
| 팀 프로젝트 | 여러 멤버가 에이전트 결과를 리뷰해야 할 때 |
| 코드 리뷰 중심 | 에이전트 결과물의 품질 검증이 중요할 때 |
| 대규모 마이그레이션 | 여러 모듈을 칸반 카드로 관리하며 순차 처리 |
| 원격 작업 | Web UI로 어디서든 에이전트 관리 |
