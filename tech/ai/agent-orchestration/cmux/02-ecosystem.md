## 2. 대안 도구 비교

| 비교 항목 | cmux | tmux | iTerm2 |
|-----------|------|------|--------|
| 주요 특징 | AI 에이전트 특화 | 범용 멀티플렉서 | macOS 터미널 |
| AI 특화 UX | ✓ (배지, 알림) | ✗ | ✗ |
| 소켓 API | ✓ | ✗ (직접 구현) | AppleScript |
| 내장 브라우저 | ✓ | ✗ | ✗ |
| 원격 접속 | ✗ | ✓ (SSH) | ✗ |
| 러닝커브 | 낮음 | 높음 | 낮음 |
| 가격 | 무료 (OSS) | 무료 (OSS) | 무료 |

### cmux만의 강점

- **AI 에이전트 전용 UX** — 일반 터미널에는 없는 에이전트 모니터링 기능
- **에이전트 무관** — 특정 에이전트에 종속되지 않는 유일한 터미널
- **소켓 API** — 프로그래밍 방식 자동화

### cmux의 약점

- 격리/오케스트레이션 기능 없음 (Claude Squad, Conductor 대비)
- tmux처럼 원격 세션 지원 불가
