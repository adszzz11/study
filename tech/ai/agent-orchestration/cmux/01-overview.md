## 1. What - cmux란?

> **한 줄 정의**: 모든 AI 코딩 에이전트를 위한 네이티브 macOS 터미널 멀티플렉서

### 핵심 개념

cmux는 "AI 에이전트를 위한 터미널"이라는 컨셉으로, iTerm2나 기본 터미널 대신 사용하는 전용 터미널 앱이다. 일반 터미널과의 핵심 차이는 AI 에이전트 워크플로우에 특화된 UX — 수직 탭, 알림 배지, 분할 패널, 내장 브라우저, 소켓 API 등을 제공한다.

### 주요 용어

| 용어 | 설명 |
|------|------|
| Vertical Tabs | 수직 탭 — 여러 에이전트 세션을 좌측에 나열 |
| Split Panes | 화면 분할로 여러 에이전트를 동시에 모니터링 |
| Unread Badges | 에이전트 출력이 있으면 알림 표시 |
| Socket API | 외부에서 cmux를 프로그래밍 방식으로 제어 |
| Embedded Browser | 터미널 내 브라우저로 문서 참조 가능 |

### 동작 방식

```
cmux 터미널 앱
├── 탭 1: Claude Code (프로젝트 A)
├── 탭 2: Gemini CLI (프로젝트 A - 다른 작업)
├── 탭 3: Codex (프로젝트 B)
├── 탭 4: Aider (프로젝트 C)
└── 내장 브라우저: 문서 참조
     ↑
     소켓 API로 외부 제어 가능
```

## 2. Why - 왜 cmux인가?

### 해결하려는 문제

- 여러 AI 에이전트를 동시에 돌릴 때 터미널 관리가 복잡
- 특정 에이전트에 종속되지 않는 범용 도구 필요
- 에이전트 출력을 놓치지 않고 모니터링해야 함

### 기존 방식의 한계

| 문제 | 기존 방식 | cmux |
|------|----------|------|
| 다중 세션 관리 | iTerm2 탭 / tmux | AI 에이전트 전용 UX |
| 알림 | 없음 (직접 확인) | 읽지 않은 출력 배지 |
| 에이전트 호환성 | 도구별로 다름 | 모든 CLI 에이전트 지원 |
| 문서 참조 | 브라우저 전환 | 내장 브라우저 |
| 자동화 | 스크립트 직접 작성 | 소켓 API |

## 3. 핵심 특징

### 장점

- **에이전트 무관**: Claude Code, Gemini CLI, Codex, Aider, Kiro 등 모든 CLI 도구 사용 가능
- **네이티브 macOS**: 빠른 성능, OS 통합
- **알림 시스템**: 에이전트 출력 시 배지/알림으로 놓치지 않음
- **소켓 API**: 프로그래밍 방식으로 자동화 가능
- **무료 오픈소스**: 비용 부담 없음

### 단점

- **macOS 전용**: Linux/Windows 미지원
- **격리 없음**: git worktree 같은 작업 격리 기능 미제공 (순수 터미널)
- **오케스트레이션 부재**: 작업 분배/관리 기능 없음 — 단순 터미널 멀티플렉서
- **상대적으로 새로운 프로젝트**: 안정성 검증 필요

---

## 4. 사용 사례

### 적합한 경우

| 사용 사례 | 설명 |
|----------|------|
| 다양한 에이전트 동시 사용 | Claude Code + Gemini CLI + Codex를 한 화면에서 |
| 모니터링 중심 워크플로우 | 여러 에이전트의 진행 상황을 실시간 확인 |
| 에이전트 비교 테스트 | 같은 작업을 다른 에이전트에게 주고 결과 비교 |
| 자동화 파이프라인 | 소켓 API로 외부에서 에이전트 제어 |
