# Cloudflare R2 개요

## 핵심 개념

### R2란?
Cloudflare R2는 S3 호환 오브젝트 스토리지 서비스입니다. "R2"라는 이름은 AWS의 S3에서 각 문자를 하나씩 앞당긴 것에서 유래했습니다(S→R, 3→2).

### 오브젝트 스토리지 기본 개념

```
┌─────────────────────────────────────────┐
│              Cloudflare R2              │
│  ┌─────────────────────────────────┐    │
│  │         Bucket (버킷)            │    │
│  │  ┌─────────────────────────┐    │    │
│  │  │   Object (객체)          │    │    │
│  │  │  ┌─────────────────┐    │    │    │
│  │  │  │ Key (키/경로)   │    │    │    │
│  │  │  │ Data (데이터)   │    │    │    │
│  │  │  │ Metadata       │    │    │    │
│  │  │  └─────────────────┘    │    │    │
│  │  └─────────────────────────┘    │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

- **버킷(Bucket)**: 객체를 담는 컨테이너, 전역적으로 고유한 이름 필요
- **객체(Object)**: 저장되는 데이터 단위 (파일 + 메타데이터)
- **키(Key)**: 버킷 내 객체의 고유 식별자 (경로처럼 사용)

### R2 아키텍처

```
┌──────────────┐     ┌─────────────────────────────────────────┐
│   Client     │────▶│         Cloudflare Edge Network         │
└──────────────┘     │  ┌─────────┐   ┌─────────────────────┐  │
                     │  │ Workers │◀─▶│         R2          │  │
                     │  │ (선택)  │   │   Object Storage    │  │
                     │  └─────────┘   └─────────────────────┘  │
                     │        ▲                                 │
                     │        │ S3 API                         │
                     │        ▼                                 │
                     │  ┌─────────────────────────────────┐    │
                     │  │   S3 Compatible API Endpoint    │    │
                     │  │ <ACCOUNT_ID>.r2.cloudflarestorage.com│
                     │  └─────────────────────────────────┘    │
                     └─────────────────────────────────────────┘
```

## 핵심 장점

### 1. 이그레스(Egress) 비용 제로
가장 큰 차별점입니다. AWS S3의 경우 데이터를 다운로드할 때마다 비용이 발생하지만, R2는 **데이터 전송 비용이 무료**입니다.

```
예시: 월 1TB 데이터 다운로드 시
├── AWS S3:  ~$90/월 (이그레스 비용)
├── R2:      $0/월
└── 절감액:  $90/월 (연 $1,080 절감)
```

### 2. S3 API 100% 호환
기존에 S3를 사용하던 애플리케이션을 최소한의 수정으로 R2로 마이그레이션할 수 있습니다.

```python
# 엔드포인트만 변경하면 됨
# AWS S3
s3_client = boto3.client('s3')

# Cloudflare R2
r2_client = boto3.client('s3',
    endpoint_url='https://<ACCOUNT_ID>.r2.cloudflarestorage.com',
    aws_access_key_id='<R2_ACCESS_KEY>',
    aws_secret_access_key='<R2_SECRET_KEY>'
)
```

### 3. Workers 네이티브 통합
Cloudflare Workers와 바인딩을 통해 엣지에서 직접 R2에 접근할 수 있습니다.

```javascript
// Workers에서 R2 직접 접근
export default {
  async fetch(request, env) {
    const object = await env.MY_BUCKET.get("my-file.txt");
    return new Response(object.body);
  }
}
```

### 4. 넉넉한 무료 티어

| 항목 | 무료 한도 |
|------|----------|
| 저장 공간 | 10GB/월 |
| Class A 연산 | 1,000만 회/월 (PUT, POST, LIST 등) |
| Class B 연산 | 1억 회/월 (GET, HEAD 등) |
| 이그레스 | 무제한 무료 |

### 5. 글로벌 성능
Cloudflare의 글로벌 네트워크(310+ 도시)를 활용하여 전 세계 어디서든 빠른 접근이 가능합니다.

## 단점 및 제한사항

### 1. 기능적 제한
```
지원하지 않는 S3 기능:
├── 객체 잠금 (Object Lock)
├── 버전 관리 (Versioning) - 베타 지원 중
├── 복제 (Replication)
├── 서버 측 암호화 커스텀 키 (SSE-C)
└── 일부 ACL 기능
```

### 2. 지역 제한
- 단일 지역 저장 (자동 글로벌 복제 없음)
- 특정 리전 선택 불가 (Cloudflare가 자동 결정)

### 3. 생태계
- AWS S3 대비 서드파티 도구 지원이 적음
- 일부 S3 전용 서비스와 호환성 이슈 가능

### 4. 대용량 파일 제한
- 단일 객체 최대 크기: 5TB
- 멀티파트 업로드 파트 크기: 5MB ~ 5GB

## 사용 사례

### 적합한 경우

#### 1. 정적 자산 호스팅
```
웹사이트 이미지, CSS, JavaScript 등
├── 장점: 이그레스 무료로 트래픽 비용 절감
├── 장점: Workers로 이미지 최적화 가능
└── 예시: 블로그, 포트폴리오, 랜딩 페이지
```

#### 2. 미디어 스트리밍/다운로드
```
동영상, 팟캐스트, 소프트웨어 배포
├── 장점: 다운로드 비용 $0
├── 장점: 대용량 파일 처리 가능
└── 예시: VOD 서비스, 파일 호스팅
```

#### 3. 백업 및 아카이브
```
데이터 백업, 로그 저장
├── 장점: 저렴한 저장 비용 ($0.015/GB/월)
├── 장점: 복원 시 이그레스 무료
└── 예시: 데이터베이스 백업, 로그 아카이브
```

#### 4. CDN 원본 스토리지
```
글로벌 콘텐츠 배포의 원본
├── 장점: Cloudflare CDN과 네이티브 통합
├── 장점: 오리진 비용 최소화
└── 예시: 글로벌 서비스, 게임 자산 배포
```

### 부적합한 경우

| 시나리오 | 이유 | 대안 |
|---------|------|------|
| 트랜잭션 데이터 | 오브젝트 스토리지는 ACID 미지원 | RDB, D1 |
| 실시간 협업 | 파일 락킹 미지원 | Google Drive, Dropbox |
| 버전 관리 필수 | 버전관리 제한적 | AWS S3, Git LFS |
| 특정 리전 필수 | 리전 선택 불가 | AWS S3, GCS |

## 가격 정책

### 기본 가격 (2024년 기준)

| 항목 | 가격 |
|------|------|
| 저장 | $0.015/GB/월 |
| Class A 연산 | $4.50/1백만 회 |
| Class B 연산 | $0.36/1백만 회 |
| 이그레스 | **$0 (무료)** |

### Class A vs Class B 연산

```
Class A (쓰기 연산) - $4.50/백만
├── PutObject
├── CopyObject
├── CreateMultipartUpload
├── UploadPart
├── CompleteMultipartUpload
└── ListBuckets, ListObjects

Class B (읽기 연산) - $0.36/백만
├── GetObject
├── HeadObject
└── GetObjectAttributes
```

### 비용 예시

```
시나리오: 중소규모 웹사이트
├── 저장: 100GB = $1.50/월
├── 업로드: 10만 회 (Class A) = $0.45/월
├── 다운로드: 500만 회 (Class B) = $1.80/월
├── 이그레스: 500GB = $0/월
└── 총계: ~$3.75/월

동일 조건 AWS S3:
├── 저장: 100GB = $2.30/월
├── 요청: ~$2.50/월
├── 이그레스: 500GB = ~$45/월
└── 총계: ~$50/월
```
