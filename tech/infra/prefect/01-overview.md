# Part 1: 개요

## 1.1 정의 및 핵심 개념

**Prefect**는 Python 함수를 프로덕션급 데이터 파이프라인으로 변환하는 워크플로우 오케스트레이션 프레임워크입니다.

### 핵심 구성요소

| 개념             | 설명                        | Git 비유       |
| -------------- | ------------------------- | ------------ |
| **Flow**       | 워크플로우의 컨테이너, 최상위 함수       | Repository   |
| **Task**       | 개별 작업 단위, 재사용 가능          | Commit       |
| **Deployment** | Flow를 실행 가능한 상태로 패키징      | Release      |
| **Work Pool**  | 실행 인프라 정의 (K8s, Docker 등) | CI/CD Runner |

### 개념 다이어그램

```
Flow (전체 파이프라인)
  └── Task A (데이터 추출)
  └── Task B (데이터 변환)
  └── Task C (데이터 적재)
```

---

## 1.2 왜 Prefect인가?

### 장점

| 장점 | 설명 |
|------|------|
| **간단함** | Python 함수에 데코레이터만 추가하면 끝 |
| **유연함** | 로컬 개발 → 프로덕션 배포까지 동일한 코드 |
| **강력한 실패 처리** | 자동 재시도, 상태 추적, 알림 |
| **하이브리드 실행** | 클라우드 컨트롤 플레인 + 자체 인프라 실행 |
| **Prefect 3.0** | 2024년 출시, 런타임 오버헤드 90% 감소 |

### 단점

| 단점 | 설명 |
|------|------|
| **데이터 리니지** | Dagster 대비 약함 |
| **엔터프라이즈 기능** | 유료 Prefect Cloud에 집중 |
| **커뮤니티 규모** | Airflow 대비 작은 커뮤니티/에코시스템 |

## 1.3 주요 사용 사례

### 적합한 경우

- **ETL/ELT 데이터 파이프라인**: 데이터 추출, 변환, 적재 자동화
- **ML 모델 학습/배포 워크플로우**: MLOps 파이프라인 구축
- **이벤트 기반 자동화**: 특정 이벤트 발생 시 작업 트리거
- **정기 리포트 생성**: 스케줄 기반 데이터 리포트
