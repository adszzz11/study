---

## 디자인시스템 구성 요소

### 필수 토큰
- **Primitive Token**: RGBA 값 (기본 색상)
- **Semantic Token**: 의미 기반 정의 (primary, secondary, error 등)

### 컴포넌트 정의
- 버튼 (Button): Primary, Secondary, Tertiary, Ghost
- 입력 필드 (Input): Text, Password, Search
- 카드 (Card): 기본, 리스트, 상세
- 모달 (Modal): Alert, Confirm, Bottom Sheet
- 네비게이션: Tab Bar, Navigation Bar

---

## 개발자들이 디자인에게 원하는 것

| 요청 파트 | 핵심 요청 |
|----------|----------|
| **FE** | 디자인시스템 기반, 라이브러리 재현 가능 수준, Token 일관성 |
| **iOS** | 토큰화, 컴포넌트 통일, 구현 시간 고려, Animation 조율 |
| **AOS** | **Android 특성 고려**, 폴더블 대응, 플랫폼별 맞춤 디자인 |

---

## DO (해주세요)

- **디자인시스템 기반 구성을 먼저 진행**
- **Theme, Token을 일관성 있게 구성**
- **라이브러리에서 재현 가능한 수준의 디자인** (Material UI, Ant Design, shadcn/ui 참고)
- **구현 시간 고려한 대안 제시**
- **Android 특성 고려** (폴더블, 해상도 다양성)

---

## DON'T (피해주세요)

- **iPhone 디자인을 Android에 그대로 적용**
- **폴더블(Fold) 기기 미고려**
- 과도한 Animation 요구 (구현 시간 미고려)
- 디자인시스템과 맞지 않는 새 컨셉 무작정 추가
- 너무 컬러풀한 디자인 (통일성 저해)

---

## Android 기기 다양성 대응 (중요)

AOS 개발자가 특히 강조한 부분

| 기기 유형 | 대응 필요 수준 |
|----------|---------------|
| 일반 스마트폰 | 기본 |
| **폴더블 (Fold)** | **필수** |
| 플립 | 선택 |
| 특수 해상도 | 케이스별 협의 |

### AOS 개발자의 요청

Android 플랫폼의 특성상 폴더블 기기와 다양한 화면 크기를 반드시 고려해주시기 바랍니다.

---

## 권장 UI 라이브러리 (참고용)

디자이너가 참고하면 협업이 수월해지는 라이브러리:
- Material UI
- Ant Design
- shadcn/ui
- App UI

---

## 디자이너의 협조 요청 사항

- 시장조사, 설문 신뢰도가 낮을 경우 기한이 늘어날 수 있음
- 프로젝트 초반에는 탐색적 작업이 많아 유연한 일정 관리 필요
- 반복적인 수정 작업이 발생할 수 있음을 고려
- UI/UX 의사결정은 디자이너 주도로 진행
