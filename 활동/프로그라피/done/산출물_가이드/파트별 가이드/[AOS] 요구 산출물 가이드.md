# AOS (Android) 파트 요구 산출물 가이드

> AOS가 다른 파트에 요청하는 산출물 및 협업 가이드

---

## 핵심 역할

- Android 앱 개발
- 다양한 기기 대응 (폴더블, 다양한 해상도)
- API 연동

## 다른 파트에 요청하는 산출물

### 기획/PO에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 디테일한 Wireframe | 🔴 필수 | 기획 의도가 분명해야 함 |

**기획에게 요청사항:**
- [ ] 기획 의도를 명확하게 표현 (맞고 틀리고 상관없이)
- [ ] **테스트베드 이후 기획 변경 지양** (중요!)
- [ ] 근거 없는 고집 지양

### 디자인에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 프로토타입 | 🔴 필수 | WireFrame 포함 |
| 일관된 화면 디자인 | 🔴 필수 | 통일된 UI |
| Animation Diagram | 🟡 권장 | 인터랙션 정의 |

**디자인에게 요청사항:**
- [ ] **Android 특성 고려** (iOS와 다름!)
- [ ] **폴더블(Fold) 기기 고려**
- [ ] **다양한 화면 크기 고려** (넓은 화면 / 좁은 화면)
- [ ] 디자인 통일성 유지
- [ ] iPhone 디자인 그대로 적용 지양

### 백엔드에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| API 문서 (Swagger/Docs) | 🔴 필수 | API 연동 |

**백엔드에게 요청사항:**
- [ ] **nullable 표시 정확하게** (중요!)
- [ ] 배포 시 커뮤니케이션
- [ ] Entity 값을 원형으로 최대한 내려주기
- [ ] Schema 과도하게 쪼개지 않기
- [ ] 배포 관련 소통 없이 변경 지양

### QA에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| QA 워크플로우 | 🟡 권장 | 업무 프로세스 정의 |
| 테스트 케이스 | 🔴 필수 | 기능별 테스트 항목 |
| 테스트 결과 | 🔴 필수 | 실행 결과 리포트 |

**QA에게 요청사항:**
- [ ] 구체적인 버그 리포트 (**기기 명시!**)
- [ ] **모든 파트에 균등한 테스트** (특정 파트만 누락 X)
- [ ] 해상도 차이로 인한 이슈 고려
- [ ] "안 돼요"만 말하지 않기 (기기/상황 설명 포함)
- [ ] "기획에서는 저렇게 되어있는데"만 말하지 않기 (해상도 다양성 고려)

## 폴더블 기기 레이아웃 가이드

### 화면 상태별 대응

| 화면 상태 | 설명 | 대응 방안 |
|----------|------|----------|
| **Folded** | 외부 화면 (커버) | 핵심 기능만 표시 |
| **Unfolded** | 내부 큰 화면 | 2-pane 레이아웃 고려 |
| **Flex Mode** | 반쯤 접힌 상태 | 상단/하단 분리 UI |
| **Tabletop** | 가로로 반 접힘 | 화상통화 등 최적화 |

### 폴더블 레이아웃 예시

```
[Folded - 외부 화면]
┌─────────────┐
│   핵심 정보  │
│   간략 표시  │
└─────────────┘

[Unfolded - 내부 화면]
┌───────────────────────────────┐
│         │                     │
│  목록   │      상세 정보      │
│  영역   │        영역         │
│         │                     │
└───────────────────────────────┘

[Flex Mode - 반쯤 접힌 상태]
┌─────────────────┐
│    컨텐츠 영역   │ ← 시청 영역
├─────────────────┤ ← 힌지 라인
│    컨트롤 영역   │ ← 조작 영역
└─────────────────┘
```

### Jetpack WindowManager 활용

```kotlin
// 폴더블 상태 감지
lifecycleScope.launch {
    WindowInfoTracker.getOrCreate(this@Activity)
        .windowLayoutInfo(this@Activity)
        .collect { layoutInfo ->
            val foldingFeature = layoutInfo.displayFeatures
                .filterIsInstance<FoldingFeature>()
                .firstOrNull()

            when {
                foldingFeature == null -> handleNormalMode()
                foldingFeature.state == FoldingFeature.State.HALF_OPENED ->
                    handleFlexMode(foldingFeature)
                foldingFeature.state == FoldingFeature.State.FLAT ->
                    handleUnfoldedMode()
            }
        }
}
```
