# 요구사항 분류 요약 (Requirements Classification Summary)

> Assignment 요구사항을 "확실한 것"과 "모호한 것"으로 분류한 결과 요약

## 📋 개요

이 문서는 Agentic Document Scraper 과제의 모든 요구사항을 분석하고 분류한 결과를 요약합니다.

**분류 기준:**
- **확실한 것**: Assignment Description에 명시되어 즉시 구현 가능
- **모호한 것**: 추가 검토 또는 클라이언트 질의가 필요

---

## 📊 분류 결과 통계

| 분류 | 항목 수 | 상태 | 처리 방법 |
|------|---------|------|-----------|
| ✅ **확실한 요구사항** | 15개 | 확정 | 즉시 구현 가능 |
| 🔍 **기술적 검토 사항** | 7개 영역 | 검토 중 | 합리적 가정 하에 진행 |
| ❓ **클라이언트 질의 사항** | 20개 질문 | 대기 중 | 클라이언트 확인 필요 |

## ✅ 1. 확실한 요구사항 (Confirmed Requirements)

> **위치**: [`specs/1-확실한-요구사항/`](./specs/1-확실한-요구사항/)

| - Cherokee County | (비법인 지역) | ✅ |
| - City of Holly Springs | Georgia | ✅ |
| - City of Alpharetta | Georgia | ✅ |
| - City of Marietta | Georgia | ✅ |
| **문서 타입** | 4가지 | 100% |

## 🔍 2. 기술적 검토 사항 (Technical Review Items)

> **위치**: [`specs/2-검토-필요-사항/기술적-검토-사항.md`](./specs/2-검토-필요-사항/기술적-검토-사항.md)

### 개요
합리적인 가정 하에 진행 가능하나, 기술적 의사결정과 트레이드오프 고려가 필요한 사항들입니다.

### 주요 검토 영역

| # | 검토 항목 | 선택지 수 | 제안 | 상태 |
|---|----------|-----------|------|------|
| 1 | **Agentic 시스템 구현 범위** | 3개 | Moderate Agentic (Hybrid) | ⭐⭐⭐⭐⭐ |
| 2 | **아키텍처 설계** | 3개 | Plugin Architecture | ⭐⭐⭐⭐⭐ |
| 3 | **LLM 통합 방식** | 다수 | Claude 3.5 Sonnet + Hybrid 호출 | ⭐⭐⭐⭐⭐ |
| 4 | **데이터 저장 방식** | 다수 | Local → S3, SQLite → PostgreSQL | ⭐⭐⭐⭐ |
| 5 | **에러 핸들링 전략** | 6개 | Exponential backoff + LLM 재분석 | ⭐⭐⭐⭐ |
| 6 | **스케줄링 방식** | 4개 | Polling (매일 1회) | ⭐⭐⭐⭐ |
| 7 | **테스트 전략** | 4개 | Integration 우선 + Mock HTML | ⭐⭐⭐⭐ |

### 핵심 제안

#### 2.1 Moderate Agentic (Hybrid 방식)
```
Phase 1: LLM이 사이트 구조 분석 → 규칙 생성
Phase 2: 규칙 기반으로 빠르게 실행
Phase 3: 실패 시 LLM 재개입 → 규칙 업데이트
```

**장점:**
- 비용 효율적 (초기: $20-40, 월간: $10-20)
- 빠른 실행 (규칙 기반)
- 적응성 있음 (사이트 변경 대응)

---

#### 2.2 Plugin Architecture
```
scraper/
├── core/                    # 공통 로직
├── jurisdictions/           # 지자체별 플러그인
├── utils/                   # 유틸리티
└── config/                  # 설정 파일
```

**장점:**
- 새 지자체 추가 용이
- 코드 재사용
- 테스트 및 유지보수 편리

## ❓ 3. 클라이언트 질의 사항 (Client Questions)

> **위치**: [`specs/2-검토-필요-사항/클라이언트-질의-사항.md`](./specs/2-검토-필요-사항/클라이언트-질의-사항.md)

### 개요
프로덕션 배포 전에 클라이언트에게 확인이 필요한 비즈니스 요구사항 및 운영 정책입니다.

### 우선순위별 분류

#### ⚡ CRITICAL (MVP 설계에 필수) - 4개

| # | 질문 | 영향 받는 설계 | 현재 가정 |
|---|------|----------------|-----------|
| 1 | **문서 활용 방법은?** | DB 스키마, 메타데이터 범위, OCR 필요성 | 검색 + AI 분석 |
| 2 | **문서 누락 허용 범위는?** | 검증 로직, 알림 수준, 재시도 전략 | 95% 이상 |
| 3 | **실시간성 요구사항은?** | Polling 주기, 인프라 비용, 알림 정책 | 다음 날까지 |
| 4 | **초기 Backfill 기간은?** | 작업량, 스토리지 비용, 실행 시간 | 1년 |

---

#### 📊 HIGH (구현 방향 결정) - 3개

| # | 질문 | 영향 받는 설계 | 현재 가정 |
|---|------|----------------|-----------|
| 5 | **향후 확장 계획은?** | 아키텍처 확장성, 자동화 수준 | 10개 이내 |
| 6 | **문서 타입 우선순위는?** | MVP 범위, 개발 순서 | Minutes > Agendas > Packets > Videos |
| 7 | **API 비용 예산은?** | LLM 사용 빈도, 캐싱 전략 | $100-500 |

---

#### 🔧 MEDIUM (운영 정책) - 13개

| 카테고리 | 질문 개수 | 예시 |
|----------|-----------|------|
| **운영** | 5개 | 알림 방법, 보관 기간, 중복 처리 등 |
| **인프라** | 4개 | 배포 환경, 시스템 통합, 동시 실행 등 |
| **법적/정책** | 3개 | robots.txt, Rate limiting, User-Agent |
| **유지보수** | 3개 | 사이트 변경 대응, 품질 검증, SLA 등 |

## 🎯 구현 전략

### Phase 1: MVP 개발 (클라이언트 질의 전에도 시작 가능)

#### 기본 가정
```markdown
1. 문서 활용: 검색 및 AI 분석
2. 문서 우선순위: Minutes > Agendas > Packets > Videos
3. Backfill 기간: 1년 (조정 가능하게 설계)
4. 배포 환경: 로컬 개발 (클라우드 이전 가능하게 설계)
5. 실시간성: 일배치 (매일 1회)
6. API 예산: 중간 수준 (Hybrid 방식)
7. 아키텍처: Plugin 구조
8. LLM: Claude 3.5 Sonnet
```

#### 개발 순서
1. ✅ **1개 지자체 선택** (예: Alpharetta)
2. ✅ **1개 문서 타입** (Minutes)
3. ✅ **최근 3-6개월** 데이터로 테스트
4. ✅ **Hybrid Agentic** 방식 검증
5. ✅ **비용 측정** 및 안정성 확인

## 🚀 즉시 시작 가능 범위

### ✅ 확정된 사항으로 즉시 개발 가능

1. **대상 범위**
   - 4개 지자체 웹사이트 구조 분석
   - 4가지 문서 타입 수집 로직 설계

2. **기본 인프라**
   - Python 프로젝트 구조 생성
   - 기본 스크래핑 프레임워크 (Playwright, BeautifulSoup)
   - 로컬 파일 시스템 저장 구조

3. **MVP 프로토타입**
   - 1개 지자체 (Alpharetta)
   - 1개 문서 타입 (Minutes)
   - Backfill 모드 (최근 3-6개월)

## 📈 예상 규모 및 비용

### 데이터 규모 (1년 Backfill 기준)

```
4개 지자체 × 12개월 × 월 2회 회의 = ~96개 회의
96개 회의 × 평균 3.5개 문서 = ~336개 파일
336개 파일 × 평균 5MB = ~1.7GB
```

### API 비용 (Hybrid 방식)

| 단계 | 비용 | 설명 |
|------|------|------|
| **초기 학습** | $20-40 | 4개 지자체 사이트 분석 |
| **Backfill** | $10-20 | 문서 분류 및 검증 |
| **월간 운영** | $10-20 | 규칙 검증 + 가끔 재학습 |
| **총 첫 달** | ~$50 | - |
