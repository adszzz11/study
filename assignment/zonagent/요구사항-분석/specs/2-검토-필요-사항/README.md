## 🔍 검토 필요 사항 분류

### 1. 기술적 검토 사항 (가정 하에 진행 가능)
**특징:** 합리적인 가정을 통해 구현을 진행할 수 있으나, 나중에 조정이 필요할 수 있음

**주요 항목:**
- "Agentic" 시스템의 구현 범위
- 아키텍처 설계 (Monolithic vs Plugin vs Full Agentic)
- LLM 통합 방식 (Claude vs OpenAI, 호출 시점, 프롬프트 전략)
- 데이터 저장 방식 (Local vs S3, SQLite vs PostgreSQL)
- 에러 핸들링 및 재시도 전략
- 스케줄링 방식 (Polling vs Webhook)
- 테스트 전략

### 2. 클라이언트 질의 사항 (확인 필요)
**특징:** 비즈니스 요구사항이나 운영 정책과 관련되어 클라이언트 확인이 필요

**우선순위별 분류:**

#### CRITICAL (MVP 설계에 영향)
1. 수집한 문서를 어떻게 활용하는가?
2. 문서 누락의 허용 범위는?
3. 실시간성 요구사항은?
4. 초기 Backfill 기간은?

#### HIGH (구현 방향 결정)
5. 향후 확장 계획은?
6. 문서 타입별 우선순위는?
7. API 비용 예산 가이드?

#### MEDIUM (운영 정책)
8. 스크래핑 실패 시 알림 방법?
9. 문서 보관 기간?
10. 중복 문서 처리?

## ⚖️ 가정(Assumptions) vs 질의(Questions) 기준

| 구분 | 가정 하에 진행 가능 | 반드시 질의 필요 |
|------|-------------------|-----------------|
| **기술 선택** | ✅ 기술 스택, 아키텍처 패턴 | ❌ 기존 시스템 통합 방식 |
| **데이터 구조** | ✅ 저장 포맷, DB 스키마 | ❌ 문서 활용 방법 |
| **운영 정책** | ✅ 로깅, 모니터링 기본 수준 | ❌ SLA, 알림 정책 |
| **범위** | ✅ 문서 타입 우선순위 (Minutes 우선) | ❌ 정확한 Backfill 기간 |

## 🎯 MVP 개발 전략

### 합리적 가정으로 진행 가능
다음 가정들을 바탕으로 MVP 개발을 시작할 수 있습니다:

```markdown
## 기본 가정 (MVP 개발용)

1. **문서 활용**: 검색 및 AI 분석 (메타데이터 추출 필요)
2. **문서 우선순위**: Minutes > Agendas > Packets > Videos
3. **Backfill 기간**: 1년 (조정 가능하게 설계)
4. **배포 환경**: 로컬 개발 (클라우드 이전 가능하게 설계)
5. **실시간성**: 일배치 (매일 1회)
6. **API 비용**: 중간 수준 (Hybrid Agentic 방식으로 최적화)
7. **아키텍처**: Plugin 구조 (확장성 확보)
8. **LLM**: Claude 3.5 Sonnet (긴 컨텍스트 처리)
```

### 클라이언트 질의가 필수인 경우
다음 4가지는 MVP 설계 전에 반드시 확인 필요:

1. **배포 환경** - 인프라 전체 설계에 영향
2. **문서 활용 방법** - DB 스키마 및 메타데이터 추출 범위 결정
3. **초기 Backfill 정확한 기간** - 작업량 및 스토리지 산정
4. **문서 타입 우선순위** - MVP 범위 결정
