## 1. "Agentic" 시스템 구현 범위

### 문제 정의
- "Agentic"의 정확한 정의가 모호함
- 얼마나 자율적이어야 하는가?
- LLM 활용 범위는?

### 선택지 비교

| 접근법 | 설명 | 장점 | 단점 | 추천도 |
|--------|------|------|------|--------|
| **Minimal Agentic** | LLM으로 문서 분류만 수행 | - 빠른 실행<br>- 낮은 비용 | - Agentic 특성 약함<br>- 사이트 변경에 취약 | ⭐⭐ |
| **Moderate Agentic** | LLM으로 페이지 구조 이해 + 규칙 생성 | - 비용 효율적<br>- 적응성 있음<br>- 안정적 | - 초기 설계 복잡도 중간 | ⭐⭐⭐⭐⭐ |
| **Full Agentic** | LLM이 모든 결정 수행 (자율 에이전트) | - 완전 자율적<br>- 새 사이트 쉽게 추가 | - 높은 비용<br>- 느린 속도<br>- 안정성 문제 | ⭐⭐⭐ |

### 제안: Moderate Agentic (Hybrid 방식)

**구현 전략:**
```
Phase 1: 초기 학습
- LLM이 사이트 구조 분석
- CSS Selector / XPath 규칙 생성
- 규칙을 JSON으로 저장

Phase 2: 규칙 기반 실행
- 저장된 규칙으로 빠르게 실행
- 비용 절감 및 속도 향상

Phase 3: 변경 감지 및 적응
- 규칙 실패 시 LLM 재개입
- 사이트 구조 변경 자동 감지
- 새로운 규칙 생성 및 업데이트
```

**예상 비용:**
- 초기 학습: $5-10 per jurisdiction
- 규칙 기반 실행: ~$0 (API 호출 없음)
- 재학습: $5-10 per change

## 3. LLM 통합 방식

### 결정 사항

#### 3.1 LLM 제공자 선택

| 제공자 | 장점 | 단점 | 추천 |
|--------|------|------|------|
| **Claude (Anthropic)** | - 200K 토큰 (긴 HTML 처리)<br>- 구조화된 출력<br>- 정확한 지시 따르기 | - 상대적으로 높은 비용 | ⭐⭐⭐⭐⭐ Primary |
| **GPT-4 Turbo (OpenAI)** | - 함수 호출 지원<br>- 넓은 생태계 | - 짧은 컨텍스트 (128K)<br>- HTML 처리 시 토큰 낭비 | ⭐⭐⭐ Fallback |
| **GPT-4o-mini** | - 저렴한 비용 | - 낮은 정확도 | ⭐⭐ 단순 작업용 |

**제안:** Claude 3.5 Sonnet (Primary) + GPT-4 Turbo (Fallback)

---

#### 3.2 LLM 호출 시점

| 전략 | 설명 | 비용 | 속도 | 적응성 |
|------|------|------|------|--------|
| **매 페이지마다** | 모든 페이지에서 LLM 호출 | 매우 높음 | 느림 | 매우 높음 |
| **초기만** | 처음 한 번만 규칙 생성 | 낮음 | 빠름 | 낮음 |
| **실패 시만** | 규칙 실패 시 재호출 | 중간 | 빠름 | 높음 |
| **주기적 검증** | 주 1회 규칙 유효성 확인 | 낮음 | 빠름 | 중간 |

**제안:** 초기 + 실패 시 + 주기적 검증 (Hybrid)

```python
# 의사 코드
def scrape_page(url, jurisdiction):
    # 1. 규칙 로드 시도
    rules = load_rules(jurisdiction)

    if rules and rules.is_valid():
        # 2. 규칙 기반 실행
        try:
            return scrape_with_rules(url, rules)
        except ScrapingError:
            # 3. 실패 시 LLM 재호출
            new_rules = llm_analyze_page(url)
            save_rules(jurisdiction, new_rules)
            return scrape_with_rules(url, new_rules)
    else:
        # 4. 규칙 없으면 LLM 호출
        rules = llm_analyze_page(url)
        save_rules(jurisdiction, rules)
        return scrape_with_rules(url, rules)
```

```sql
CREATE TABLE documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    -- 식별 정보
    jurisdiction TEXT NOT NULL,           -- 'cherokee_county'
    doc_type TEXT NOT NULL,               -- 'minutes', 'agenda', 'packet', 'video'
    meeting_date DATE NOT NULL,

    -- 문서 정보

    description TEXT,
    source_url TEXT NOT NULL,

    -- 파일 정보
    file_path TEXT,                       -- Local path or S3 key
    file_size INTEGER,                    -- Bytes
    file_format TEXT,                     -- 'pdf', 'html', 'video_link'

    -- 메타데이터
    scraped_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP,
    metadata JSON,                        -- 추가 정보 (유연성)

    -- 상태 관리
    status TEXT DEFAULT 'pending',        -- 'pending', 'downloaded', 'failed'
    error_message TEXT,
    retry_count INTEGER DEFAULT 0,

    -- 인덱스
    UNIQUE(jurisdiction, source_url),     -- 중복 방지
    INDEX idx_jurisdiction (jurisdiction),
    INDEX idx_meeting_date (meeting_date),
    INDEX idx_doc_type (doc_type)
);

CREATE TABLE scraping_runs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    jurisdiction TEXT NOT NULL,
    run_type TEXT NOT NULL,               -- 'backfill', 'continuous'
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    finished_at TIMESTAMP,
    status TEXT,                          -- 'running', 'completed', 'failed'
    documents_found INTEGER DEFAULT 0,
    documents_downloaded INTEGER DEFAULT 0,
    errors JSON
);
```

## 5. 에러 핸들링 및 재시도 전략

### 5.1 에러 타입별 전략

| 에러 타입 | 전략 | 재시도 횟수 | 대기 시간 |
|-----------|------|-------------|-----------|
| **Network Error** | Exponential backoff | 3회 | 1s, 2s, 4s |
| **Timeout** | Retry with increased timeout | 2회 | 5s, 10s |
| **404 Not Found** | 로깅만 (재시도 X) | 0회 | - |
| **Rate Limiting (429)** | 대기 후 재시도 | 5회 | Retry-After header |
| **Parsing Failure** | LLM 재분석 | 1회 | - |
| **Site Structure Change** | LLM 재학습 | 1회 | - |

---

### 5.2 구현 예시

```python
import asyncio
from tenacity import (
    retry,
    stop_after_attempt,
    wait_exponential,
    retry_if_exception_type
)

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=1, max=10),
    retry=retry_if_exception_type(NetworkError)
)
async def download_document(url: str) -> bytes:
    async with httpx.AsyncClient() as client:
        response = await client.get(url, timeout=30)
        response.raise_for_status()
        return response.content
```

## 6. 스케줄링 및 Continuous Update 구현

### 6.1 스케줄링 방식 비교

| 방법 | 설명 | 적합성 | 구현 복잡도 |
|------|------|--------|-------------|
| **Polling** | 정기적으로 사이트 방문 | ⭐⭐⭐⭐⭐ | 낮음 |
| **Webhook** | 사이트가 변경 알림 | ⭐ (불가능) | - |
| **RSS/Atom Feed** | 피드 구독 | ⭐⭐ (일부만 지원) | 낮음 |
| **HTML Diff** | 페이지 변경 감지 | ⭐⭐⭐ | 중간 |

**제안:** Polling (매일 1회)

---

### 6.2 구현 방식

#### Option A: Cron Job
```bash
# crontab -e
0 2 * * * cd /path/to/scraper && python main.py --mode continuous
```

**장점:** 간단, OS 레벨 스케줄링
**단점:** 관리 어려움, 로깅 분산

---

#### Option B: APScheduler (추천)
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

@scheduler.scheduled_job('cron', hour=2)
async def daily_update():
    for jurisdiction in jurisdictions:
        scraper = get_scraper(jurisdiction)
        await scraper.scrape_since(last_update_date)

scheduler.start()
```

**장점:** Python 통합, 동적 스케줄 변경
**단점:** 프로세스 관리 필요

---

#### Option C: Airflow
**장점:** 강력한 모니터링, 재시도, 의존성 관리
**단점:** 높은 복잡도, 인프라 필요

**제안:** APScheduler (MVP) → Airflow (프로덕션)

---

### 6.3 변경 감지 최적화

```python
async def scrape_since(self, last_date: date):
    # 1. 마지막 수집 날짜 이후만 확인
    new_documents = await self.find_documents_since(last_date)

    # 2. 이미 수집한 문서는 스킵 (checksum 비교)
    new_documents = filter_already_scraped(new_documents)

    # 3. 변경 없으면 조기 종료
    if not new_documents:
        logger.info(f"No new documents for {self.jurisdiction}")
        return

    # 4. 새 문서만 다운로드
    await self.download_documents(new_documents)
```
