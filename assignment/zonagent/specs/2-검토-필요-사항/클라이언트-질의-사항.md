# 클라이언트 질의 사항 (Client Questions)

> 프로덕션 배포 전에 클라이언트에게 확인이 필요한 비즈니스 요구사항 및 운영 정책

## 📋 개요

이 문서는 시스템 설계 및 운영에 영향을 미치는 질문들을 우선순위별로 정리합니다.
각 질문은 왜 중요한지, 어떤 설계 결정에 영향을 미치는지 명시합니다.

---

## ⚡ CRITICAL (MVP 설계에 필수적인 질문)

> 이 질문들은 MVP 개발 시작 전에 반드시 확인이 필요합니다.

### Q1. 수집한 문서를 어떻게 활용하는가?

**질문:**
수집된 문서들을 어떤 목적으로 사용할 예정인가요?

**선택지:**
- [ ] **검색 엔진** - 전문 검색 시스템 구축
- [ ] **AI/LLM 분석** - 문서 내용 추출 및 자연어 처리
- [ ] **단순 아카이빙** - 원본 파일만 보관
- [ ] **데이터 추출** - 구조화된 데이터로 변환
- [ ] 기타: _____________________

**왜 중요한가:**
활용 방법에 따라 메타데이터 추출 범위와 저장 형식이 달라집니다.

**영향 받는 설계:**
- **검색 엔진**: 전문 인덱싱, OCR 필요, 텍스트 추출
- **AI 분석**: PDF → 텍스트 변환, 토큰 최적화
- **아카이빙**: 원본만 저장, 최소 메타데이터
- **데이터 추출**: 구조화된 파싱, 스키마 설계

**현재 가정:**
검색 및 AI 분석 (메타데이터 추출 포함)

---

### Q2. 문서 누락의 허용 범위는?

**질문:**
스크래핑 과정에서 일부 문서가 누락되는 것이 얼마나 치명적인가요?

**선택지:**
- [ ] **100% 완벽** - 단 하나도 빠져서는 안 됨 → 수동 검증 필요
- [ ] **95% 이상** - 대부분 수집되면 OK → 자동화 + 샘플 검증
- [ ] **80% 이상** - 주요 문서만 수집 → 완전 자동화

**왜 중요한가:**
허용 범위에 따라 검증 로직 및 알림 수준이 결정됩니다.

**영향 받는 설계:**
- **100%**: 수동 검증, 크로스체크, 알림 시스템
- **95%**: 자동 검증, 샘플링, 주기적 리뷰
- **80%**: Best-effort, 에러 로깅만

**현재 가정:**
95% 이상 (자동화 + 품질 체크)

### Q3. 실시간성 요구사항은?

**질문:**
새 문서가 게시된 후 얼마나 빨리 수집되어야 하나요?

**선택지:**
- [ ] **1시간 이내** - 실시간 모니터링 (비용↑)
- [ ] **당일 내** - 하루 2-3회 polling
- [ ] **다음 날까지** - 매일 1회 (새벽)
- [ ] **일주일 내** - 주 1-2회

**왜 중요한가:**
실시간성에 따라 polling 주기 및 인프라 비용이 달라집니다.

**영향 받는 설계:**
- **1시간**: 고빈도 polling, 서버 상시 운영, 높은 비용
- **당일**: 하루 2-3회 스케줄링
- **다음 날**: 매일 새벽 1회 (권장)
- **일주일**: 주간 배치

**현재 가정:**
다음 날까지 (매일 새벽 2시 실행)

### Q4. 초기 Backfill 기간은?

**질문:**
정확히 몇 년치 과거 데이터가 필요한가요?

**선택지:**
- [ ] **6개월**
- [ ] **1년**
- [ ] **2년**
- [ ] **특정 날짜부터**: _____________________

**왜 중요한가:**
Backfill 기간에 따라 초기 작업량 및 스토리지 비용이 결정됩니다.

**영향 받는 설계:**
- 예상 문서 수 산정
- 스토리지 비용 계산
- 초기 실행 시간 추정

**예상 규모 (1년 기준):**
- 4개 지자체 x 12개월 x 월 2회 회의 = ~96개 회의
- 회의당 3-4개 문서 = ~300-400개 파일
- 평균 파일 크기 5MB = ~1.5-2GB

**현재 가정:**
1년 (조정 가능하게 설계)

- **10개 이내**: Plugin architecture
- **50개+**: Full Agentic, 자동화 필수
- **수백 개**: 완전 자율 시스템

**현재 가정:**

### Q7. API 비용 예산 가이드는?

**질문:**
월간 LLM API 비용 예산은 얼마인가요?

**선택지:**
- [ ] **$100 미만** - 최소화 (규칙 기반 위주)
- [ ] **$100-500** - 중간 (Hybrid)
- [ ] **$500-1000** - 높음 (적극적 LLM 활용)
- [ ] **제한 없음** - 효과적이면 OK

- Agentic 수준

**예상 비용 (Hybrid 방식):**
- 초기 학습: $20-40 (4개 지자체)
- 월간 운영: $10-30 (규칙 검증 + 가끔 재학습)
- **총 첫 달**: ~$50
- **이후 월**: ~$10-20

**현재 가정:**
$100-500 (Moderate Agentic 방식)

### Q11. 배포 환경은? (CRITICAL)

**질문:**
시스템을 어디서 실행할 예정인가요?

**선택지:**
- [ ] **AWS** (어떤 서비스? EC2, Lambda, ECS?)
- [ ] **GCP**
- [ ] **Azure**
- [ ] **온프레미스 서버**
- [ ] **로컬 머신** (개발용)

**왜 중요한가:**

- 스케줄링 도구 (Lambda vs Cloud Functions vs Cron)
- DB 선택 (RDS vs Cloud SQL vs PostgreSQL)
- 비용 구조

**현재 가정:**
로컬 개발 → AWS 배포 (S3 + EC2 또는 Lambda)
