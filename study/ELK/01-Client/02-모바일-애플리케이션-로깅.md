## 모바일 로깅의 도전 과제

모바일 애플리케이션에서 ELK로 로그를 전송하는 것은 웹보다 더 많은 제약이 있습니다.

### 주요 도전 과제

| 도전 과제 | 설명 | 영향 |
|---------|------|------|
| **제한된 대역폭** | 셀룰러 네트워크의 느린 속도 | 데이터 사용량 고려 필요 |
| **불안정한 연결** | 지하철, 엘리베이터 등에서 연결 끊김 | 오프라인 대응 필수 |
| **배터리 소모** | 빈번한 네트워크 요청 | 전송 빈도 최소화 |
| **사용자 데이터 요금** | 과도한 로그 전송 시 비용 발생 | 사용자 경험 저하 |
| **앱 라이프사이클** | 백그라운드 제약 | 전송 타이밍 중요 |

**출처**: [Why is it not popular to have a mobile application send crash log to ELK - Stack Overflow](https://stackoverflow.com/questions/62880629/why-is-it-not-popular-to-have-a-mobile-application-send-crash-log-to-elk)

### 왜 모바일에서 ELK가 덜 사용되는가?

1. **네트워크 제약**: 모바일은 제한된 대역폭과 불안정한 네트워크 환경
2. **전용 솔루션 존재**: Firebase Crashlytics, Sentry 등이 모바일에 최적화
3. **복잡성**: ELK Stack 직접 구성보다 managed 서비스가 더 효율적
4. **배터리 및 데이터 사용**: 사용자 경험에 직접적 영향

## 권장 아키텍처

### 기본 아키텍처

```
[모바일 앱]
     ↓
[로컬 DB/파일]  ← 오프라인 저장
     ↓
[조건부 전송]  ← Wi-Fi 연결시에만
     ↓
[백엔드 API]
     ↓
[ELK Stack]
```

### 처리 흐름

1. **로그 생성**: 앱 내에서 이벤트 발생
2. **로컬 저장**: SQLite, Realm, 파일 등에 저장
3. **조건 확인**:
   - Wi-Fi 연결 여부
   - 배터리 충전 중 여부 (선택적)
   - 마지막 전송 시간
4. **배치 전송**: 조건 만족시 백엔드로 전송
5. **전송 완료**: 로컬 데이터 삭제

## 전송 방법

### 1. HTTP/HTTPS (가장 일반적)

**장점**:
- 간단한 구현
- 대부분의 플랫폼 지원
- 표준 프로토콜

**단점**:
- 오버헤드 존재
- 실시간성 낮음

### 2. MQTT

**장점**:
- 경량 프로토콜
- 불안정한 네트워크에 강함
- Pub/Sub 패턴

**단점**:
- 추가 인프라 필요 (MQTT 브로커)
- 복잡도 증가

### 3. WebSocket

**장점**:
- 실시간 양방향 통신
- 효율적인 데이터 전송

**단점**:
- 배터리 소모 많음
- 모바일 환경에 부적합

### 4. Firebase + Cloud Functions

**장점**:
- 관리형 서비스
- 자동 재시도
- 오프라인 지원

**단점**:
- 간접적 방법
- Firebase 의존성

## 구현 예시

### iOS (Swift) 구현

```swift
// LogManager.swift
import Foundation

class LogManager {
    static let shared = LogManager()

    private let logQueue = DispatchQueue(label: "com.app.logqueue")
    private var logBuffer: [LogEntry] = []
    private let maxBufferSize = 50
    private let apiEndpoint = "https://api.example.com/logs"

    private init() {
        // 앱 시작시 저장된 로그 로드
        loadPersistedLogs()

        // 백그라운드 진입시 로그 저장
        NotificationCenter.default.addObserver(
            self,
            selector: #selector(appDidEnterBackground),
            name: UIApplication.didEnterBackgroundNotification,
            object: nil
        )

        // Wi-Fi 연결시 자동 전송
        startNetworkMonitoring()
    }

    struct LogEntry: Codable {
        let timestamp: Date
        let level: String
        let message: String
        let context: [String: String]
    }

    // MARK: - Public Methods

    func log(level: String, message: String, context: [String: String] = [:]) {
        let entry = LogEntry(
            timestamp: Date(),
            level: level,
            message: message,
            context: mergeContext(context)
        )

        logQueue.async {
            self.logBuffer.append(entry)

            if self.logBuffer.count >= self.maxBufferSize {
                self.flush()
            }
        }
    }

    func info(_ message: String, context: [String: String] = [:]) {
        log(level: "INFO", message: message, context: context)
    }

    func error(_ message: String, context: [String: String] = [:]) {
        log(level: "ERROR", message: message, context: context)
    }

    // MARK: - Private Methods

    private func flush() {
        guard !logBuffer.isEmpty else { return }
        guard isWiFiConnected() else {
            // Wi-Fi가 아니면 로컬에만 저장
            persistLogs()
            return
        }

        let logsToSend = logBuffer
        logBuffer.removeAll()

        sendLogs(logsToSend) { success in
            if !success {
                // 실패시 다시 버퍼에 추가
                self.logQueue.async {
                    self.logBuffer.append(contentsOf: logsToSend)
                    self.persistLogs()
                }
            }
        }
    }

    private func sendLogs(_ logs: [LogEntry], completion: @escaping (Bool) -> Void) {
        guard let url = URL(string: apiEndpoint) else {
            completion(false)
            return
        }

        var request = URLRequest(url: url)
        request.httpMethod = "POST"
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")

        // 인증 토큰 추가
        if let token = UserDefaults.standard.string(forKey: "authToken") {
            request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
        }

        do {
            let encoder = JSONEncoder()
            encoder.dateEncodingStrategy = .iso8601
            request.httpBody = try encoder.encode(["logs": logs])

            URLSession.shared.dataTask(with: request) { data, response, error in
                if let error = error {
                    print("Failed to send logs: \(error)")
                    completion(false)
                    return
                }

                if let httpResponse = response as? HTTPURLResponse,
                   httpResponse.statusCode == 200 {
                    completion(true)
                } else {
                    completion(false)
                }
            }.resume()
        } catch {
            print("Failed to encode logs: \(error)")
            completion(false)
        }
    }

    private func persistLogs() {
        let encoder = JSONEncoder()
        if let encoded = try? encoder.encode(logBuffer) {
            UserDefaults.standard.set(encoded, forKey: "persistedLogs")
        }
    }

    private func loadPersistedLogs() {
        if let data = UserDefaults.standard.data(forKey: "persistedLogs") {
            let decoder = JSONDecoder()
            if let logs = try? decoder.decode([LogEntry].self, from: data) {
                logBuffer = logs
            }
        }
    }

    private func isWiFiConnected() -> Bool {
        // Reachability 라이브러리 사용 권장
        // 여기서는 간단히 구현
        return true // 실제 구현 필요
    }

    private func startNetworkMonitoring() {
        // Wi-Fi 연결시 자동으로 flush
        // Reachability 사용하여 네트워크 변경 감지
    }

    private func mergeContext(_ custom: [String: String]) -> [String: String] {
        var context = custom
        context["appVersion"] = Bundle.main.infoDictionary?["CFBundleShortVersionString"] as? String
        context["osVersion"] = UIDevice.current.systemVersion
        context["deviceModel"] = UIDevice.current.model
        return context
    }

    @objc private func appDidEnterBackground() {
        persistLogs()
    }
}

// MARK: - Usage
// LogManager.shared.info("User logged in", context: ["userId": "123"])
// LogManager.shared.error("Network error", context: ["endpoint": "/api/users"])
```

### Android (Kotlin) 구현

```kotlin
// LogManager.kt
package com.example.app

import android.content.Context
import android.net.ConnectivityManager
import android.net.NetworkCapabilities
import androidx.work.*
import kotlinx.coroutines.*
import kotlinx.serialization.Serializable
import kotlinx.serialization.encodeToString
import kotlinx.serialization.json.Json
import okhttp3.*
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.RequestBody.Companion.toRequestBody
import java.util.concurrent.TimeUnit

@Serializable
data class LogEntry(
    val timestamp: Long,
    val level: String,
    val message: String,
    val context: Map<String, String>
)

class LogManager private constructor(private val context: Context) {

    private val logBuffer = mutableListOf<LogEntry>()
    private val maxBufferSize = 50
    private val apiEndpoint = "https://api.example.com/logs"
    private val prefs = context.getSharedPreferences("logs", Context.MODE_PRIVATE)
    private val json = Json { ignoreUnknownKeys = true }

    companion object {
        @Volatile
        private var INSTANCE: LogManager? = null

        fun getInstance(context: Context): LogManager {
            return INSTANCE ?: synchronized(this) {
                INSTANCE ?: LogManager(context.applicationContext).also {
                    INSTANCE = it
                }
            }
        }
    }

    init {
        loadPersistedLogs()
        schedulePeriodicFlush()
    }

    fun log(level: String, message: String, context: Map<String, String> = emptyMap()) {
        val entry = LogEntry(
            timestamp = System.currentTimeMillis(),
            level = level,
            message = message,
            context = mergeContext(context)
        )

        synchronized(logBuffer) {
            logBuffer.add(entry)

            if (logBuffer.size >= maxBufferSize) {
                flush()
            }
        }
    }

    fun info(message: String, context: Map<String, String> = emptyMap()) {
        log("INFO", message, context)
    }

    fun error(message: String, context: Map<String, String> = emptyMap()) {
        log("ERROR", message, context)
    }

    private fun flush() {
        if (logBuffer.isEmpty()) return

        // Wi-Fi 연결 확인
        if (!isWiFiConnected()) {
            persistLogs()
            return
        }

        val logsToSend = synchronized(logBuffer) {
            logBuffer.toList().also { logBuffer.clear() }
        }

        // WorkManager를 사용한 백그라운드 전송
        val workRequest = OneTimeWorkRequestBuilder<LogUploadWorker>()
            .setConstraints(
                Constraints.Builder()
                    .setRequiredNetworkType(NetworkType.CONNECTED)
                    .build()
            )
            .setInputData(
                workDataOf("logs" to json.encodeToString(logsToSend))
            )
            .build()

        WorkManager.getInstance(context).enqueue(workRequest)
    }

    private fun persistLogs() {
        val encoded = json.encodeToString(logBuffer)
        prefs.edit().putString("persisted_logs", encoded).apply()
    }

    private fun loadPersistedLogs() {
        val encoded = prefs.getString("persisted_logs", null) ?: return
        try {
            val logs = json.decodeFromString<List<LogEntry>>(encoded)
            logBuffer.addAll(logs)
        } catch (e: Exception) {
            e.printStackTrace()
        }
    }

    private fun isWiFiConnected(): Boolean {
        val cm = context.getSystemService(Context.CONNECTIVITY_SERVICE) as ConnectivityManager
        val network = cm.activeNetwork ?: return false
        val capabilities = cm.getNetworkCapabilities(network) ?: return false
        return capabilities.hasTransport(NetworkCapabilities.TRANSPORT_WIFI)
    }

    private fun schedulePeriodicFlush() {
        val workRequest = PeriodicWorkRequestBuilder<LogFlushWorker>(
            15, TimeUnit.MINUTES
        ).build()

        WorkManager.getInstance(context)
            .enqueueUniquePeriodicWork(
                "log_flush",
                ExistingPeriodicWorkPolicy.KEEP,
                workRequest
            )
    }

    private fun mergeContext(custom: Map<String, String>): Map<String, String> {
        return custom + mapOf(
            "appVersion" to context.packageManager
                .getPackageInfo(context.packageName, 0).versionName,
            "osVersion" to android.os.Build.VERSION.RELEASE,
            "deviceModel" to android.os.Build.MODEL
        )
    }
}

// WorkManager Worker for uploading logs
class LogUploadWorker(
    context: Context,
    params: WorkerParameters
) : CoroutineWorker(context, params) {

    override suspend fun doWork(): Result {
        val logsJson = inputData.getString("logs") ?: return Result.failure()

        return try {
            val client = OkHttpClient()
            val request = Request.Builder()
                .url("https://api.example.com/logs")
                .post(
                    logsJson.toRequestBody("application/json".toMediaType())
                )
                .build()

            val response = client.newCall(request).execute()

            if (response.isSuccessful) {
                Result.success()
            } else {
                Result.retry()
            }
        } catch (e: Exception) {
            Result.retry()
        }
    }
}

class LogFlushWorker(
    context: Context,
    params: WorkerParameters
) : Worker(context, params) {

    override fun doWork(): Result {
        LogManager.getInstance(applicationContext).flush()
        return Result.success()
    }
}

// Usage:
// val logger = LogManager.getInstance(context)
// logger.info("User action", mapOf("action" to "button_click"))
// logger.error("API error", mapOf("endpoint" to "/api/users"))
```

## 대안 솔루션

ELK Stack 대신 모바일에 특화된 솔루션을 사용하는 것도 일반적입니다.

### 1. Firebase Crashlytics

**장점**:
- 무료
- 크래시 리포트 자동 수집
- 실시간 알림
- Firebase 생태계 통합

**단점**:
- Firebase 종속
- 커스터마이징 제한

```swift
// iOS
import FirebaseCrashlytics

Crashlytics.crashlytics().log("User action performed")
Crashlytics.crashlytics().setCustomValue(userId, forKey: "user_id")
```

```kotlin
// Android
FirebaseCrashlytics.getInstance().log("User action performed")
FirebaseCrashlytics.getInstance().setCustomKey("user_id", userId)
```

### 2. Sentry

**장점**:
- 모바일 SDK 우수
- 상세한 에러 트래킹
- Release tracking
- ELK로 데이터 포워딩 가능

**비용**: 유료 (무료 티어 있음)

```swift
// iOS
import Sentry

SentrySDK.start { options in
    options.dsn = "your-dsn"
    options.environment = "production"
}
```

### 3. Bugfender

**장점**:
- 모바일 로깅 전문
- 오프라인 지원 우수
- 디바이스별 로그 확인

### 4. Loggly

**장점**:
- 클라우드 로그 관리
- 모바일 SDK 제공
- ELK 대안

## Best Practices

### ✅ 권장사항

1. **Wi-Fi에서만 전송**
   - 사용자 데이터 요금 절약
   - 빠른 전송 속도

2. **배치 전송**
   - 개별 전송 대신 모아서 전송
   - 배터리 절약

3. **로컬 저장 우선**
   - SQLite, Realm 등 사용
   - 오프라인 대응

4. **중요 로그만 전송**
   - ERROR, CRITICAL 우선
   - DEBUG는 개발 환경에서만

5. **압축 사용**
   - 전송 데이터 압축
   - 대역폭 절약

### ❌ 주의사항

1. **실시간 전송 금지**
   - 배터리 소모
   - 네트워크 부하

2. **대용량 로그 전송 금지**
   - 스택 트레이스 크기 제한
   - 이미지, 동영상 포함 금지

3. **무제한 저장 금지**
   - 디스크 공간 제한
   - 오래된 로그 자동 삭제
