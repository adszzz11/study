## 핵심 원칙

### ❌ 절대 금지
```
클라이언트 → Elasticsearch (직접 연결)
```

**위험 요소:**
- 자격 증명 노출
- 악의적 데이터 주입
- DDoS 공격 대상
- 인증/권한 관리 불가

### ✅ 권장 방식
```
클라이언트 → 백엔드 API → ELK Stack
```

**장점:**
- 보안 강화
- 데이터 검증
- 인증/권한 관리
- 로그 변환 및 보강

## 클라이언트 vs 서버 로깅 비교

| 구분 | 서버 측 | 클라이언트 측 |
|-----|--------|-------------|
| 네트워크 | 안정적 | 불안정적 |
| 대역폭 | 풍부 | 제한적 |
| 보안 | 내부 네트워크 | 공개 네트워크 |
| 직접 연결 | 가능 | **불가능/비권장** |
| 오프라인 대응 | 불필요 | 필수 |
