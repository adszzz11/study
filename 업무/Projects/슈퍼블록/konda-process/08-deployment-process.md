# 배포 프로세스 가이드

## 1. 개요

본 문서는 Konda 프로젝트의 배포 프로세스를 정의합니다. 모든 배포는 내부에서 전담하며, 외주사는 코드를 전달하고 배포를 요청합니다.

---

## 2. 배포 프로세스 개요

### 2.1 배포 주체

| 항목 | 담당 |
|------|------|
| 배포 실행 | 내부 팀 (전담) |
| 배포 요청 | 외주사 또는 내부 개발팀 |
| 배포 승인 | PM + QA |

### 2.2 전체 배포 흐름

```
┌──────────────────────────────────────────────────────────────┐
│                      배포 프로세스                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 개발 완료 ──→ 2. QA 통과 ──→ 3. 배포 요청                │
│                                      │                       │
│                                      ▼                       │
│                              4. 코드 확인                    │
│                                 (내부)                       │
│                                      │                       │
│                    ┌─────────────────┴─────────────────┐     │
│                    ▼                                   ▼     │
│              이상 있음                            이상 없음  │
│                    │                                   │     │
│                    ▼                                   ▼     │
│            수정 요청 & 대기               5. 스테이징 배포   │
│                                               │              │
│                                               ▼              │
│                                       6. 스테이징 테스트     │
│                                               │              │
│                                               ▼              │
│                                       7. 프로덕션 배포       │
│                                               │              │
│                                               ▼              │
│                                       8. 배포 완료 알림      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

---

## 3. 배포 요청 프로세스

### 3.1 배포 요청 채널

**Slack #proj-konda-dev** 채널에서 배포를 요청합니다.

### 3.2 배포 요청 형식

```markdown
**[배포 요청]**

## 배포 대상
- [ ] konda-api (비금융 BE)
- [ ] konda-landing (랜딩 페이지)

## 환경
- [ ] 스테이징
- [ ] 프로덕션

## 배포 정보
- Repository: [GitHub 링크]
- Branch: main
- Commit: [SHA]
- 버전: [버전 태그, 있다면]

## 변경 내용
1. [변경 사항 1]
2. [변경 사항 2]
3. [변경 사항 3]

## 관련 이슈/PR
- [#이슈번호 또는 PR 링크]

## QA 완료 여부
- [ ] 외주사 1차 QA 완료
- [ ] 내부 최종 검수 완료 (프로덕션 시)

## 희망 배포일
- 스테이징: YYYY-MM-DD HH:MM
- 프로덕션: YYYY-MM-DD HH:MM

## 특이 사항
- [환경변수 변경, DB 마이그레이션 등]

@내부PM @내부개발팀
```

### 3.3 배포 요청 조건

#### 스테이징 배포
- 외주사 1차 QA 완료
- main 브랜치 최신화

#### 프로덕션 배포
- 스테이징 배포 후 내부 검수 완료
- PM 승인
- QA 승인

---

## 4. 외주사 → 내부 코드 전달

### 4.1 코드 전달 방식

외주사는 GitHub Repository의 main 브랜치를 최신화하여 전달합니다.

```
외주사 작업
    │
    ▼
feature 브랜치 → main 병합
    │
    ▼
main 브랜치 푸시
    │
    ▼
배포 요청 (Slack)
```

### 4.2 코드 확인 항목

내부에서 코드 전달 시 확인하는 항목:

- [ ] 빌드 성공 여부
- [ ] 기본 보안 검토
- [ ] 환경 설정 확인
- [ ] 의존성 확인

---

## 5. 배포 절차

### 5.1 스테이징 배포

```
1. 배포 요청 접수 확인
2. 코드 pull
3. 빌드 & 테스트
4. 스테이징 환경 배포
5. 기본 동작 확인
6. 배포 완료 알림
```

### 5.2 프로덕션 배포

```
1. 스테이징 검수 완료 확인
2. PM/QA 승인 확인
3. 배포 시간 확정
4. 프로덕션 배포 (점검 시간 또는 저트래픽 시간)
5. 배포 후 모니터링
6. 이상 없으면 완료 알림
7. 이상 있으면 롤백
```

### 5.3 배포 완료 알림

```markdown
**[배포 완료]**

## 배포 대상
- 서비스: [konda-api / konda-landing]
- 환경: 스테이징 / 프로덕션

## 배포 정보
- 배포 시간: YYYY-MM-DD HH:MM
- 버전: [버전/커밋]
- 배포자: [이름]

## 배포 내용
- [변경 사항 요약]

## 확인 필요 사항
- [확인이 필요한 항목이 있다면]

@배포요청자 @관련팀
```

## 6. 배포 일정 및 주기

### 6.1 정기 배포

| 항목 | 일정 |
|------|------|
| 스테이징 | 필요 시 (요청 후 1영업일 내) |
| 프로덕션 | 주 1회 (요일 협의) 또는 필요 시 |

### 6.2 긴급 배포

- Critical 버그 수정 시
- 보안 취약점 패치 시
- PM 승인 후 즉시 배포

### 6.3 배포 금지 시간

- 금요일 오후 ~ 월요일 오전 (특별한 경우 제외)
- 공휴일 전날 오후
- 트래픽 최대 시간대 (협의 필요)

---

## 7. 롤백 절차

### 7.1 롤백 조건

- 배포 후 주요 기능 장애 발생
- 심각한 성능 저하
- 보안 이슈 발견

### 7.2 롤백 프로세스

```
1. 장애 발생 감지
2. 장애 심각도 판단
3. 롤백 결정 (PM 또는 긴급 시 개발 리드)
4. 이전 버전으로 롤백
5. 서비스 정상화 확인
6. 장애 원인 분석
7. 수정 후 재배포 계획
```

### 7.3 롤백 알림

```markdown
**[롤백 알림]**

## 롤백 정보
- 서비스: [서비스명]
- 환경: 프로덕션
- 롤백 시간: YYYY-MM-DD HH:MM
- 롤백 버전: [이전 버전]

## 롤백 사유
- [장애 내용]

## 현재 상태
- 서비스 정상화 완료

## 후속 조치
- [원인 분석 및 수정 계획]

@전체팀
```

## 8. 배포 체크리스트

### 8.1 배포 전 체크리스트

- [ ] QA 완료 확인
- [ ] 코드 리뷰 완료
- [ ] 환경 변수 설정 확인
- [ ] DB 마이그레이션 확인 (있다면)
- [ ] 의존 서비스 상태 확인
- [ ] 롤백 계획 수립

### 8.2 배포 후 체크리스트

- [ ] 서비스 정상 동작 확인
- [ ] 주요 API 응답 확인
- [ ] 에러 로그 모니터링
- [ ] 성능 지표 확인
- [ ] 배포 완료 알림 발송
