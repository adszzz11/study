# 디자인 프로세스 가이드

## 1. 개요

본 문서는 Konda 프로젝트의 디자인 프로세스를 정의합니다. 디자인 작업은 외주사에서 수행하며, Figma를 사용하여 핸드오프합니다.

---

## 2. 디자인 프로세스 흐름

│      (선행)           (외주사)           (스프린트 단위)       │
│                                              │               │
│                         ┌────────────────────┘               │
│                         ▼                                    │
│               ┌─────────────────┐                            │

## 3. Figma 프로젝트 구조

### 3.1 프로젝트 구성

```
Konda (Team)
├── Design System
│   ├── Colors
│   ├── Typography
│   ├── Components
│   ├── Icons
│   └── Spacing & Grid
│
├── App Design
│   ├── [Sprint 1] 로그인/회원가입
│   ├── [Sprint 2] 메인 화면
│   ├── [Sprint 3] 기능 A
│   └── ...
│
├── Landing Page Design
│   └── 랜딩 페이지
│
└── Archive
    └── 이전 버전/폐기 디자인
```

### 3.2 페이지 네이밍 규칙

```
[Sprint N] 기능명

예시:
- [Sprint 1] 로그인/회원가입
- [Sprint 2] 메인 피드
- [Sprint 3] 결제 플로우
```

### 3.3 프레임 네이밍 규칙

```
[상태] 화면명 - 세부상태

예시:
- [Default] 로그인 화면
- [Filled] 로그인 화면 - 입력완료
- [Error] 로그인 화면 - 비밀번호 오류
- [Loading] 로그인 화면 - 로딩중
```

## 4. 디자인 핸드오프 방법

### 4.1 Figma Dev Mode 활용

개발자는 Figma Dev Mode를 통해 다음 정보를 확인합니다:

| 정보 | 확인 방법 |
|------|----------|
| 크기/간격 | 요소 선택 시 우측 패널 |
| 색상 | 스타일 패널 (토큰 확인) |
| 타이포그래피 | 텍스트 스타일 패널 |
| 에셋 내보내기 | Export 기능 |
| CSS/코드 | Code 패널 |

### 4.2 핸드오프 체크리스트

디자이너가 핸드오프 전 확인할 사항:

- [ ] 모든 요소에 Auto Layout 적용
- [ ] 컴포넌트화 완료
- [ ] 색상/텍스트 스타일 적용
- [ ] 아이콘 에셋 Export 설정
- [ ] 각 상태별 화면 정의 (Default, Hover, Active, Disabled 등)
- [ ] 인터랙션/애니메이션 명시 (필요 시)
- [ ] 반응형 고려 사항 명시

### 4.3 핸드오프 알림

```markdown
**[디자인 핸드오프] Sprint N - 기능명**

## Figma 링크
[링크]

## 핸드오프 범위
- 화면 1: [화면명]
- 화면 2: [화면명]

## 주요 안내 사항
- [개발 시 참고할 특이사항]

## 에셋 정보
- 아이콘: Figma에서 Export
- 이미지: [경로/방법]

@App개발팀
```

## 5. 디자인 검수 프로세스

### 5.1 스프린트 단위 검수

각 스프린트 종료 시 디자인 검수를 진행합니다.

```
스프린트 시작
      │
      ▼
해당 스프린트 디자인 작업 (외주사)
      │
      ▼
스프린트 중간 체크 (선택)
      │
      ▼
스프린트 종료 전 검수 요청
      │
      ▼
내부 검수 (PM + App팀)
  - 기획 반영 여부
  - UI 일관성
  - 개발 가능성
      │
      ▼
피드백 전달 및 반영
      │
      ▼
디자인 확정
```

### 5.2 검수 체크리스트

#### 기획 반영 검수
- [ ] 기획서 요구사항 모두 반영
- [ ] 화면 흐름 일치
- [ ] 예외 상황 화면 포함

#### UI 일관성 검수
- [ ] 디자인 시스템 준수
- [ ] 색상/폰트 일관성
- [ ] 컴포넌트 일관성
- [ ] 간격/정렬 일관성

#### 개발 가능성 검수
- [ ] 구현 가능한 디자인
- [ ] 성능 영향 고려
- [ ] 플랫폼별 특성 고려 (iOS/Android)

### 5.3 검수 결과 형식

```markdown
**[디자인 검수 결과] Sprint N - 기능명**

## 검수 결과: 승인 / 수정 후 확정 / 재작업 필요

## 검수 내용

### 기획 반영
- [ ] 완료 / 수정 필요: [내용]

### UI 일관성
- [ ] 완료 / 수정 필요: [내용]

### 개발 가능성
- [ ] 완료 / 수정 필요: [내용]

## 피드백 상세

### 필수 수정 사항
1. [화면명] - [수정 내용]

### 권장 수정 사항
1. [화면명] - [권장 내용]

## 수정 기한
- YYYY-MM-DD까지 수정 완료 요청
```

## 6. 디자인 피드백 방법

### 6.1 피드백 채널

- **Slack #proj-konda-all**: 간단한 피드백, 질의
- **Figma 코멘트**: 특정 요소에 대한 상세 피드백
- **정기 미팅**: 전체적인 방향성 논의

### 6.2 Figma 코멘트 활용

```
피드백 작성 시:
1. 해당 요소/프레임에 직접 코멘트
2. 코멘트 형식: [유형] 내용
   - [수정] 색상을 Primary로 변경 필요
   - [질문] 이 버튼은 어떤 동작을 하나요?
   - [제안] 여백을 16px로 조정하면 좋겠습니다
3. 담당자 태그 (@디자이너)
```

### 6.3 피드백 반영 확인

```
피드백 → 디자이너 반영 → Figma 코멘트에 "Resolve" 체크
                         → Slack에 완료 알림
```

## 7. 디자인 시스템

### 7.1 구성 요소

| 요소 | 설명 | Figma 위치 |
|------|------|-----------|
| Colors | 브랜드/시스템 색상 | Design System > Colors |
| Typography | 텍스트 스타일 | Design System > Typography |
| Icons | 아이콘 세트 | Design System > Icons |
| Components | 재사용 컴포넌트 | Design System > Components |
| Spacing | 간격 체계 | Design System > Spacing |

### 7.2 디자인 토큰

개발팀과 공유할 디자인 토큰 형식:

```json
{
  "colors": {
    "primary": {
      "50": "#E3F2FD",
      "100": "#BBDEFB",
      "500": "#2196F3",
      "700": "#1976D2"
    },
    "semantic": {
      "success": "#4CAF50",
      "error": "#F44336",
      "warning": "#FF9800"
    }
  },
  "typography": {
    "heading1": {
      "fontSize": "24px",
      "fontWeight": "700",
      "lineHeight": "32px"
    }
  },
  "spacing": {
    "xs": "4px",
    "sm": "8px",
    "md": "16px",
    "lg": "24px",
    "xl": "32px"
  }
}
```

## 8. 에셋 관리

### 8.1 아이콘 내보내기

- **형식**: SVG (벡터), PNG (래스터 필요 시)
- **크기**: 24x24 기본, 필요 시 16x16, 32x32
- **네이밍**: `ic_[카테고리]_[이름].svg`

```
예시:
- ic_nav_home.svg
- ic_action_share.svg
- ic_status_success.svg
```

### 8.2 이미지 내보내기

- **형식**: PNG, WebP
- **배수**: @1x, @2x, @3x
- **네이밍**: `img_[화면]_[설명].png`
