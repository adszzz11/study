# R&R (역할과 책임) 정의서

## 1. 개요

본 문서는 Konda 프로젝트의 내부 담당자와 외주사 담당자 간의 역할과 책임을 명확히 정의합니다.

---

## 2. 조직 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    Konda 프로젝트 조직                       │
├─────────────────────────┬───────────────────────────────────┤
│        내부 (다날)        │        외주사 (슈퍼블록)            │
├─────────────────────────┼───────────────────────────────────┤
│  • PM                   │  • 기획 담당자                      │
│  • App 개발 담당자        │  • 디자인 담당자                    │
│  • BE(금융) 개발 담당자    │  • BE(비금융) 개발 담당자           │
│  • QA 담당자             │  • 랜딩 페이지 개발 담당자           │
└─────────────────────────┴───────────────────────────────────┘
```

## 5. 의사결정 권한 매트릭스

| 의사결정 영역        | 결정 권한     | 검토/협의            | 통보        |
| -------------- | --------- | ---------------- | --------- |
| **기획 확정**      | PM        | 기획 담당자, 개발 담당자   | 디자인 담당자   |
| **디자인 확정**     | PM        | 디자인 담당자, App 담당자 | 개발 담당자    |
| **기술 스택 (내부)** | 내부 개발 담당자 | PM               | 외주사       |
| **기술 스택 (외주)** | 외주사       | PM               | 내부 개발 담당자 |
| **일정 변경**      | PM        | 양측 개발 담당자        | 전체        |
| **긴급 이슈 대응**   | PM        | 해당 담당자           | 전체        |
| **배포 승인**      | PM + QA   | 개발 담당자           | 전체        |
| **보안 정책**      | PM (내부)   | 전체               | -         |

### 의사결정 프로세스

```
1. 제안/요청 → 2. 검토/협의 → 3. 결정 → 4. 통보 → 5. 실행
```

## 7. 긴급 연락망

```
긴급 상황 발생 시 연락 순서:

1차: Slack 멘션 (@담당자)
  ↓ (30분 내 응답 없을 시)
2차: 전화 연락
  ↓ (1시간 내 해결 불가 시)
3차: PM 에스컬레이션
```

### 긴급 상황 정의
- **Critical**: 서비스 전체 장애, 보안 이슈
- **High**: 주요 기능 장애, 일정 지연 우려
- **Medium**: 일부 기능 이슈, 협의 필요 사항
