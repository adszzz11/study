# 산출물 인수인계 가이드

## 1. 개요

본 문서는 Konda 프로젝트의 외주사 산출물 인수인계 프로세스를 정의합니다. 지속적인 운영 이관 방식으로 프로젝트 진행 중 단계별로 산출물을 인수받습니다.

---

## 2. 인수인계 방식

### 2.1 지속적 운영 이관

```
┌──────────────────────────────────────────────────────────────┐
│               지속적 운영 이관 프로세스                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│    스프린트 1        스프린트 2        스프린트 N              │
│        │                │                │                   │
│        ▼                ▼                ▼                   │
│    ┌───────┐        ┌───────┐        ┌───────┐              │
│    │ 개발  │        │ 개발  │        │ 개발  │              │
│    └───┬───┘        └───┬───┘        └───┬───┘              │
│        │                │                │                   │
│        ▼                ▼                ▼                   │
│    ┌───────┐        ┌───────┐        ┌───────┐              │
│    │ 인수  │        │ 인수  │        │ 인수  │              │
│    └───┬───┘        └───┬───┘        └───┬───┘              │
│        │                │                │                   │
│        └────────────────┴────────────────┘                   │
│                         │                                    │

│                  │  최종 인수   │                             │
│                  │ (프로젝트   │                             │

│                  └─────────────┘                             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 인수인계 시점

| 시점 | 인수 내용 |
|------|----------|
| 스프린트 종료 시 | 해당 스프린트 산출물 |
| 기능 완료 시 | 기능별 문서 및 코드 |
| 프로젝트 종료 시 | 전체 산출물 최종 인수 |

## 3. 인수인계 산출물 목록

| 정책 문서 | Markdown | 필요 시 | 기획팀 |

### 3.2 디자인 산출물

| 산출물 | 형식 | 인수 시점 | 담당 |
|--------|------|----------|------|
| Figma 디자인 파일 | Figma | 스프린트별 | 디자인팀 |
| 디자인 시스템 | Figma | 초기 + 업데이트 시 | 디자인팀 |

| 디자인 가이드 | Markdown/PDF | 초기 | 디자인팀 |

### 3.3 개발 산출물

| 산출물 | 형식 | 인수 시점 | 담당 |

| 소스 코드 | GitHub Repo | 지속적 | 개발팀 |
| API 명세서 | OpenAPI/Swagger | API 완료 시 | 개발팀 |
| ERD/DB 스키마 | 문서/다이어그램 | 개발 완료 시 | 개발팀 |

### 3.4 QA 산출물

| 산출물 | 형식 | 인수 시점 | 담당 |
|--------|------|----------|------|
| 테스트 케이스 | Markdown/시트 | QA 완료 시 | QA/개발팀 |
| 테스트 결과 보고서 | Markdown | QA 완료 시 | QA/개발팀 |

## 4. 인수인계 프로세스

### 4.1 스프린트별 인수인계

```
스프린트 종료 1-2일 전
        │
        ▼
외주사: 인수인계 준비
  - 산출물 정리
  - 문서 업데이트
  - 인수인계 요청
        │
        ▼
내부: 산출물 확인
  - 산출물 목록 확인
  - 문서 검토
  - 코드 확인
        │
        ▼
인수 완료 또는 보완 요청
        │
        ▼
완료 시 인수 확인서 작성
```

### 4.2 인수인계 요청 형식

```markdown
**[인수인계 요청] Sprint N**

## 인수인계 범위
- 기간: YYYY-MM-DD ~ YYYY-MM-DD
- 기능: [개발된 기능 목록]

## 산출물 목록

### 기획
- [ ] [기획서 링크]

### 디자인
- [ ] [Figma 링크]
- [ ] [에셋 위치]

### 개발
- [ ] 소스 코드 (main 브랜치 최신화)
- [ ] [API 명세서 링크]

### 문서
- [ ] [기술 문서 링크]

## 특이 사항
- [인수 시 참고할 사항]

@내부PM
```

### 4.3 인수 확인

```markdown
**[인수 확인] Sprint N**

## 인수 일자
- YYYY-MM-DD

## 인수 항목
- [x] 기획서
- [x] 디자인 파일
- [x] 소스 코드
- [x] API 명세서

## 확인자
- 내부: [이름]
- 외주사: [이름]

## 비고
- [특이 사항]
```

## 5. 최종 인수인계 (프로젝트 종료)

### 5.1 최종 인수인계 체크리스트

#### 기획 산출물
- [ ] 전체 기획서 (최종 버전)
- [ ] 모든 기능 명세서
- [ ] 화면 정의서
- [ ] 정책 문서

#### 디자인 산출물
- [ ] Figma 프로젝트 전체 (편집 권한 이전 또는 복사본)
- [ ] 디자인 시스템
- [ ] 모든 에셋 파일
- [ ] 디자인 가이드 문서

#### 개발 산출물
- [ ] 소스 코드 (전체, 히스토리 포함)
- [ ] API 명세서 (Swagger 등)
- [ ] DB 스키마/ERD
- [ ] 환경 설정 문서
- [ ] 배포 가이드
- [ ] 개발 가이드

#### 기타 산출물
- [ ] 테스트 케이스/결과
- [ ] 회의록
- [ ] 이슈 히스토리

### 5.2 계정/권한 정리

| 항목 | 조치 |
|------|------|
| GitHub Organization | 외주사 멤버 제거 |
| Slack 채널 | 외주사 멤버 제거 |
| Figma | 편집 권한 제거 또는 프로젝트 복사 |
| 기타 도구 | 접근 권한 회수 |

### 5.3 최종 인수인계 미팅

```
1. 전체 산출물 확인
2. 미비 사항 확인 및 보완 요청
3. 기술 인수인계 (핵심 로직 설명)
4. 운영 가이드 전달
5. 유지보수 협의 (필요 시)
6. 최종 인수 확인서 서명
```

## 6. 기술 문서화 요구사항

### 6.1 필수 문서

| 문서 | 내용 | 형식 |
|------|------|------|
| README | 프로젝트 개요, 설치/실행 방법 | Markdown |
| API 명세 | 전체 API 엔드포인트 설명 | OpenAPI |
| DB 스키마 | 테이블 구조 및 관계 | ERD + 설명 |
| 환경 설정 | 환경변수, 설정 파일 설명 | Markdown |

### 6.2 README 필수 항목

```markdown
# 프로젝트명

## 개요
[프로젝트 설명]

## 기술 스택
- [사용 기술 목록]

## 설치 및 실행
### 사전 요구사항
- [필요한 환경]

### 설치
[설치 명령어]

### 실행
[실행 명령어]

## 환경 변수
| 변수명 | 설명 | 필수 |
|--------|------|------|
| | | |

## 프로젝트 구조
[디렉토리 구조 설명]

## API 문서
[API 문서 링크]

## 배포
[배포 방법]
```

### 6.3 코드 문서화

- 주요 함수/클래스에 주석 작성
- 복잡한 로직에 설명 추가
- API 엔드포인트에 설명 추가
