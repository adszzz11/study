# 프로젝트 종료 체크리스트

## 1. 개요

본 문서는 Konda 프로젝트 종료 시 수행해야 할 체크리스트를 정의합니다. 모든 산출물 인수, 권한 회수, 문서 정리 등을 체계적으로 진행합니다.

---

## 2. 프로젝트 종료 프로세스

### 2.1 종료 프로세스 흐름

```
┌──────────────────────────────────────────────────────────────┐
│                   프로젝트 종료 프로세스                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 종료 준비 ──→ 2. 산출물 인수 ──→ 3. 권한/계정 회수        │
│   (2주 전)         (1주 전)           (종료 시)              │
│                                          │                   │
│                                          ▼                   │
│  4. 문서 정리 ──→ 5. 최종 확인 ──→ 6. 프로젝트 종료           │
│                     (미팅)              (확인서)              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 종료 일정

| 시점 | 활동 | 담당 |
|------|------|------|
| D-14 | 종료 준비 시작 | PM |
| D-7 | 산출물 최종 인수 | PM + 개발팀 |
| D-3 | 권한 회수 준비 | PM + 담당자 |
| D-day | 최종 확인 미팅 | 전체 |
| D+1 | 권한 완전 회수 | PM |
| D+7 | 프로젝트 아카이빙 | PM |

## 8. 최종 확인서 서명

### 8.1 프로젝트 종료 확인서

```
프로젝트 종료 확인서

프로젝트명: Konda 서비스 개발
기간: YYYY-MM-DD ~ YYYY-MM-DD

본 확인서로 아래 사항을 확인합니다.

1. 모든 계약상 산출물이 인수되었음
2. 산출물의 품질이 합의된 기준을 충족함
3. 모든 권한 및 계정이 회수/정리되었음
4. 기술 이관이 완료되었음
5. 하자보수 조건에 합의함

[내부]
확인일: _______________
담당자: _______________ (서명)
직책: _______________

[외주사]
확인일: _______________
담당자: _______________ (서명)
직책: _______________
```

### 8.2 서명 절차

```
1. 모든 체크리스트 완료 확인
2. 종료 확인서 작성
3. 내부 담당자 서명
4. 외주사 담당자 서명
5. 양측 사본 보관
```
