# NDA 및 보안서약

## 1. 개요

본 문서는 Konda 프로젝트 참여에 필요한 비밀유지계약(NDA) 및 보안서약 내용을 정의합니다. 모든 외주사 참여자는 프로젝트 참여 전 본 문서의 내용에 동의하고 서명해야 합니다.

---

## 2. 비밀유지계약 (NDA)

### 2.1 기밀정보의 정의

본 계약에서 "기밀정보"란 다음을 포함하되 이에 한정되지 않습니다:

1. **사업 정보**
   - Konda 서비스 사업 계획 및 전략
   - 마케팅 계획 및 고객 정보
   - 재무 정보 및 가격 정책

2. **기술 정보**
   - 소스 코드 및 알고리즘
   - 시스템 아키텍처 및 설계 문서
   - API 명세 및 기술 문서
   - 데이터베이스 스키마

3. **금융 정보**
   - 금융 거래 로직
   - 결제 시스템 정보
   - 보안 인증 정보

4. **사용자 정보**
   - 개인정보
   - 사용자 행동 데이터
   - 거래 내역

### 2.2 기밀유지 의무

수령자는 다음 의무를 준수해야 합니다:

1. **비공개 의무**
   - 기밀정보를 제3자에게 공개하지 않음
   - 프로젝트 목적 외 사용 금지

2. **보호 의무**
   - 합리적인 보안 조치 적용
   - 기밀정보에 대한 접근 권한 최소화

3. **반환 의무**
   - 프로젝트 종료 시 모든 기밀정보 반환 또는 파기
   - 요청 시 파기 증명서 제출

### 2.3 예외 사항

다음의 경우 기밀유지 의무가 면제됩니다:

- 공개 당시 이미 공개된 정보
- 수령자가 독자적으로 개발한 정보
- 법적 요구에 의한 공개 (사전 통지 필요)

### 2.4 기밀유지 기간

- **프로젝트 진행 중**: 모든 기밀정보에 대해 유지
- **프로젝트 종료 후**: 3년간 유지

## 3. 보안서약

### 3.1 보안 준수 사항

프로젝트 참여자는 다음 보안 사항을 준수해야 합니다:

#### 3.1.1 계정 및 접근 관리
- [ ] 개인 계정 공유 금지
- [ ] 강력한 비밀번호 사용 (최소 12자, 대소문자+숫자+특수문자)
- [ ] 2단계 인증(2FA) 활성화
- [ ] 퇴사/프로젝트 이탈 시 즉시 계정 반납

#### 3.1.2 데이터 보호
- [ ] 업무용 기기에서만 작업
- [ ] 개인 기기로 기밀정보 복사 금지
- [ ] 화면 캡처/녹화 시 승인 필요
- [ ] 공용 네트워크에서 작업 시 VPN 사용

#### 3.1.3 코드 및 문서 관리
- [ ] 승인된 저장소에만 코드 저장
- [ ] 개인 저장소/클라우드에 코드 업로드 금지
- [ ] 기밀 문서 외부 공유 금지

#### 3.1.4 금융 데이터 처리
- [ ] 금융 데이터 접근 최소화
- [ ] 테스트 시 실제 금융 데이터 사용 금지
- [ ] 금융 정보 로깅 시 마스킹 처리

### 3.2 금지 행위

다음 행위는 엄격히 금지됩니다:

1. **정보 유출**
   - 소스 코드 외부 유출
   - 기밀 문서 무단 공유
   - 고객 정보 유출

2. **무단 접근**
   - 권한 없는 시스템 접근
   - 타인 계정 사용
   - 보안 우회 시도

3. **악성 행위**
   - 악성 코드 삽입
   - 시스템 취약점 악용
   - 데이터 무단 수정/삭제

### 3.3 보안 사고 보고

보안 사고 발생 또는 의심 시:

1. **즉시 보고**: 발견 후 1시간 이내
2. **보고 대상**: 내부 PM 및 보안 담당자
3. **보고 내용**:
   - 사고 발생 일시
   - 사고 내용 및 영향 범위
   - 초기 대응 조치

## 4. 서약서 양식

```
보안 서약서

본인은 Konda 프로젝트 참여에 있어 다음 사항을 준수할 것을 서약합니다.

1. 프로젝트 수행 중 취득한 모든 기밀정보를 보호하고,
   프로젝트 목적 외 사용하지 않겠습니다.

2. 기밀정보를 제3자에게 공개하거나 유출하지 않겠습니다.

3. 본 문서에 명시된 모든 보안 준수 사항을 이행하겠습니다.

4. 보안 사고 발생 시 즉시 보고하겠습니다.

5. 프로젝트 종료 시 모든 기밀정보를 반환하거나 파기하겠습니다.

6. 본 서약을 위반할 경우 민형사상 책임을 질 수 있음을 인지합니다.


서약일: _______________

소속: _______________

성명: _______________ (서명)

연락처: _______________
```

## 5. 위반 시 조치

### 5.1 위반 단계별 조치

| 위반 수준 | 예시 | 조치 |
|----------|------|------|
| 경미 | 보안 교육 미이수 | 경고 및 재교육 |
| 중간 | 보안 정책 미준수 | 시스템 접근 제한, 서면 경고 |
| 심각 | 기밀정보 유출 | 프로젝트 제외, 법적 조치 검토 |
| 치명적 | 의도적 정보 유출/악용 | 즉시 계약 해지, 법적 조치 |

### 5.2 법적 책임

- 기밀유지 의무 위반 시 손해배상 책임
- 개인정보보호법 위반 시 형사 처벌 가능
- 금융 정보 유출 시 관련 법규에 따른 처벌

## 6. 서명 절차

### 6.1 서명 프로세스

```
1. 보안 교육 이수
   ↓
2. NDA 및 보안서약 문서 수령
   ↓
3. 내용 검토 및 질의
   ↓
4. 서약서 작성 및 서명
   ↓
5. 내부 PM에게 제출
   ↓
6. 시스템 접근 권한 부여
```

### 6.2 제출 서류

- [ ] 보안 서약서 (서명본)
- [ ] 신분증 사본 (본인 확인용)
- [ ] 보안 교육 이수 확인서

### 6.3 제출처

- **제출 방법**: 스캔본 이메일 제출 또는 원본 직접 제출
- **제출 대상**: 내부 PM
