본 문서는 Konda 프로젝트의 개발/스테이징/프로덕션 환경 정보를 안내합니다.

## 2. 환경 구분

### 2.1 환경 개요

| 환경 | 용도 | 접근 권한 |
|------|------|----------|
| Development (Dev) | 개발 및 단위 테스트 | 개발팀 전체 |
| Staging (Stg) | QA 및 통합 테스트 | 개발팀 + QA |
| Production (Prod) | 실서비스 | 제한된 인원 |

### 2.2 환경 흐름

```
Development → Staging → Production
  (개발)      (테스트)    (운영)
```

## 7. 네트워크 구성

### 7.1 네트워크 다이어그램 (개략)

```
┌─────────────────────────────────────────────────────────────┐
│                         인터넷                               │
└─────────────────────────┬───────────────────────────────────┘
                          │
                    ┌─────┴─────┐
                    │   CDN     │
                    └─────┬─────┘
                          │
              ┌───────────┴───────────┐
              │                       │
        ┌─────┴─────┐           ┌─────┴─────┐
        │ 랜딩 페이지 │           │ API 서버  │
        └───────────┘           └─────┬─────┘
                                      │
                          ┌───────────┴───────────┐
                          │                       │
                    ┌─────┴─────┐           ┌─────┴─────┐
                    │ BE(비금융) │           │ BE(금융)   │
                    └─────┬─────┘           └─────┬─────┘
                          │                       │
                    ┌─────┴─────┐           ┌─────┴─────┐
                    │    DB     │           │    DB     │
                    │  (비금융)  │           │  (금융)   │
                    └───────────┘           └───────────┘
```

### 7.2 접근 제한

- 스테이징/프로덕션 서버: VPN 또는 IP 화이트리스트
- 금융 시스템: 내부 네트워크에서만 접근
- DB: 애플리케이션 서버에서만 접근
