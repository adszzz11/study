## 3. 전체 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────┐
│                        사용자                                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     Konda App (내부)                         │
│                    - iOS / Android                          │
└─────────────────────────────────────────────────────────────┘
                    │                    │
                    ▼                    ▼
┌───────────────────────────┐  ┌───────────────────────────┐
│     BE - 비금융 (외주사)     │  │     BE - 금융 (내부)        │
│                           │  │                           │
│  - 일반 서비스 API          │  │  - 금융 거래 API            │
│  - 사용자 관리 API          │  │  - 결제 API                │
│  - 콘텐츠 API              │  │  - 자산 관리 API            │
└───────────────────────────┘  └───────────────────────────┘
```

### 3.1 API 연동 구조
- **App ↔ BE(비금융)**: 직접 통신
- **App ↔ BE(금융)**: 직접 통신
- **BE(비금융) ↔ BE(금융)**: 직접 통신 없음 (App을 통해 필요시 연동)

### 3.2 저장소 구조
- **내부 Repository**: App, BE(금융) 코드
- **외부 Repository**: BE(비금융), 랜딩 페이지 코드
