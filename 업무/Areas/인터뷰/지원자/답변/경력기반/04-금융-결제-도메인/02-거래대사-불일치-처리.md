# 거래대사 불일치가 발생했을 때 처리 프로세스는?

## 답변

선불전자지급수단 및 카드 결제 시스템에서 거래대사는 자사 시스템과 외부 결제 시스템(VAN사, 카드사) 간의 거래 정합성을 검증하는 핵심 프로세스입니다. 불일치 발생 시 처리 프로세스는 다음과 같습니다.

**1. 자동 대사 프로세스 (Daily Reconciliation)**
- 매일 새벽 배치 작업으로 전일 거래 데이터를 자동 대사
- 자사 거래 내역과 외부사 정산 파일(VAN사/카드사)을 비교
- 거래 일련번호, 금액, 거래 시각을 기준으로 매칭

**2. 불일치 탐지 및 분류**
불일치가 발견되면 즉시 다음 3가지 유형으로 분류:
- **금액 불일치**: 거래는 존재하나 금액이 다른 경우
- **거래 누락**: 한쪽 시스템에만 거래가 존재하는 경우
- **중복 거래**: 동일 거래가 중복 기록된 경우

**3. 불일치 원인 분석 및 처리**
- 자동 재대사: 1시간 후 한 번 더 대사 시도 (지연 전송 가능성)
- 수동 확인: 원본 거래 로그, API 호출 이력, 네트워크 로그 확인
- 보상 트랜잭션: 시스템 오류로 인한 불일치는 자동 보정
- 수동 조정: 복잡한 케이스는 운영팀이 양측 시스템 확인 후 수동 처리

**4. 모니터링 및 알림**
- 불일치 건수가 임계값(예: 10건) 초과 시 Slack/이메일 즉시 알림
- 불일치 금액이 큰 경우(예: 100만원 이상) 즉시 에스컬레이션

**5. 재발 방지**
- 불일치 원인 분석 후 시스템 개선 (타임아웃 조정, 재시도 로직 보강 등)
- 주간 회고를 통한 불일치 패턴 분석 및 예방 조치

## 핵심 키워드

- 거래 대사 (Transaction Reconciliation)
- 불일치 처리
- 정합성 검증
- 보상 트랜잭션
- 수동 조정

## 거래대사 프로세스

### 1. 자동 대사

```java
@Service
public class ReconciliationService {

    @Autowired
    private TransactionHistoryRepository transactionRepository;

    @Autowired
    private VanSettlementRepository vanSettlementRepository;

    @Autowired
    private ReconciliationResultRepository resultRepository;

    /**
     * 일별 자동 대사 배치 작업
     * 매일 새벽 2시 실행
     */
    @Scheduled(cron = "0 0 2 * * *")
    public void dailyReconciliation() {
        LocalDate yesterday = LocalDate.now().minusDays(1);

        log.info("거래 대사 시작: {}", yesterday);

        // 1. 자사 거래 내역 조회
        List<TransactionHistory> ourTransactions = transactionRepository
            .findByTransactionDate(yesterday);

        // 2. VAN사 정산 파일 조회
        List<VanSettlement> vanTransactions = vanSettlementRepository
            .findBySettlementDate(yesterday);

        // 3. 매칭 처리
        ReconciliationResult result = matchTransactions(ourTransactions, vanTransactions);

        // 4. 불일치 건 처리
        if (result.hasUnmatched()) {
            handleUnmatchedTransactions(result);
        }

        // 5. 대사 결과 저장
        resultRepository.save(result);

        log.info("거래 대사 완료: 총 {}건, 일치 {}건, 불일치 {}건",
            result.getTotalCount(),
            result.getMatchedCount(),
            result.getUnmatchedCount()
        );
    }

    /**
     * 거래 매칭 로직
     */
    private ReconciliationResult matchTransactions(
        List<TransactionHistory> ourTransactions,
        List<VanSettlement> vanTransactions) {

        ReconciliationResult result = new ReconciliationResult();

        // 빠른 조회를 위해 Map으로 변환
        Map<String, VanSettlement> vanMap = vanTransactions.stream()
            .collect(Collectors.toMap(VanSettlement::getApprovalNo, v -> v));

        for (TransactionHistory ourTx : ourTransactions) {
            VanSettlement vanTx = vanMap.get(ourTx.getApprovalNo());

            if (vanTx == null) {
                // 거래 누락: 자사에는 있으나 VAN사에는 없음
                result.addUnmatched(new UnmatchedTransaction(
                    ourTx, null, UnmatchedType.MISSING_IN_VAN
                ));
            } else if (!ourTx.getAmount().equals(vanTx.getAmount())) {
                // 금액 불일치
                result.addUnmatched(new UnmatchedTransaction(
                    ourTx, vanTx, UnmatchedType.AMOUNT_MISMATCH
                ));
            } else {
                // 정상 매칭
                result.addMatched(ourTx, vanTx);
            }

            vanMap.remove(ourTx.getApprovalNo());
        }

        // VAN사에만 있는 거래 (자사에 누락)
        for (VanSettlement remainingVanTx : vanMap.values()) {
            result.addUnmatched(new UnmatchedTransaction(
                null, remainingVanTx, UnmatchedType.MISSING_IN_OUR_SYSTEM
            ));
        }

        return result;
    }
}
```

```java
                // VAN사에는 있으나 자사에 없음

                    "거래 응답 수신 실패",

                    "거래 로그 누락"
                ));
                analysis.setRecommendedAction("원본 API 로그 확인 및 수동 복구");
                break;

            case AMOUNT_MISMATCH:
                // 금액 불일치
                analysis.setPossibleCauses(Arrays.asList(
                    "부분 취소 처리 시차",
                    "수수료 차이",
                    "데이터 파싱 오류"
                ));
                analysis.setRecommendedAction("거래 상세 내역 비교 후 수동 조정");
                break;
        }

        return analysis;
    }
}
```

```java
@Service
public class AmountMismatchHandler {

    /**
     * 금액 불일치 처리
     */
    @Transactional
    public void handleAmountMismatch(UnmatchedTransaction unmatched) {
        TransactionHistory ourTx = transactionRepository
            .findById(unmatched.getOurTransactionId())
            .orElseThrow();

        VanSettlement vanTx = vanSettlementRepository
            .findById(unmatched.getVanTransactionId())
            .orElseThrow();

        BigDecimal diff = ourTx.getAmount().subtract(vanTx.getAmount());

        // 차액이 미미한 경우 (예: 1원 이하 - 반올림 오차)
        if (diff.abs().compareTo(BigDecimal.ONE) <= 0) {
            log.warn("반올림 오차로 판단되는 미미한 금액 불일치: 차액 {}원", diff);
            autoCorrect(ourTx, vanTx);
            return;
        }

        // 부분 취소 여부 확인
        List<TransactionHistory> cancellations = transactionRepository
            .findCancellationsByOriginalTx(ourTx.getTransactionId());

        BigDecimal totalCancelled = cancellations.stream()
            .map(TransactionHistory::getAmount)
            .reduce(BigDecimal.ZERO, BigDecimal::add);

        if (ourTx.getAmount().subtract(totalCancelled).equals(vanTx.getAmount())) {
            log.info("부분 취소로 인한 금액 차이 확인됨");
            markAsResolved(unmatched, "부분 취소로 인한 정상 차이");
            return;
        }

        // 그 외의 경우 수동 처리 필요
```

### Case 2: 거래 누락

```java
@Service
public class MissingTransactionHandler {

    @Autowired
    private ApiCallLogRepository apiCallLogRepository;

    @Autowired
    private NetworkLogRepository networkLogRepository;

    /**
     * 거래 누락 처리 - 자사에는 있으나 VAN사에 없는 경우
     */
    @Transactional
    public void handleMissingInVan(UnmatchedTransaction unmatched) {
        TransactionHistory ourTx = transactionRepository
            .findById(unmatched.getOurTransactionId())
            .orElseThrow();

        // 1. API 호출 로그 확인
        Optional<ApiCallLog> apiLog = apiCallLogRepository
            .findByTransactionId(ourTx.getTransactionId());

        if (apiLog.isEmpty()) {
            log.error("API 호출 로그 없음: {}", ourTx.getTransactionId());
            manualReview(unmatched, "API 호출 로그 없음 - 시스템 오류 의심");
            return;
        }

        // 2. VAN사 응답 코드 확인
        String responseCode = apiLog.get().getResponseCode();
        if ("0000".equals(responseCode)) {
            // 정상 응답이었으나 VAN사 정산 파일에 누락
            // 1시간 후 재대사 스케줄링
            scheduleRetryReconciliation(unmatched, LocalDateTime.now().plusHours(1));
        } else if ("9999".equals(responseCode)) {
            // 타임아웃
            checkTransactionStatus(ourTx);  // VAN사에 거래 조회 API 호출
        } else {
            // 거래 실패 - 자사 시스템에서 롤백 필요
            rollbackTransaction(ourTx);
        }
    }

    /**
     * 거래 누락 처리 - VAN사에는 있으나 자사에 없는 경우
     */
    @Transactional
    public void handleMissingInOurSystem(UnmatchedTransaction unmatched) {
        VanSettlement vanTx = vanSettlementRepository
            .findById(unmatched.getVanTransactionId())
            .orElseThrow();

        // 원본 로그 확인
        Optional<NetworkLog> networkLog = networkLogRepository
            .findByApprovalNo(vanTx.getApprovalNo());

        if (networkLog.isPresent()) {
            // 거래는 발생했으나 DB 저장 실패 케이스
            recoverTransaction(vanTx, networkLog.get());
        } else {
            // 거래 기록 자체가 없음 - 심각한 오류
            criticalAlert(unmatched, "거래 로그 완전 누락");
        }
    }

    /**
     * 거래 복구
     */
    private void recoverTransaction(VanSettlement vanTx, NetworkLog networkLog) {
        TransactionHistory recovered = TransactionHistory.builder()
            .transactionId(networkLog.getTransactionId())
            .amount(vanTx.getAmount())
            .approvalNo(vanTx.getApprovalNo())
            .transactionAt(vanTx.getTransactionAt())
            .build();

        transactionRepository.save(recovered);

        auditLogService.logTransactionRecovered(recovered, "대사 중 복구됨");
    }
}
```

### Case 3: 중복 거래

```java
@Service
public class DuplicateTransactionHandler {

    /**
     * 중복 거래 처리
     */
    @Transactional
    public void handleDuplicate(UnmatchedTransaction unmatched) {
        // 동일 승인번호로 여러 건이 기록된 경우
        List<TransactionHistory> duplicates = transactionRepository
            .findByApprovalNo(unmatched.getOurApprovalNo());

        if (duplicates.size() <= 1) {
            log.warn("중복 아님: {}", unmatched.getOurApprovalNo());
            return;
        }

        // 최초 거래 식별 (가장 빠른 created_at)
        TransactionHistory original = duplicates.stream()
            .min(Comparator.comparing(TransactionHistory::getCreatedAt))
            .orElseThrow();

        // 중복 거래 목록
        List<TransactionHistory> realDuplicates = duplicates.stream()
            .filter(tx -> !tx.getTransactionId().equals(original.getTransactionId()))
            .collect(Collectors.toList());

        // 중복 거래 처리
        for (TransactionHistory duplicate : realDuplicates) {
            // 1. 잔액 복구 (중복 차감분 반환)
            restoreBalance(duplicate);

            // 2. 거래 상태 변경 (취소 처리)
            duplicate.setStatus(TransactionStatus.CANCELLED);
            duplicate.setCancelReason("중복 거래 자동 취소");
            transactionRepository.save(duplicate);

            // 3. 감사 로그
            auditLogService.logDuplicateCancelled(duplicate);
        }

        log.info("중복 거래 처리 완료: 원본 {}, 취소 {}건",
            original.getTransactionId(), realDuplicates.size());
    }
}
```

## 모니터링 및 알림

```java
@Service
public class ReconciliationMonitoringService {

    @Autowired
    private SlackNotificationService slackService;

    @Autowired
    private EmailService emailService;

    /**
     * 불일치 건 모니터링 및 알림
     */
    public void monitorUnmatchedTransactions(ReconciliationResult result) {
        int unmatchedCount = result.getUnmatchedCount();
        BigDecimal unmatchedAmount = result.getUnmatchedAmount();

        // 1. 건수 기준 알림
        if (unmatchedCount > 10) {
            slackService.sendAlert(
                "#payment-reconciliation",
                String.format("거래 대사 불일치 다량 발생: %d건", unmatchedCount)
            );
        }

        // 2. 금액 기준 알림
        if (unmatchedAmount.compareTo(new BigDecimal("1000000")) > 0) {
            emailService.sendToManagers(
                "거래 대사 고액 불일치 발생",
                String.format("총 불일치 금액: %s원", unmatchedAmount)
            );
        }

        // 3. 개별 고액 거래 즉시 알림
        result.getUnmatchedList().stream()
            .filter(u -> u.getAmount().compareTo(new BigDecimal("100000")) > 0)
            .forEach(u -> {
                slackService.sendUrgentAlert(
                    "#payment-urgent",
                    String.format("고액 거래 불일치: %s원, 승인번호: %s",
                        u.getAmount(), u.getOurApprovalNo())
                );
            });
    }

    /**
     * 대사 실패 알림
     */
    public void alertReconciliationFailure(Exception e) {
        slackService.sendCriticalAlert(
            "#payment-urgent",
            String.format("거래 대사 배치 실패: %s", e.getMessage())
        );

        emailService.sendToDevTeam(
            "거래 대사 시스템 장애",
            ExceptionUtils.getStackTrace(e)
        );
    }
}
```

```java
@Configuration
public class PaymentConfiguration {

    /**
     * VAN사 API 호출 재시도 설정
     */
    @Bean
    public RetryTemplate vanApiRetryTemplate() {
        RetryTemplate retryTemplate = new RetryTemplate();

        // 고정 지연 재시도 정책 (3초 간격, 최대 3회)
        FixedBackOffPolicy backOffPolicy = new FixedBackOffPolicy();
        backOffPolicy.setBackOffPeriod(3000L);
        retryTemplate.setBackOffPolicy(backOffPolicy);

        // 재시도 정책
        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);
        retryTemplate.setRetryPolicy(retryPolicy);

        return retryTemplate;
    }

    /**
     * 타임아웃 설정 강화
     */
    @Bean
    public RestTemplate vanApiRestTemplate() {
        HttpComponentsClientHttpRequestFactory factory =
            new HttpComponentsClientHttpRequestFactory();

        factory.setConnectTimeout(5000);     // 연결 타임아웃: 5초
        factory.setReadTimeout(30000);       // 읽기 타임아웃: 30초

        return new RestTemplate(factory);
    }
}
```

```java
@Aspect
@Component
public class PaymentLoggingAspect {

    /**
     * 모든 VAN사 API 호출 로깅
     */
    @Around("execution(* com.example.payment.van..*(..))")
    public Object logVanApiCall(ProceedingJoinPoint joinPoint) throws Throwable {
        String apiName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();

        String requestId = UUID.randomUUID().toString();

        log.info("[VAN_API_CALL] requestId={}, api={}, request={}",
            requestId, apiName, args);

        long startTime = System.currentTimeMillis();
        Object result = null;
        try {
            result = joinPoint.proceed();
            long elapsed = System.currentTimeMillis() - startTime;

            log.info("[VAN_API_SUCCESS] requestId={}, elapsed={}ms, response={}",
                requestId, elapsed, result);

            return result;
        } catch (Exception e) {
            long elapsed = System.currentTimeMillis() - startTime;

            log.error("[VAN_API_ERROR] requestId={}, elapsed={}ms, error={}",
                requestId, elapsed, e.getMessage(), e);

            throw e;
        }
    }
}
```

```java
@Component
public class ReconciliationHealthIndicator implements HealthIndicator {

    @Autowired
    private ReconciliationResultRepository resultRepository;

    @Override
    public Health health() {
        LocalDate today = LocalDate.now();
        LocalDate yesterday = today.minusDays(1);

        // 전일 대사 결과 확인
        Optional<ReconciliationResult> result = resultRepository
            .findByReconciliationDate(yesterday);

        if (result.isEmpty()) {
            return Health.down()
                .withDetail("reason", "전일 대사 결과 없음")
                .build();
        }

        ReconciliationResult r = result.get();
        if (r.getUnmatchedCount() > 50) {
            return Health.down()
                .withDetail("reason", "불일치 건수 과다")
                .withDetail("unmatchedCount", r.getUnmatchedCount())
                .build();
        }

        return Health.up()
            .withDetail("totalCount", r.getTotalCount())
            .withDetail("matchedCount", r.getMatchedCount())
            .withDetail("unmatchedCount", r.getUnmatchedCount())
            .build();
    }
}
```
