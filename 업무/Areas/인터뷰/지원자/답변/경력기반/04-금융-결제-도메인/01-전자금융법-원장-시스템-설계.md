# 전자금융법 개정에 따른 선불전자지급수단 원장 시스템 설계 시 법적 요구사항을 어떻게 반영했나요?

## 답변

전자금융법 개정에 따른 선불전자지급수단 원장 시스템 설계 시, 법적 요구사항을 충족하기 위해 다음과 같은 핵심 사항들을 시스템에 반영했습니다.

**1. 전자금융거래 기록 보존 의무 (전자금융법 제22조)**
- 모든 거래에 대해 거래일시, 거래금액, 거래상대방 정보를 5년간 보존할 수 있도록 원장 테이블과 히스토리 테이블을 설계
- 불변성 보장을 위해 Append-only 방식으로 모든 잔액 변동 이력을 저장
- 삭제가 불가능하도록 물리적 삭제 대신 논리적 삭제 방식 적용

**2. 선불전자지급수단 발행 한도 관리 (전자금융법 시행령 제12조)**
- 실시간 발행한도 집계 시스템 구현으로 법정 한도 초과 방지
- 1인당 최대 200만원 한도를 DB 제약조건과 애플리케이션 레벨에서 이중 검증
- 일별/월별 집계 배치를 통해 규제 보고 자동화

**3. 감사 추적(Audit Trail) 의무**
- 모든 트랜잭션에 대해 WHO(누가), WHEN(언제), WHAT(무엇을), HOW(어떻게) 정보를 로깅
- 변조 불가능한 감사 로그 테이블 설계 (INSERT ONLY, UPDATE/DELETE 권한 제거)
- 금융감독원 검사 시 즉시 제출 가능하도록 표준화된 감사 리포트 생성 기능 구현

**4. 실명 확인 및 본인 인증**
- NICE 본인인증 연동을 통한 실명 확인 프로세스 필수화
- 개인정보는 암호화 저장 (AES-256) 및 마스킹 처리

**5. 전자금융사고 대응 체계**
- 이상 거래 탐지 시스템(FDS) 구축으로 실시간 이상 패턴 모니터링
- 사고 발생 시 즉시 거래 정지 및 롤백 가능한 보상 트랜잭션 설계

## 법적 요구사항

### 1. 필수 준수 사항
- **전자금융거래 기록 보존**: 5년간 모든 거래 이력 보관 (전자금융법 제22조)
- **선불전자지급수단 발행한도**: 1인당 200만원 이하 (시행령 제12조)
- **실명 확인 의무**: 금융실명법에 따른 본인 인증 필수
- **거래내역 제공**: 이용자 요청 시 즉시 거래내역 제공 (법 제12조)
- **전자금융사고 보고**: 사고 발생 시 금융위원회/금융감독원 즉시 보고

### 2. 데이터 보관 의무
- **보관 기간**: 최소 5년 (전자금융법), 세법상 10년
- **보관 항목**:
  - 거래일시 (정확한 타임스탬프, 밀리초 단위)
  - 거래금액 (충전/사용/환불 구분)
  - 거래상대방 정보 (가맹점 정보)
  - 거래채널 (앱/웹/API)
  - 거래 승인번호
  - 거래 전후 잔액
  - 사용자 식별정보 (암호화)
- **변조 방지**: Audit Log는 불변성 보장, Hash 체인 또는 Checksum 활용

### 3. 보안 요구사항
- **개인정보 암호화**: AES-256 알고리즘 적용
- **접근 통제**: 역할 기반 접근 제어(RBAC)
- **전송 구간 암호화**: TLS 1.2 이상
- **로그 관리**: 보안 로그 6개월 이상 보관
- **DB 접근 제어**: 프로시저를 통한 간접 접근, 직접 DML 차단

### 데이터 모델

```sql
-- 원장 마스터 테이블 (잔액 관리)
CREATE TABLE prepaid_ledger (
    ledger_id VARCHAR(20) PRIMARY KEY,
    user_id VARCHAR(20) NOT NULL,
    balance DECIMAL(15,2) NOT NULL DEFAULT 0,
    total_charged DECIMAL(15,2) NOT NULL DEFAULT 0,  -- 총 충전액 (한도 관리용)
    status VARCHAR(10) NOT NULL,  -- ACTIVE, SUSPENDED, CLOSED
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    version INT NOT NULL DEFAULT 1,  -- Optimistic Lock
    CONSTRAINT balance_non_negative CHECK (balance >= 0),
    CONSTRAINT charge_limit CHECK (total_charged <= 2000000),  -- 200만원 한도
    INDEX idx_user_id (user_id)
);

-- 거래 이력 테이블 (모든 거래 기록, Append Only)
CREATE TABLE transaction_history (
    transaction_id VARCHAR(30) PRIMARY KEY,
    ledger_id VARCHAR(20) NOT NULL,
    transaction_type VARCHAR(10) NOT NULL,  -- CHARGE, USE, REFUND, EXPIRE
    amount DECIMAL(15,2) NOT NULL,
    balance_before DECIMAL(15,2) NOT NULL,
    balance_after DECIMAL(15,2) NOT NULL,
    merchant_id VARCHAR(20),  -- 사용처
    approval_no VARCHAR(20),
    transaction_at TIMESTAMP(3) NOT NULL,  -- 밀리초 단위
    channel VARCHAR(10) NOT NULL,  -- APP, WEB, API
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (ledger_id) REFERENCES prepaid_ledger(ledger_id),
    INDEX idx_ledger_transaction_at (ledger_id, transaction_at),
    INDEX idx_transaction_at (transaction_at)
);

-- 감사 로그 테이블 (변조 방지)
CREATE TABLE audit_log (
    audit_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    event_type VARCHAR(50) NOT NULL,  -- LEDGER_CREATE, BALANCE_CHANGE, etc.
    entity_type VARCHAR(50) NOT NULL,
    entity_id VARCHAR(30) NOT NULL,
    user_id VARCHAR(20),
    admin_id VARCHAR(20),
    action VARCHAR(20) NOT NULL,  -- CREATE, UPDATE, DELETE
    before_value TEXT,
    after_value TEXT,
    ip_address VARCHAR(45),
    user_agent VARCHAR(200),
    event_at TIMESTAMP(3) NOT NULL,
    checksum VARCHAR(64),  -- SHA-256 해시값
    INDEX idx_event_at (event_at),
    INDEX idx_entity (entity_type, entity_id)
) ENGINE=InnoDB;

-- DB 레벨 권한: INSERT만 허용, UPDATE/DELETE 차단
REVOKE UPDATE, DELETE ON audit_log FROM app_user;
GRANT INSERT, SELECT ON audit_log TO app_user;
```

### 트랜잭션 처리

```java
@Service
@Transactional
public class PrepaidLedgerService {

    @Autowired
    private PrepaidLedgerRepository ledgerRepository;

    @Autowired
    private TransactionHistoryRepository transactionRepository;

    @Autowired
    private AuditLogService auditLogService;

    /**
     * 선불금 충전 - 법정 한도 체크 포함
     */
    public TransactionResult charge(ChargeRequest request) {
        // 1. 원장 조회 및 비관적 락
        PrepaidLedger ledger = ledgerRepository
            .findByUserIdWithLock(request.getUserId())
            .orElseThrow(() -> new LedgerNotFoundException());

        // 2. 충전 한도 검증 (전자금융법 시행령 제12조)
        BigDecimal newTotalCharged = ledger.getTotalCharged().add(request.getAmount());
        if (newTotalCharged.compareTo(new BigDecimal("2000000")) > 0) {
            auditLogService.logChargeRejected(ledger, request, "CHARGE_LIMIT_EXCEEDED");
            throw new ChargeLimitExceededException("1인당 충전 한도(200만원)를 초과했습니다.");
        }

        // 3. 잔액 업데이트 (Optimistic Lock)
        BigDecimal balanceBefore = ledger.getBalance();
        ledger.charge(request.getAmount());
        ledgerRepository.save(ledger);

        // 4. 거래 이력 저장 (Append Only)
        TransactionHistory history = TransactionHistory.builder()
            .transactionId(generateTransactionId())
            .ledgerId(ledger.getLedgerId())
            .transactionType(TransactionType.CHARGE)
            .amount(request.getAmount())
            .balanceBefore(balanceBefore)
            .balanceAfter(ledger.getBalance())
            .approvalNo(request.getApprovalNo())
            .transactionAt(LocalDateTime.now())
            .channel(request.getChannel())
            .build();
        transactionRepository.save(history);

        // 5. 감사 로그 기록
        auditLogService.logBalanceChange(
            ledger.getUserId(),
            "CHARGE",
            balanceBefore,
            ledger.getBalance(),
            history.getTransactionId()
        );

        return TransactionResult.success(history);
    }

    /**
     * 선불금 사용 - 잔액 부족 체크
     */
    public TransactionResult use(UseRequest request) {
        PrepaidLedger ledger = ledgerRepository
            .findByUserIdWithLock(request.getUserId())
            .orElseThrow(() -> new LedgerNotFoundException());

        // 잔액 검증
        if (ledger.getBalance().compareTo(request.getAmount()) < 0) {
            auditLogService.logUseRejected(ledger, request, "INSUFFICIENT_BALANCE");
            throw new InsufficientBalanceException("잔액이 부족합니다.");
        }

        BigDecimal balanceBefore = ledger.getBalance();
        ledger.use(request.getAmount());
        ledgerRepository.save(ledger);

        TransactionHistory history = TransactionHistory.builder()
            .transactionId(generateTransactionId())
            .ledgerId(ledger.getLedgerId())
            .transactionType(TransactionType.USE)
            .amount(request.getAmount())
            .balanceBefore(balanceBefore)
            .balanceAfter(ledger.getBalance())
            .merchantId(request.getMerchantId())
            .transactionAt(LocalDateTime.now())
            .channel(request.getChannel())
            .build();
        transactionRepository.save(history);

        auditLogService.logBalanceChange(
            ledger.getUserId(),
            "USE",
            balanceBefore,
            ledger.getBalance(),
            history.getTransactionId()
        );

        return TransactionResult.success(history);
    }
}
```

### 감사 로그

```java
@Service
public class AuditLogService {

    @Autowired
    private AuditLogRepository auditLogRepository;

    /**
     * 잔액 변경 감사 로그
     */
    public void logBalanceChange(String userId, String action,
                                  BigDecimal before, BigDecimal after,
                                  String transactionId) {
        AuditLog log = AuditLog.builder()
            .eventType("BALANCE_CHANGE")
            .entityType("PREPAID_LEDGER")
            .entityId(userId)
            .userId(userId)
            .action(action)
            .beforeValue(before.toString())
            .afterValue(after.toString())
            .ipAddress(getClientIp())
            .userAgent(getUserAgent())
            .eventAt(LocalDateTime.now())
            .build();

        // 변조 방지 체크섬 생성
        log.setChecksum(calculateChecksum(log));

        auditLogRepository.save(log);
    }

    /**
     * 거부된 거래 로그 (이상 거래 탐지용)
     */
    public void logChargeRejected(PrepaidLedger ledger, ChargeRequest request, String reason) {
        AuditLog log = AuditLog.builder()
            .eventType("CHARGE_REJECTED")
            .entityType("PREPAID_LEDGER")
            .entityId(ledger.getLedgerId())
            .userId(ledger.getUserId())
            .action("CHARGE_ATTEMPT")
            .afterValue(String.format("amount=%s, reason=%s", request.getAmount(), reason))
            .ipAddress(getClientIp())
            .eventAt(LocalDateTime.now())
            .build();

        log.setChecksum(calculateChecksum(log));
        auditLogRepository.save(log);
    }

    /**
     * SHA-256 체크섬 계산 (변조 방지)
     */
    private String calculateChecksum(AuditLog log) {
        String data = String.format("%s|%s|%s|%s|%s|%s",
            log.getEventType(),
            log.getEntityId(),
            log.getAction(),
            log.getBeforeValue(),
            log.getAfterValue(),
            log.getEventAt()
        );

        try {
            MessageDigest digest = MessageDigest.getInstance("SHA-256");
            byte[] hash = digest.digest(data.getBytes(StandardCharsets.UTF_8));
            return Base64.getEncoder().encodeToString(hash);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("SHA-256 알고리즘을 찾을 수 없습니다.", e);
        }
    }
}
```

## 검증 및 인증

**1. 금융감독원 전자금융시스템 사전 신고**
- 시스템 구축 완료 후 금융감독원에 선불전자지급수단 발행업 신고
- 시스템 구성도, 보안 정책, 거래 프로세스 문서화 제출

**2. 외부 보안 감사**
- 연 1회 정보보호 관리체계(ISMS) 인증 심사
- 모의 해킹 테스트 및 취약점 점검

**3. 내부 통제 시스템**
- 분기별 내부 감사를 통한 법적 준수 여부 점검
- 이상 거래 탐지 로그 정기 리뷰

**4. 금융감독원 정기 검사 대응**
- 거래 데이터 추출 쿼리 사전 준비
- 감사 리포트 자동 생성 기능 구현

```sql
-- 금융감독원 정기 검사용 거래 내역 추출 쿼리
SELECT
    th.transaction_id AS "거래번호",
    DATE_FORMAT(th.transaction_at, '%Y-%m-%d %H:%i:%s') AS "거래일시",
    th.transaction_type AS "거래유형",
    FORMAT(th.amount, 0) AS "거래금액",
    FORMAT(th.balance_after, 0) AS "거래후잔액",
    COALESCE(th.merchant_id, '-') AS "가맹점",
    th.channel AS "거래채널"
FROM transaction_history th
WHERE th.transaction_at BETWEEN '2024-01-01' AND '2024-12-31'
ORDER BY th.transaction_at DESC;

-- 일별 충전액 집계 (한도 관리 증적)
SELECT
    DATE(transaction_at) AS "거래일",
    COUNT(*) AS "충전건수",
    FORMAT(SUM(amount), 0) AS "총충전액",
    FORMAT(AVG(amount), 0) AS "평균충전액"
FROM transaction_history
WHERE transaction_type = 'CHARGE'
GROUP BY DATE(transaction_at)
ORDER BY DATE(transaction_at) DESC;
```
