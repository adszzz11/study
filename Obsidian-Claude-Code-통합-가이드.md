# Obsidian과 Claude Code 통합 가이드

## 개요

Obsidian은 강력한 로컬 우선 마크다운 노트 앱이며, Claude Code는 AI 기반 코드 어시스턴트입니다. 두 도구를 결합하면 지식 관리와 문서 작성을 크게 향상시킬 수 있습니다.

## 기본 설정

### 1. CLAUDE.md 파일 설정

Repository 루트에 `CLAUDE.md` 파일을 생성하여 Claude Code에게 컨텍스트를 제공합니다.

```markdown
# CLAUDE.md

## Repository Overview
이 저장소는 [목적 설명] 입니다.

## Repository Structure
- study/: 학습 노트
- marp/: 프레젠테이션 파일
- ...

## Working with Notes
- 모든 문서는 한글 마크다운 형식
- 주제별 폴더 구조 사용
```

### 2. .gitignore 설정

Obsidian 설정 파일 중 개인적인 것들은 제외:

```
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.DS_Store
```

## 주요 활용 방법

### 1. 노트 작성 자동화

**새로운 학습 노트 생성:**
```
"Java의 Optional에 대한 학습 노트를 study/Java/ 디렉토리에 만들어줘"
```

Claude Code가 자동으로:
- 적절한 구조로 마크다운 생성
- 코드 예제 포함
- 참조 링크 추가

**데일리 노트 템플릿:**
```
"오늘 날짜로 데일리 노트를 만들어줘. 템플릿에는 TODO, 학습내용, 회고 섹션을 포함해줘"
```

### 2. 문서 구조화 및 리팩토링

**기존 노트 정리:**
```
"study/HTTP/ 폴더의 모든 노트를 읽고 중복된 내용을 통합해줘"
```

**목차 자동 생성:**
```
"이 노트에 자동 목차를 추가해줘"
```

**태그 및 메타데이터 추가:**
```
"study/Spring/ 폴더의 모든 파일에 YAML frontmatter를 추가하고 적절한 태그를 달아줘"
```

### 3. 검색 및 정보 추출

**특정 주제 검색:**
```
"이 vault에서 'GC 튜닝'에 대한 내용을 모두 찾아줘"
```

**요약 및 정리:**
```
"study/Java/JVM.md 파일을 읽고 핵심 내용을 3줄로 요약해줘"
```

### 4. Marp 프레젠테이션 작성

**새 프레젠테이션 생성:**
```
"Spring Boot 아키텍처에 대한 Marp 프레젠테이션을 만들어줘.
슬라이드는 10장 정도로, 코드 예제를 포함해줘"
```

Marp 프론트매터 예시:
```yaml
---
marp: true
theme: default
paginate: true
---
```

### 5. 링크 관리

**백링크 확인:**
```
"이 노트를 참조하는 다른 노트들을 찾아줘"
```

**링크 검증:**
```
"깨진 내부 링크를 찾아서 고쳐줘"
```

**위키링크 변환:**
```
"모든 마크다운 링크를 Obsidian 위키링크 [[]] 형식으로 변환해줘"
```

### 6. 컨텐츠 생성

**학습 자료 생성:**
```
"HTTP 캐싱에 대한 상세한 가이드를 작성해줘.
다이어그램은 mermaid로, 예제는 실제 코드로 포함해줘"
```

**북 리뷰 템플릿:**
```
"book/ 폴더에 책 리뷰 템플릿을 만들어줘.
평점, 한줄평, 핵심내용, 인상깊은 구절 섹션을 포함해줘"
```

**컨퍼런스 노트 템플릿:**
```
"conference/ 폴더에 컨퍼런스 노트 템플릿을 만들어줘.
이벤트명, 발표자, 핵심 내용, 질의응답, 참고 링크 섹션을 포함해줘"
```

### 7. 코드 스니펫 관리

**코드 블록 정리:**
```
"이 노트의 모든 자바 코드에 적절한 주석을 추가해줘"
```

**실행 가능한 예제:**
```
"이 개념을 설명하는 실행 가능한 예제 코드를 추가해줘"
```

## 고급 워크플로우

### 1. 자동 인덱싱

모든 학습 노트의 인덱스 페이지 생성:

```
"study/ 폴더의 모든 서브폴더와 파일을 스캔해서
자동으로 인덱스 페이지를 만들어줘. 각 섹션별로 파일 목록과
간단한 설명을 포함해줘"
```

### 2. 지식 그래프 분석

```
"가장 많이 참조되는 노트 10개를 찾아줘"
"고립된 노트(링크가 없는)를 찾아서 리스트로 만들어줘"
```

### 3. 일괄 처리

**이미지 경로 수정:**
```
"marp/ 폴더의 모든 md 파일에서 이미지 경로를
상대경로로 변경해줘"
```

**날짜 포맷 통일:**
```
"모든 파일의 YAML frontmatter에서 날짜 형식을
YYYY-MM-DD로 통일해줘"
```

### 4. 스터디 그룹 협업

**변경사항 요약:**
```
"최근 커밋에서 변경된 노트들의 내용을 요약해줘"
```

**PR 생성:**
```
"새로 추가한 Spring 노트들을 정리해서 PR을 만들어줘"
```

## 베스트 프랙티스

### 1. 명확한 폴더 구조 유지
```
study/
  ├── Java/
  ├── HTTP/
  ├── Spring/
  └── ...
marp/
book/
conference/
```

### 2. 일관된 네이밍 컨벤션
- 파일명: `주제-세부사항.md`
- 날짜: `YYYY-MM-DD.md` (데일리 노트)
- 프레젠테이션: `제목.md` (marp 폴더 내)

### 3. CLAUDE.md 최신 상태 유지
- 새 폴더 추가 시 구조 업데이트
- 주요 규칙이나 포맷 변경 시 반영

### 4. Git 활용
```bash
# 정기적인 커밋
git add .
git commit -m "study: Java GC 노트 추가"

# Claude Code에게 커밋 요청
"학습 내용을 정리해서 커밋해줘"
```

## 유용한 프롬프트 모음

- "X에 대한 학습 노트를 작성해줘. 개념, 예제, 실무 활용을 포함해줘"
- "이 주제에 대한 Q&A 형식의 노트를 만들어줘"
- "X와 Y를 비교하는 표를 만들어줘"

### 정리 및 개선
- "이 노트의 가독성을 개선해줘"
- "핵심 내용을 불릿포인트로 정리해줘"
- "이 내용을 초보자가 이해하기 쉽게 다시 써줘"

### 검색 및 분석
- "X에 대한 내용이 있는 모든 노트를 찾아줘"
- "이번 주에 작성한 노트들을 요약해줘"
- "가장 긴 노트 5개를 찾아줘"

### Marp 프레젠테이션
- "이 노트 내용으로 15분짜리 발표 자료를 만들어줘"
- "슬라이드에 mermaid 다이어그램을 추가해줘"
- "프레젠테이션의 테마를 변경해줘"

### 컨퍼런스 노트
- "어제 참석한 AWS 밋업 노트를 conference/ 폴더에 정리해줘"
- "conference/ 폴더의 모든 노트에서 'MSA' 관련 내용을 찾아줘"
- "최근 3개월 동안 참석한 컨퍼런스 목록을 만들어줘"
- "이 컨퍼런스 노트의 핵심 내용을 블로그 포스트 형식으로 변환해줘"

## 문제 해결

### Claude Code가 파일을 찾지 못할 때
```
"현재 디렉토리 구조를 보여줘"
"study/ 폴더의 모든 .md 파일을 찾아줘"
```

### 한글 인코딩 문제
- 파일을 UTF-8로 저장했는지 확인
- Obsidian 설정에서 기본 인코딩 확인

### 대용량 vault 처리
- 특정 폴더만 지정해서 작업
- 파일 개수가 많을 때는 배치로 나눠서 처리

## 결론

Obsidian과 Claude Code를 함께 사용하면:
- 노트 작성 속도 향상
- 일관된 문서 구조 유지
- 자동화된 정리 및 인덱싱
- 효율적인 지식 관리

핵심은 명확한 구조를 유지하고, CLAUDE.md로 컨텍스트를 제공하며, 구체적인 프롬프트를 사용하는 것입니다.
