# 클라우드 플랫폼 경험을 설명해주세요

## English Question

**Describe your experience with cloud platforms (AWS, GCP, Azure).**

## 질문 번역

클라우드 플랫폼 (AWS, GCP, Azure) 경험을 설명해주세요.

## English Answer

**Primary Experience: AWS**
At Danal, we ran our Kubernetes infrastructure on AWS. My hands-on experience:

**Compute & Containers:**
- EKS (Elastic Kubernetes Service) - Primary platform for our microservices
- EC2 - For workloads that needed dedicated resources
- Managed node groups with auto-scaling

**Networking:**
- VPC design with public/private subnets
- ALB/NLB for load balancing
- Route 53 for DNS management
- VPN connections for hybrid cloud

**Data Services:**
- RDS (PostgreSQL, MySQL) - Managed databases
- ElastiCache (Redis) - Caching layer
- S3 - Object storage for logs, backups, static assets

**Observability:**
- CloudWatch for metrics and logs
- We also used external tools (Datadog, ELK) for deeper insights

**Security:**
- IAM policies and roles
- Security groups and NACLs
- Secrets Manager for credential management

**Secondary Experience: GCP**
- Used BigQuery for analytics workloads
- Cloud Functions for event-driven processing
- Familiar with GKE concepts

**What I'd Bring to Zonagent:**
I'm cloud-agnostic in my thinking. The principles of cloud architecture—scalability, resilience, cost optimization—translate across providers. I'd adapt to whatever stack you're using or help evaluate options if you're making decisions.

## 한글 번역

**주요 경험: AWS**

**컴퓨트 & 컨테이너:**

**네트워킹:**
- 퍼블릭/프라이빗 서브넷을 갖춘 VPC 설계
- 로드 밸런싱용 ALB/NLB
- DNS 관리용 Route 53
- 하이브리드 클라우드용 VPN 연결

**데이터 서비스:**
- RDS (PostgreSQL, MySQL) - 관리형 데이터베이스
- ElastiCache (Redis) - 캐싱 레이어

- 메트릭과 로그용 CloudWatch

- IAM 정책과 역할
