# 첫 주에 제가 너무 바빠서 온보딩을 못해준다면 어떻게 하시겠습니까

## English Question

**What would you do in your first week if I was too busy to onboard you?**

## 질문 번역

첫 주에 제가 너무 바빠서 온보딩을 못해준다면 어떻게 하시겠습니까?

---

## English Answer

That's fine—I'd onboard myself.

**Day 1-2: Environment Setup**
- Get development environment running
- Access to repositories, documentation, communication tools
- Read every document I can find: READMEs, architecture docs, past PRs

**Day 2-3: Understand the Codebase**
- Trace through the main code paths
- Identify how data flows through the system
- Note questions for later (but don't block on them)
- Run the application locally, poke around

**Day 3-4: Find Something to Contribute**
- Look for small bugs or improvements
- Check open issues tagged "good first issue" or similar
- Submit a small PR to understand the review process
- Even if it's just fixing a typo in docs—start contributing

**Day 4-5: Connect with the Team**
- Introduce myself to team members asynchronously
- Ask clarifying questions about what I've learned
- Understand current priorities and where I can help most

**Throughout the Week:**
- Document what I learn for future engineers
- Take notes on things that confused me
- Start building context on the domain and product

**My Philosophy:**
In a startup, nobody has time to hand-hold. The most valuable people are those who can figure things out independently while being thoughtful about when to ask for help.

By end of week one, I want to have shipped something—even if it's small.

## 한글 번역

괜찮습니다—스스로 온보딩하겠습니다.

**1-2일차: 환경 설정**
- 개발 환경 실행
- 저장소, 문서, 커뮤니케이션 도구 접근
- 찾을 수 있는 모든 문서 읽기: README, 아키텍처 문서, 과거 PR

**2-3일차: 코드베이스 이해**
- 메인 코드 경로 추적
- 시스템을 통해 데이터가 어떻게 흐르는지 파악
- 나중을 위한 질문 메모 (하지만 그것에 막히지 않음)
- 로컬에서 애플리케이션 실행, 둘러보기

**3-4일차: 기여할 것 찾기**
- 작은 버그나 개선 사항 찾기
- "good first issue" 등으로 태그된 열린 이슈 확인
- 리뷰 프로세스를 이해하기 위해 작은 PR 제출
- 문서의 오타 수정뿐이라도—기여 시작

**4-5일차: 팀과 연결**
- 팀원들에게 비동기적으로 자기 소개
- 배운 것에 대해 명확한 질문
- 현재 우선순위와 가장 도울 수 있는 곳 이해

**주 내내:**
- 미래 엔지니어를 위해 배운 것 문서화
- 혼란스러웠던 것들 메모
- 도메인과 제품에 대한 컨텍스트 구축 시작

**제 철학:**
스타트업에서는 아무도 손잡아줄 시간이 없습니다. 가장 가치 있는 사람은 도움을 요청할 때를 신중하게 생각하면서 독립적으로 알아낼 수 있는 사람입니다.

첫 주 끝에, 작더라도 무언가를 출시하고 싶습니다.
