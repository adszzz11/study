# AI/ML 시스템이 어떻게 작동하는지 이해하고 계신가요

## English Question

**What's your understanding of how AI/ML systems work?**

## 질문 번역

AI/ML 시스템이 어떻게 작동하는지 이해하고 계신가요?

---

## English Answer

I'm not an ML engineer, but I understand enough to be a productive collaborator. Here's my mental model:

**The High-Level Pipeline:**
1. **Data Collection & Preparation**: Often 80% of the work. Cleaning, labeling, transforming data into a format models can learn from.
2. **Training**: Feed data through algorithms to learn patterns. Requires experimentation—hyperparameters, architectures, training strategies.
3. **Evaluation**: Measure model performance against holdout data. Precision, recall, accuracy depending on the use case.
4. **Inference**: Deploy the trained model to make predictions on new data. This is where backend engineering matters most.

**Where I Add Value:**
- **Data pipelines**: Building robust systems that collect, clean, and prepare data for ML
- **Inference infrastructure**: Designing APIs to serve model predictions at scale
- **MLOps**: CI/CD for models, versioning, monitoring prediction quality over time
- **Integration**: Connecting ML outputs to the rest of the application

**What I've Done:**
At Danal, I've integrated AI tools into our development workflow—code review assistants, automated documentation. I understand the practical challenges: latency requirements, cost management, handling edge cases models don't cover.

**My Approach:**
I don't need to build the model, but I need to understand:
- What data it needs
- How fast predictions must be
- What happens when the model is wrong
- How to monitor for model drift

This lets me be an effective partner to ML engineers.

## 한글 번역

ML 엔지니어는 아니지만, 생산적인 협력자가 될 만큼 충분히 이해합니다. 제 멘탈 모델:

**하이레벨 파이프라인:**
1. **데이터 수집 & 준비**: 종종 작업의 80%. 모델이 배울 수 있는 형식으로 데이터 정리, 라벨링, 변환.
2. **훈련**: 패턴을 학습하기 위해 알고리즘에 데이터 공급. 실험 필요—하이퍼파라미터, 아키텍처, 훈련 전략.
3. **평가**: 홀드아웃 데이터에 대한 모델 성능 측정. 사용 사례에 따라 정밀도, 재현율, 정확도.
4. **추론**: 새 데이터에 대한 예측을 위해 훈련된 모델 배포. 백엔드 엔지니어링이 가장 중요한 곳.

**제가 가치를 더하는 곳:**
- **데이터 파이프라인**: ML을 위한 데이터를 수집, 정리, 준비하는 강건한 시스템 구축
- **추론 인프라**: 대규모로 모델 예측을 제공하는 API 설계
- **MLOps**: 모델을 위한 CI/CD, 버전 관리, 시간에 따른 예측 품질 모니터링
- **통합**: ML 출력을 나머지 애플리케이션에 연결

**해온 것:**
다날에서 개발 워크플로우에 AI 도구를 통합했습니다—코드 리뷰 어시스턴트, 자동화된 문서화. 실용적 도전을 이해합니다: 지연 요구사항, 비용 관리, 모델이 커버하지 못하는 엣지 케이스 처리.

**제 접근:**
모델을 구축할 필요는 없지만, 이해해야 합니다:
- 어떤 데이터가 필요한지
- 예측이 얼마나 빨라야 하는지
- 모델이 틀렸을 때 어떻게 되는지
- 모델 드리프트를 어떻게 모니터링하는지

이것이 ML 엔지니어에게 효과적인 파트너가 되게 합니다.
