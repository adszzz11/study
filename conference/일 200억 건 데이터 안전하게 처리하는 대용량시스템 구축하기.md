## 데이터 통합
Back Pressure
: 생산자 속도를 조절해 소비자가 안정적으로 데이터를 처리할 수 있게 도와주는 메커니즘

EDA

data input -> publisher -> kafka event
 -> listener -> data process event
 
publisher 서버를 앞에 두면 좋은점
1. 기술 추상화 가능
2. 메시지 형상 관리 가능
3. WOOWA RPC Server
	- 자바 인터페이스가 IDL
4. 순서 보장 가능

listener 서버를 앞에 두면 좋은점
1. back pressure 가능
	- concurrency -> consumer가 처리할 thread 개수를 따로 가져갈 수 있음
2.  시간대로 다시 생성 가능
3. 컨슘 제어 가능
