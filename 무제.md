## 코드 품질 & 설계

- **방어적 프로그래밍** - 외부 입력을 절대 믿지 마라
- **에러 핸들링 전략** - 언제 예외를 던지고, 언제 복구하고, 언제 무시할까
- **로깅 잘 남기기** - 장애 상황에서 원인을 찾을 수 있는 로그란
- **설정과 코드의 분리** - 환경별 설정 관리, 시크릿 관리
- **멱등성 보장하기** - 같은 요청이 여러 번 와도 안전한 API 설계
- **트랜잭션 경계 설정** - 어디서 시작하고 어디서 끝낼 것인가
- **동시성 문제 다루기** - 락, 데드락, 레이스 컨디션 예방
- **의존성 관리** - 라이브러리 버전 관리와 업데이트 전략

---

## 데이터베이스

- **인덱스, 언제 어떻게 걸까** - 무조건 거는 게 답이 아니다
- **쿼리 성능 분석하기** - 실행 계획 읽는 법
- **N+1 문제와 해결** - ORM 사용 시 흔한 실수
- **데이터베이스 마이그레이션 전략** - 무중단 스키마 변경
- **백업과 복구 전략** - 백업은 복구 테스트 전까지 믿지 마라
- **커넥션 풀 관리** - 적정 사이즈와 타임아웃 설정

---

## API 설계 & 통신

- **좋은 API 설계 원칙** - RESTful 관례, 버저닝, 네이밍
- **API 문서화** - 코드와 문서의 동기화
- **타임아웃과 재시도 전략** - 언제, 몇 번, 어떤 조건으로
- **서킷 브레이커 패턴** - 장애 전파 방지
- **비동기 처리 vs 동기 처리** - 언제 무엇을 선택할까
- **메시지 큐 활용** - 이벤트 드리븐 아키텍처의 기초

---

## 운영 & 모니터링

- **모니터링 3요소** - 메트릭, 로그, 트레이스
- **알림 설계** - 의미 있는 알림 vs 노이즈
- **헬스체크 제대로 구현하기** - liveness vs readiness
- **그레이스풀 셧다운** - 요청 처리 중 종료되면 안 된다
- **장애 대응 프로세스** - 발생 → 인지 → 대응 → 복구 → 회고
- **포스트모템 작성법** - 비난 없이 원인을 찾는 문화
- **용량 계획** - 트래픽 예측과 스케일링 전략

## 배포 & 인프라

- **CI/CD 파이프라인 구축** - 자동화의 기본
- **배포 전략** - 롤링, 블루-그린, 카나리 비교
- **롤백 전략** - 배포는 언제든 실패할 수 있다
- **환경 일관성** - 로컬, 개발, 스테이징, 프로덕션 차이 줄이기
- **인프라를 코드로** - IaC의 필요성과 시작하기
- **컨테이너 기초** - Docker 잘 쓰는 법
- **쿠버네티스 입문** - 왜 쓰는지, 기본 개념
