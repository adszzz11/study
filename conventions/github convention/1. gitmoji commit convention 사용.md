## 💡 왜 Gitmoji를 사용할까?

| 이유                  | 설명                                      |
| ------------------- | --------------------------------------- |
| 📌 **가독성**          | 이모지로 커밋 종류를 직관적으로 파악 가능                 |
| 📌 **일관성**          | 커밋 목적을 명확히 정리                           |
| 📌 **시각적 구분**       | GitHub, GitLab 등의 UI에서 커밋 히스토리를 구분하기 쉬움 |
| 📌 **자동화 도구 연계 가능** | 릴리즈 자동화나 changelog 생성 시 유용              |

---

## 🧱 대표적인 Gitmoji 목록

| 이모지                    | 이름              | 설명                  |
| ---------------------- | --------------- | ------------------- |
| ✨ `:sparkles:`         | 기능 추가 (feat)    | 새로운 기능 추가           |
| 🐛 `:bug:`             | 버그 수정 (fix)     | 버그 해결               |
| ♻️ `:recycle:`         | 리팩토링 (refactor) | 로직 변경 없이 코드 구조 개선   |
| 🔥 `:fire:`            | 코드 제거 (remove)  | 불필요한 코드/파일 제거       |
| 📦 `:package:`         | 패키지 (package)   | 새로운 의존성 추가/업데이트     |
| 🎨 `:art:`             | 코드 스타일 개선       | 들여쓰기, 세미콜론 등 스타일 변경 |
| ✅ `:white_check_mark:` | 테스트 추가          | 테스트 코드 작성           |
| 📝 `:memo:`            | 문서 수정 (docs)    | README, 주석 등 문서 작업  |
| 🚀 `:rocket:`          | 배포 (deploy)     | 배포 관련 작업            |
| 🚧 `:construction:`    | 작업 중 (wip)      | 아직 완료되지 않은 작업       |
| 🔧 `:wrench:`          | 설정 변경 (config)  | CI, lint 등 설정 수정    |
| ⬆️ `:arrow_up:`        | 버전 업그레이드        | 패키지나 라이브러리 업        |
| ⬇️ `:arrow_down:`      | 버전 다운그레이드       | 패키지나 라이브러리 다운       |
이 외의 Gitmoji는 사용하지 않습니다. 새로운 그림이 나왔을 때 상호 간 헷갈릴 수 있기 때문입니다.
