# 프론트엔드 파트 요구 산출물 가이드

> 프론트엔드가 다른 파트에 요청하는 산출물 및 협업 가이드

---

## 핵심 역할

- 웹 프론트엔드 개발
- UI 컴포넌트 구현
- API 연동

---

## 프론트엔드가 생성하는 산출물

| 우선순위 | 산출물 | 설명 |
|---------|--------|------|
| 🔴 필수 | 화면 구현 | Figma 기반 UI 구현 |
| 🟡 권장 | 컴포넌트 문서 | Storybook 등 |
| 🟡 권장 | 기술 문서 | 아키텍처, 폴더 구조 등 |

---

## 다른 파트에 요청하는 산출물

### 기획/PO에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| Wireframe | 🔴 필수 | 화면 구성 파악 |
| Flow Chart | 🔴 필수 | 사용자 흐름 이해 |
| 기능 명세서 | 🟡 권장 | 상세 기능 파악 |
| User Story | 🟡 권장 | 사용자 관점 이해 |

**기획에게 요청사항:**
- [ ] 초반에 아키텍처 근간이 되는 구조 정의
- [ ] 기획 일관성 유지 (nav bar, baseline 등)
- [ ] 세부정책 놓치면 알려주세요
- [ ] 중간에 기획 변경 지양 (특히 아키텍처 근간)

### 디자인에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| Figma 디자인 명세 | 🔴 필수 | UI 구현 기준 |
| 디자인시스템 | 🔴 필수 | 일관된 컴포넌트 구현 |
| 디자인 플로우 | 🟡 권장 | 인터랙션 이해 |

**디자인에게 요청사항:**
- [ ] 디자인시스템 기반 재사용 컴포넌트 구성
- [ ] Theme, Token을 일관성 있게 구성
- [ ] 라이브러리에서 재현 가능한 수준의 디자인
- [ ] 팀 목표에 맞는 디자인 수준 조율
  - 빠른 출시가 목표 → 과도한 디자인 지양
  - 예술성이 중요 → 디자인 강조 가능

### 백엔드에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| API 명세 (Swagger) | 🔴 필수 | API 연동 |
| API 명세 (Postman/.http) | 🟡 권장 | 테스트 용이 |
| OpenAPI Schema | 🟡 권장 | TypeScript 타입 추출 |

**백엔드에게 요청사항:**
- [ ] **nullable 명시** (가장 중요!)
- [ ] JSON Depth 4단계 이하로 구성
- [ ] 데이터 정제는 서버에서 처리
- [ ] API 변경 시 사전 공유
- [ ] 공통코드 설명 상세 작성

## API 설계 권장사항 (백엔드에게 전달)

### JSON Depth 권장

```json
// ✅ Good - Depth 4 이하
{
  "user": {
    "profile": {
      "name": "홍길동",
      "avatar": "url"
    }
  }
}

// ❌ Bad - Depth 4 초과
{
  "data": {
    "user": {
      "profile": {
        "details": {
          "name": "홍길동"
        }
      }
    }
  }
}
```

- Material UI
- Ant Design
- shadcn/ui
- App UI

## 프론트엔드로서 팀에게 요청

> 프론트엔드 개발자도 팀에게 이해를 구하는 부분

### 기획자에게

- 세부정책을 놓칠 수 있습니다
- 발견 시 알려줘서 함께 정할 수 있도록 해주세요

### 백엔드에게

- 서버에서 처리해야 할 것과 프론트에서 처리해야 할 것을 명확히 협의하고 싶습니다
- 가능하면 서버 라인에서 데이터 정제해주시면 감사하겠습니다
