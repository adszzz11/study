# AOS (Android) 파트 요구 산출물 가이드

> AOS가 다른 파트에 요청하는 산출물 및 협업 가이드

---

## 핵심 역할

- Android 앱 개발
- 다양한 기기 대응 (폴더블, 다양한 해상도)
- API 연동

---

## 업무 프로세스

```
1. PM 업무 하달
2. 테크리드/챕터리드의 인원 & 공수 산정
3. 스프린트 리뷰 (기획자 소통)
4. 스프린트 시작
   - 백엔드 소통
   - 디자이너 소통
   - 기획자 소통
   - QA 소통
5. 배포
```

## ⚠️ Android 특수 고려사항

> **다른 파트에서 꼭 알아주셔야 하는 부분입니다**

### 기기 다양성

| 기기 유형 | 고려 수준 | 비고 |
|----------|----------|------|
| 일반 스마트폰 | 기본 | |
| **폴더블 (Fold)** | **필수** | 반드시 고려해주세요 |
| 플립 | 선택 | 상대적으로 여유 |
| 특수 해상도 | 케이스별 협의 | |

### 화면 대응

- 넓은 화면 / 좁은 화면 모두 고려 필요
- **iPhone 기준 디자인을 그대로 적용하지 말아주세요**

### AOS 개발자로서 호소

1. **Android 개발자라는 것을 인지해주세요**
2. **폴더블 기기를 고려해주세요**
3. **화면 크기 다양성을 고려해주세요**
4. **iPhone 디자인 ≠ Android 디자인**

## 다른 파트에 요청하는 산출물

### 기획/PO에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 디테일한 Wireframe | 🔴 필수 | 기획 의도가 분명해야 함 |

**기획에게 요청사항:**
- [ ] 기획 의도를 명확하게 표현 (맞고 틀리고 상관없이)
- [ ] **테스트베드 이후 기획 변경 지양** (중요!)
- [ ] 근거 없는 고집 지양

### 디자인에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 프로토타입 | 🔴 필수 | WireFrame 포함 |
| 일관된 화면 디자인 | 🔴 필수 | 통일된 UI |
| Animation Diagram | 🟡 권장 | 인터랙션 정의 |

**디자인에게 요청사항:**
- [ ] **Android 특성 고려** (iOS와 다름!)
- [ ] **폴더블(Fold) 기기 고려**
- [ ] **다양한 화면 크기 고려** (넓은 화면 / 좁은 화면)
- [ ] 디자인 통일성 유지
- [ ] iPhone 디자인 그대로 적용 지양

### 백엔드에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| API 문서 (Swagger/Docs) | 🔴 필수 | API 연동 |

**백엔드에게 요청사항:**
- [ ] **nullable 표시 정확하게** (중요!)
- [ ] 배포 시 커뮤니케이션
- [ ] Entity 값을 원형으로 최대한 내려주기
- [ ] Schema 과도하게 쪼개지 않기
- [ ] 배포 관련 소통 없이 변경 지양

### QA에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| QA 워크플로우 | 🟡 권장 | 업무 프로세스 정의 |
| 테스트 케이스 | 🔴 필수 | 기능별 테스트 항목 |
| 테스트 결과 | 🔴 필수 | 실행 결과 리포트 |

**QA에게 요청사항:**
- [ ] 구체적인 버그 리포트 (**기기 명시!**)
- [ ] **모든 파트에 균등한 테스트** (특정 파트만 누락 X)
- [ ] 해상도 차이로 인한 이슈 고려
- [ ] "안 돼요"만 말하지 않기 (기기/상황 설명 포함)
- [ ] "기획에서는 저렇게 되어있는데"만 말하지 않기 (해상도 다양성 고려)

## QA 테스트 시 고려사항

### Android 기기 다양성 테스트

| 유형 | 테스트 필요 수준 |
|------|-----------------|
| 일반 스마트폰 | 기본 |
| **폴더블 (Fold)** | **필수** |
| 플립 | 선택 |
| **다양한 해상도** | **필수** |

### 해상도 관련

- 넓은 화면 / 좁은 화면 모두 테스트 필요
- 해상도로 인한 이슈는 기획 문제가 아닐 수 있음
- 기획과 다르게 보인다면 기기/해상도 정보 함께 공유

## 밍글링 체크리스트

### 기획에게 확인

- [ ] 디테일한 Wireframe 있는가?
- [ ] 테스트베드 이후 변경 최소화 동의하는가?

- [ ] 프로토타입 있는가?

- [ ] nullable 명시할 수 있는가?
- [ ] 배포 커뮤니케이션 할 수 있는가?

- [ ] 테스트 케이스 만들 수 있는가?
- [ ] 기기 명시한 버그 리포트 작성할 수 있는가?
- [ ] 폴더블 기기 테스트할 수 있는가?
