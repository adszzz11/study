## 협업 파트별 필요 명세

### PO (기획) 파트

#### 필수 산출물
| 산출물 | 설명 |
|--------|------|
| 아이디어 기획서 | 아이디어 + 진행 방향 + 일정 + 세부내용 |
| 레퍼런스 | 참고할 아이디어/디자인 |
| 화면 기술서(정책서) | 디자인시스템 포함 |
| Wireframe | 화면 구성 |

#### 기획서 포함 항목
- 아이디어가 좋은 이유
- 진행 방향
- 일정
- 세부 내용

#### 했으면 좋겠는 것
- 의견 제시 시 이유를 명확하게
- 설득력 있는 제안

#### 하지 않았으면 좋겠는 것
- 화면이 어지러운 아이디어/디자인 예시
- 이유 없이 의견만 던지기

---

### View (화면 디자인) 관점

#### 디자이너의 요청 사항
- **UX/UI 디자인 통일성**이 가장 중요
- 순간적인 아이디에이션으로 새 디자인 제시 시 주의 필요
- 컬러가 다양하거나 캐릭터/컨셉이 추가되면 해결 방안 고민 필요

#### 했으면 좋겠는 것
- 새로운 디자인 제안 시 통일성 고려
- 디자인 확장 시 기존 시스템과의 조화 고려

#### 하지 않았으면 좋겠는 것
- 너무 컬러풀한 디자인 요구
- 기존 디자인시스템과 맞지 않는 새 컨셉 무작정 추가
- 색상/캐릭터/컨셉 추가 시 적용 난이도 미고려

---

## 디자이너 관점 핵심 정리

### 협업 시 가장 중요한 것
1. **디자인 통일성** 유지
2. 제안 시 **이유와 근거** 명시
3. **디자인시스템** 기반 작업
4. 새로운 요소 추가 시 **기존 시스템과의 조화** 고려

### 피해야 할 상황
- 통일성 없는 산발적 디자인 요청
- 근거 없는 의견 제시
- 과도하게 컬러풀한 디자인
- 새 컨셉 추가 시 구현 난이도 미고려
