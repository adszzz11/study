|--------|------|----------|

| **테스트 결과 리포트** | 문서 | AOS |

### 권장 산출물

| 산출물 | 형식 | 요청 파트 |
|--------|------|----------|

| 지표 | 목표 |
|------|------|

---

## 개발자들이 QA에게 원하는 것

| 요청 파트 | 핵심 요청 |
|----------|----------|
| **iOS** | 재현 가능한 버그 리포트, 에러/상황 명세, 구체적인 상황 설명 |
| **AOS** | 자체 프로세스 정의, **기기 명시**, 해상도 고려, 균등한 테스트 |

---

## DO (해주세요)

- **구체적인 에러 상황 설명**
- **재현 가능한 버그 리포트** (기기/환경 정보 포함!)
- **자신만의 프로세스 정의 및 운영** (AOS 권장)
- **모든 파트에 균등한 테스트**

---

## DON'T (피해주세요)

- **구체적인 상황 설명 없이 보고** (원인/상황 설명 필요)
- **"기획에서는 저렇게 되어있는데"만 말하기** (해상도 다양성 고려)
- **특정 파트만 테스트 누락**
- 기기 정보 없이 버그 리포트

---

## 버그 리포트 템플릿 (권장)

| 항목 | 내용 |
|------|------|
| **환경 - 기기** | (예: Galaxy S23, iPhone 15 Pro) |
| **환경 - OS 버전** | (예: Android 14, iOS 17.2) |
| **환경 - 앱 버전** | (예: 1.2.3) |
| **재현 단계** | 1. ... 2. ... 3. ... |
| **예상 결과** | |
| **실제 결과** | |
| **심각도** | Critical / High / Medium / Low |
| **첨부** | 스크린샷/영상 |

**심각도 기준**
- Critical: 앱 크래시, 데이터 손실
- High: 주요 기능 불가
- Medium: 기능 제한적 사용 가능
- Low: 사소한 UI 문제

---

## 버그 리포트 예시

### 개선이 필요한 사례
- "안 돼요" → 구체적인 상황 추가 필요
- "기획이랑 다른데요" → 기기/해상도 정보 추가 필요
- "제 폰에서는 안 돼요" → 기기 모델명 추가 필요

### 권장 사례
- "Galaxy S23 (Android 14)에서 결제 버튼 클릭 시 앱이 크래시됩니다. 재현 단계: 1. 장바구니 진입 2. 결제하기 클릭 3. 크래시 발생"
- "기획서에는 A로 되어있는데, 해상도 320dp 이하 기기에서는 레이아웃이 깨져서 B로 대응하는 것이 어떨까요?"

## QA 워크플로우 권장 구성

| 순서 | 단계 |
|------|------|
| 1 | 테스트 케이스 작성 |
| 2 | 테스트 실행 |
| 3 | 결과 리포트 작성 |
| 4 | 버그 등록 (기기/환경 명시!) |
| 5 | 수정 확인 |
| 6 | 재테스트 |

**AOS 조언**: QA가 직접 업무 프로세스를 정의하면 더 명확해질 수 있음
