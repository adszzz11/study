## Android 특수 고려사항 (중요)

> Android 플랫폼의 특성을 반드시 고려해주시기 바랍니다.

| 기기 유형 | 고려 수준 | 비고 |
|----------|----------|------|
| 일반 스마트폰 | 기본 | |
| **폴더블 (Fold)** | **필수** | 반드시 고려! |
| 플립 | 선택 | 상대적으로 여유 |
| 특수 해상도 | 케이스별 협의 | |

- 넓은 화면 / 좁은 화면 모두 고려

## 다른 파트에 요청하는 산출물

### 기획에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 디테일한 Wireframe | 필수 | 기획 의도 명확 |

### 디자인에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| 프로토타입 | 필수 | WireFrame 포함 |
| Animation Diagram | 권장 | 인터랙션 정의 |
| 일관된 화면 디자인 | 필수 | 통일된 UI |

### 백엔드에게

| 산출물 | 우선순위 | 이유 |
|--------|---------|------|
| API 문서 (Swagger) | 필수 | API 연동 |

---

## 기획에게 원하는 것

- [ ] **기획 의도를 명확하게 표현** (맞고 틀리고 상관없이)
- [ ] 디테일한 Wireframe

### DON'T
- **테스트베드 이후 기획 변경** (개발 일정에 큰 영향)
- 근거 없는 고집

---

## 디자인에게 원하는 것

- [ ] **Android 특성 고려** (iOS와 다름!)
- [ ] **폴더블(Fold) 기기 고려**
- [ ] **다양한 화면 크기 고려**
- [ ] 디자인 통일성 유지

### DON'T
- **iPhone 디자인만 보고 Android에 동일하게 요구**
- **폴더블(Fold) 기기 미고려**
- 특수 기종 미고려

---

## 백엔드에게 원하는 것

- [ ] **nullable 표시 정확하게**
- [ ] **배포 시 커뮤니케이션**
- [ ] **Entity 값을 원형으로 내려주기** (필요한 데이터만 선별 가능하도록)

### DON'T
- Schema 과도하게 쪼개기
- nullable 표시 누락
- 배포 관련 소통 없이 변경

---

## QA에게 원하는 것

- [ ] **구체적인 버그 리포트** (기기 명시!)
- [ ] **모든 파트에 균등한 테스트** (서운함 유발 방지)
- [ ] **해상도 차이로 인한 이슈 고려**
- [ ] 자신만의 프로세스 정의 및 운영

### DON'T
- 구체적인 상황 설명 없이 보고 (기기 명시 필요)
- 해상도 다양성을 고려하지 않은 피드백
- 특정 파트만 테스트 누락
