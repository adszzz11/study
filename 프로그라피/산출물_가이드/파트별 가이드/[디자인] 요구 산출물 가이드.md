## 디자인시스템 구성 요소

### 1. 토큰 (Token)

| 유형 | 설명 | 예시 |
|------|------|------|
| Primitive Token | RGBA 값 (기본 색상) | `#FF5733`, `rgba(255,87,51,1)` |
| Semantic Token | 의미 기반 정의 | `primary`, `secondary`, `error` |

### 2. 컴포넌트

- **버튼**: Primary, Secondary, Tertiary, Ghost
- **입력 필드**: Text, Password, Search, TextArea
- **카드**: 기본, 리스트, 상세
- **모달**: Alert, Confirm, Bottom Sheet
- **네비게이션**: Tab Bar, Navigation Bar, Bottom Navigation

### 3. 레이아웃

- 그리드 시스템
- 여백 규칙 (Spacing)
- 타이포그래피 스케일

## 플랫폼별 고려사항

### iOS

| 항목 | 고려사항 |
|------|----------|
| 가이드라인 | Human Interface Guidelines 참고 |
| 컴포넌트 | iOS 네이티브 컴포넌트 스타일 고려 |

### AOS (⚠️ 특히 주의)

| 항목 | 고려사항 | 중요도 |
|------|----------|--------|
| 가이드라인 | Material Design 참고 | 높음 |
| **폴더블 대응** | Fold 기기 레이아웃 | **필수** |
| **해상도 다양성** | 넓은 화면 / 좁은 화면 | **필수** |
| 플립 | 상대적으로 여유 | 선택 |

### FE

| 항목 | 고려사항 |
|------|----------|
| 반응형 | 다양한 브라우저 크기 대응 |
| 라이브러리 호환 | Material UI, Ant Design 등에서 재현 가능한 수준 |

## Android 기기 다양성 대응

> AOS 개발자가 특히 요청하는 부분

| 기기 유형 | 대응 필요 수준 |
|----------|---------------|
| 일반 스마트폰 | 기본 |
| 폴더블 (Fold) | **필수** |
| 플립 | 선택 |
| 특수 해상도 | 케이스별 협의 |

### AOS 개발자의 호소

1. **Android 플랫폼 특성을 고려해주세요**
2. **iPhone 디자인을 그대로 Android에 적용하지 말아주세요**
3. **폴더블 기기를 고려해주세요**
4. **다양한 화면 크기를 고려해주세요**

## 협업 시 주의사항

### DO (해주세요)

- [ ] 디자인시스템 기반 구성을 먼저 진행
- [ ] Theme, Token을 일관성 있게 구성
- [ ] 라이브러리에서 재현 가능한 수준의 디자인
- [ ] 구현 시간 고려한 대안 제시
- [ ] Android 특성 고려 (폴더블, 해상도 다양성)
- [ ] 디자인 통일성 유지

### DON'T (피해주세요)

- [ ] iPhone 디자인을 Android에 그대로 적용
- [ ] 과도한 Animation 요구 (구현 시간 미고려)
- [ ] 디자인시스템과 맞지 않는 새 컨셉 무작정 추가
- [ ] 너무 컬러풀한 디자인 (통일성 저해)
- [ ] 개발자의 구현 수준을 벗어나는 UX/UI 요구

## Figma Dev Mode 핸드오프 가이드

### Dev Mode 활용 방법

| 기능 | 용도 | 개발자 활용 |
|------|------|------------|
| **Inspect** | CSS/Swift/Kotlin 코드 추출 | 스타일 값 복사 |
| **Compare** | 디자인 변경사항 비교 | 변경 영향 파악 |
| **Ready for dev** | 개발 준비 완료 표시 | 작업 우선순위 파악 |
| **Variables** | 토큰 값 확인 | 디자인 시스템 연동 |

### 핸드오프 체크리스트

```markdown
## 개발 전달 시 확인

### 화면 설계
- [ ] 모든 상태 디자인 완료 (Default, Hover, Active, Disabled, Error)
- [ ] 반응형 브레이크포인트 정의 (있는 경우)
- [ ] 로딩 상태 디자인

### 컴포넌트
- [ ] 컴포넌트 Variants 정의
- [ ] 아이콘 Export 설정 (SVG)
- [ ] 이미지 Export 설정 (1x, 2x, 3x)

### 스펙
- [ ] 여백, 크기 값 Auto Layout으로 정의
- [ ] 타이포그래피 토큰 적용
- [ ] 컬러 토큰 적용
```

## 애니메이션 명세 작성법

### 애니메이션 정의 템플릿

```markdown
## 애니메이션 명세

### 애니메이션 이름
[화면전환/버튼 클릭/모달 등]

### 트리거
- 사용자 액션: [클릭/스와이프/스크롤 등]

### 타이밍
- Duration: [200ms/300ms/500ms]
- Easing: [ease-in-out/linear/cubic-bezier]
- Delay: [0ms/100ms]

### 속성 변화
| 속성 | 시작 값 | 종료 값 |
|------|--------|--------|
| opacity | 0 | 1 |
| translateY | 20px | 0 |
| scale | 0.9 | 1 |

### 프로토타입 링크
[Figma 프로토타입 링크]
```

### 애니메이션 난이도 가이드

| 난이도 | 유형 | 구현 시간 | 예시 |
|-------|------|----------|------|
| **쉬움** | Fade, Slide | 0.5일 | 화면 전환, 모달 열림 |
| **보통** | Scale, Rotate | 1일 | 버튼 피드백, 로딩 |
| **어려움** | Path, Morph | 2-3일 | 복잡한 형태 변환 |
| **매우 어려움** | 물리 기반, 3D | 5일+ | 스프링, 파티클 |

## 다크모드/테마 가이드

### 테마 토큰 구조

```
Color Tokens
├── Primitive (원시 토큰)
│   ├── gray-100: #F5F5F5
│   ├── gray-900: #212121
│   ├── blue-500: #2196F3
│   └── ...
│
└── Semantic (의미 토큰)
    ├── Light Theme
    │   ├── background-primary: {gray-100}
    │   ├── text-primary: {gray-900}
    │   └── ...
    │
    └── Dark Theme
        ├── background-primary: {gray-900}
        ├── text-primary: {gray-100}
        └── ...
```

### 다크모드 체크리스트

- [ ] 모든 컬러에 Semantic 토큰 적용
- [ ] 다크모드용 이미지/아이콘 준비 (필요시)
- [ ] 그림자 값 다크모드 조정
- [ ] 대비 비율 확인 (WCAG AA 기준 4.5:1 이상)
