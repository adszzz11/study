# 디자인 파트 요구사항 명세서

> 개발자 인터뷰 기반 디자인 파트 요구사항 통합 정리

---

## 필수 산출물 체크리스트

### 1순위 (필수)

| 산출물 | 형식 | 설명 | 요청 파트 |
|--------|------|------|----------|
| Figma 디자인 명세 | Figma | 컴포넌트 및 화면 디자인 | FE |
| 디자인시스템 | Figma | 토큰화된 시스템, 컴포넌트 정의 | iOS, FE |
| 프로토타입 | Figma | WireFrame 포함 | AOS |

### 2순위 (권장)

| 산출물 | 형식 | 설명 | 요청 파트 |
|--------|------|------|----------|
| 디자인 플로우 | Figma | Interaction이 많은 경우 필수 | FE |
| Animation Diagram | Figma | 인터랙션 정의 | AOS |
| 일관된 화면 디자인 | - | 모나지 않은 통일된 UI | AOS |

---

## 디자인시스템 구성 요소

### 필수 토큰
- [ ] **Primitive Token**: RGBA 값 (기본 색상)
- [ ] **Semantic Token**: 의미 기반 정의

### 컴포넌트 정의 (iOS)
- [ ] 텍스트박스 디자인
- [ ] 버튼 디자인
- [ ] 기타 UI 컴포넌트

---

## DO (했으면 좋겠는 것)

### 디자인시스템
- [ ] **디자인시스템 기반 구성을 먼저 진행** (iOS, FE)
- [ ] **Theme, Token을 일관성 있게 구성** (FE)
- [ ] **토큰화가 잘 되어있으면 좋겠다** (iOS)
- [ ] **컴포넌트 통일화** (iOS)
- [ ] 화면 제작 시 정의나 예시 기반으로 작업 (iOS)

### 구현 가능성 고려
- [ ] **일반적인 라이브러리에서 재현 가능한 수준의 디자인** (FE)
  - Material UI, Ant Design, shadcn/ui 등
- [ ] **구현 시간 고려한 대안 제시** (iOS)
- [ ] 인터랙티브 구현할 시간이 없을 때 대안 생각 (iOS)

### 플랫폼 고려
- [ ] **Android 특성 고려한 디자인** (AOS)
- [ ] **다양한 화면 크기 고려** - 넓은 화면 / 좁은 화면 (AOS)
- [ ] **폴더블(Fold) 기기 고려** (AOS)

### 일관성
- [ ] **디자인 통일성 유지** (AOS)

---

## 협업 시 요청사항

### 구현 시간 고려
- [ ] Animation은 일정에 맞춰 조율 부탁드립니다 (iOS)
- [ ] 인터랙티브 요소는 구현 시간을 함께 고려해주시면 좋겠습니다 (iOS)
- [ ] 복잡한 UX/UI는 사전에 논의 후 진행하면 좋겠습니다 (FE)

### 플랫폼별 특성 고려
- [ ] Android와 iOS는 플랫폼 특성이 다르므로 별도 고려 부탁드립니다 (AOS)
- [ ] 폴더블(Fold) 기기 대응이 필요합니다 (AOS)
  - 플립은 상대적으로 여유가 있습니다
- [ ] 특수 기종은 사전 협의 후 대응 범위를 정하면 좋겠습니다 (AOS)

### 팀 목표 정렬
- [ ] 팀 목표에 맞춰 디자인 수준을 조율하면 좋겠습니다 (FE)
  - 빠른 출시가 목표일 때: 핵심 기능 중심
  - 예술성이 중요할 때: 디자인 강조 가능

---

## Android 개발 시 협조 요청사항

> AOS 개발자가 특히 요청드리는 부분

1. **Android 플랫폼 특성을 함께 고려해주시면 감사하겠습니다**
2. **폴더블 기기 대응을 함께 논의해주세요**
3. **다양한 화면 크기를 고려해주시면 좋겠습니다**

### Android 기기 다양성

| 유형 | 고려 수준 |
|------|----------|
| 일반 스마트폰 | 기본 |
| 폴더블 (Fold) | 필수 고려 |
| 플립 | 상대적으로 여유 |
| 특수 해상도 기기 | 케이스별 협의 |

## 파트별 주요 요청 요약

| 파트 | 핵심 요청 |
|------|----------|
| **FE** | 디자인시스템 기반, 라이브러리 재현 가능 수준, Token 일관성 |

| **AOS** | Android 특성 고려, 폴더블 대응, 플랫폼별 맞춤 디자인 |

## 디자이너로서 팀에게 호소하는 것

> PM.md raw 데이터 기반 디자이너 관점 정리

### 일정 관련

- [ ] 왔다갔다 하는 작업이 좀 많아도 이해 부탁드립니다
- [ ] 초반에는 매니징을 좀 덜 해도 이해해주세요
  - 시장조사, 설문 신뢰도가 낮을 때 기한이 늘어날 수 있습니다
  - 디자인 외에도 할 일이 많습니다 (BI/CI, 컨텐츠 등)

### UX/UI 관련

- [ ] UI/UX는 디자이너가 가져가야 합니다
- [ ] 다만 너무 고집부리지는 않겠습니다
- [ ] UX/UI 담당 범위가 불명확할 때 논의가 필요합니다

### 산출물

디자이너는 다양한 산출물을 담당합니다:
- 브랜드 컨셉 컬러 폰트 (BI, CI)
- 컨셉디자인
- 컬러노트
- 디자인시스템
- 와이어프레임을 예쁘게 볼 수 있는 디자인
- 컨텐츠 디자인 (마케팅용, 인스타 피딩, Performance Design)
