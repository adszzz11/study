## 필수 산출물 체크리스트

### 테스트 관련

| 산출물 | 설명 | 요청 파트 |
|--------|------|----------|
| 테스트 케이스(시나리오) | 기능별 테스트 항목 | iOS, AOS |
| 테스트 결과 | 실행 결과 리포트 | AOS |

| 산출물      | 설명               | 요청 파트 |

| QA 워크플로우 | 업무 프로세스 정의       | AOS   |
| 결함 처리 방식 | 버그 리포트 및 처리 프로세스 | iOS   |

## DO (했으면 좋겠는 것)

### 버그 리포트

- **구체적인 에러 상황 설명** (iOS)
- **재현 가능한 버그 리포트** (iOS)
- **구체적인 버그 리포트** (AOS)

### 프로세스

- **자신만의 프로세스 정의 및 운영** (AOS)
  - QA가 업무 프로세스를 정의해도 됨
  - 자신의 프로세스를 가져가는게 훨씬 명확할 수 있다

---

## 협업 시 요청사항

### 버그 리포트 작성

- 버그 발견 시 구체적인 상황을 함께 알려주시면 좋겠습니다 (iOS, AOS)
  - 원인/상황 설명이 있으면 빠른 수정이 가능합니다
  - 테스트 기기 정보를 함께 기재해주시면 감사하겠습니다

### 기획 vs 구현 차이

- 기획과 다른 부분 발견 시 해상도/기기 차이 가능성도 함께 검토 부탁드립니다 (AOS)
  - 해상도 다양성으로 인한 의도된 차이일 수 있습니다

### 테스트 범위

- 모든 파트에 대해 균등한 테스트를 부탁드립니다 (AOS)

## 버그 리포트 가이드라인

### 필수 포함 항목

| 항목 | 내용 |
|------|------|
| **환경 - 기기** | (예: Galaxy S23, iPhone 15 Pro) |
| **환경 - OS 버전** | (예: Android 14, iOS 17.2) |
| **환경 - 앱 버전** | (예: 1.2.3) |
| **재현 단계** | 1. ... 2. ... 3. ... |
| **예상 결과** | |
| **실제 결과** | |
| **첨부** | 스크린샷/영상 |

### 개선이 필요한 예시

| 안 좋은 예 | 개선 방향 |
|----------|----------|
| "안 돼요" | 구체적인 상황 추가 필요 |
| "기획이랑 다른데요" | 기기/해상도 정보 추가 필요 |
| "제 폰에서는 안 돼요" | 기기 모델명 추가 필요 |

### Good Case

| 좋은 예시 |
|----------|
| "Galaxy S23 (Android 14)에서 결제 버튼 클릭 시 앱이 크래시됩니다. 재현 단계: 1. 장바구니 진입 2. 결제하기 클릭 3. 크래시 발생" |
| "기획서에는 A로 되어있는데, 해상도 320dp 이하 기기에서는 레이아웃이 깨져서 B로 대응하는 것이 어떨까요?" |

## 테스트 시 고려사항 (AOS)

### Android 기기 다양성

| 유형 | 테스트 필요 수준 |
|------|-----------------|
| 일반 스마트폰 | 기본 |
| 폴더블 (Fold) | 필수 |
| 플립 | 선택 |
| 다양한 해상도 | 필수 |

### 해상도 다양성

- 넓은 화면 / 좁은 화면 모두 테스트
- 해상도로 인한 이슈는 기획 문제가 아닐 수 있음

---

## 파트별 주요 요청 요약

| 파트 | 핵심 요청 |
|------|----------|
| **iOS** | 재현 가능한 버그 리포트, 에러/상황 명세, 구체적인 상황 설명 |
| **AOS** | 자체 프로세스 정의, 기기 명시, 해상도 고려, 균등한 테스트 |

---

## QA 워크플로우 권장 구성 (AOS)

| 순서 | 단계 |
|------|------|
| 1 | 테스트 케이스 작성 |
| 2 | 테스트 실행 |
| 3 | 결과 리포트 작성 |
| 4 | 버그 등록 (기기/환경 명시) |
| 5 | 수정 확인 |
| 6 | 재테스트 |

QA가 직접 업무 프로세스를 정의하면 더 명확해질 수 있음
