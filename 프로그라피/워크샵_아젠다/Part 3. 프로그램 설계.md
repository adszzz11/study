# Part 3. 프로그램 설계

| 항목 | 내용 |
|------|------|
| 진행 방식 | 1/11 워크숍 (60분) |
| 목표 | 6개월 프로그램의 구조와 멘토 개입 방식 확정 |

---

## 3-0. 프로그램의 목적 재확인

**질문:** 6개월 프로그램은 왜 하는가?

**답:**
- 커뮤니티 활성화
- 수익화(1원이라도) 경험 제공
- 강제적인 환경 조성

---

## 3-1. 단계별 기본 구조 (20분)

각 단계마다 확인:
- 이 단계가 커뮤니티 형성에 기여하나?
- 수익화로 가는 과정인가?

| 단계    | 목표                              | 멘토 개입      |
| ----- | ------------------------------- | ---------- |
| 1-2개월 | 커뮤니티 형성 시작, 수익화 가능한 아이디어 선정     | 보통         |
| 3-4개월 | 커뮤니티 내 협업 활발, 실제 서비스 만들기        | 적극         |
| 5-6개월 | 1원이라도 벌어보기, 프로그램 끝나도 이어갈 동력 만들기 | 보통 (독립 준비) |

**의사결정:**
- 런칭 마일스톤 시점
- 매출 시도 마일스톤 시점

---

## 3-2. 멘토링 방식 선택 (30분)

**필수:**
- 월 2회 정기 세션

**선택 1-2개:**

| 옵션 | 부담 | 효과 |
|------|------|------|
| 원온원 | 중 | 개인 맞춤 |
| 코드 리뷰 | 중 | 기술 성장 |
| 비동기 | 낮음 | 유연성 |
| 프로젝트 방향 체크 (수익화 관점) | 낮음 | 핵심 |

**선택 사항:**
- 팀 간 교류 시간 (커뮤니티 형성)
- 수익화 경험 공유회

**포기할 것:**
- 강연/팟캐스트 (Part 4로)
- 위클리 스크럼 (시간 부족)

**의사결정:**
- 필수 외 선택할 것 1-2개
- 포기할 것 확정

---

## 3-3. 팀 구성 (10분)

**확정 사항:**
- 포지션별 혼합 기본
- 팀 매칭 미지원 (자유 팀 구성)

**결정할 것:**

| 항목 | 옵션 | 결정 |
|------|------|------|
| 관심사별 팀 | 허용? | |
| 1인 참가 | 허용? (개인 → 팀 합류 필수?) | |
| 팀 간 교류 | 어떻게 만들 것인가? | |

**의사결정:**
- 위 3가지 결정

---

## 결정할 것 요약

- 런칭/매출 마일스톤 시점
- 멘토링 방식 (필수 + 선택 1-2개)
- 팀 구성 기준 3가지
