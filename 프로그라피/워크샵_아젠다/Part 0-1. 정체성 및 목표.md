## 정체성 요약

| 항목 | 결정 |
|------|------|
| 타겟 | 주니어 (IT 업계 경력 1-3년차) |
| 문제 | 0→1→매출 사이클 경험 없음 |
| CVP | 비즈니스 사이클을 실제로 경험할 수 있는 환경 제공 |

**코어밸류**
1. 매출까지 가게 만든다
2. 모든 판단을 데이터로 한다
3. 약속을 지키는 사람만 남긴다
4. 정답을 모르고 함께 찾는다
5. 정답 없는 갈등에서 스스로 결정하게 만든다

**톤앤매너**
- DO: 숫자로 말한다 / 사실과 의견 구분 / 일단 실행 / 기준 명확히
- DON'T: 가정법 금지 / 결과 없이 위로 금지 / "나중에" 금지 / 비교 금지

## 11기 목표

### 6개월 후 커뮤니티 모습

**질문:** 11기가 끝났을 때 프로그라피 커뮤니티는 어떤 모습?

- 사람들이 무엇을 하고 있나?
- 어떻게 연결되어 있나?
- 무엇 때문에 남아있나?

### 성공의 정의

| 옵션 | 내용 | 장점 | 단점 |
|------|------|------|------|
| A | 프로그램 완주 | 측정 쉬움 | 커뮤니티 안 남을 수 있음 |
| B | 수익화 달성 | 정체성 부합 | 통제 어려움 |
| C | 커뮤니티 잔류 | 커뮤니티 핵심 | 정의 모호 |
| **D** | **복합** | 균형 | 측정 복잡 |

**권장안:** 옵션 D (완주 + 수익화 시도 + 커뮤니티 잔류)

### 최소 성공 기준 (초안)

| 구분 | 기준 |
|------|------|
| Must | 10개 팀 중 5개 팀 완주 |
| Must | 30명 중 15명 커뮤니티 잔류 |
| Must | 전 팀 매출 시도 (금액 무관) |
| Nice | 3개 팀 실제 매출 발생 |
