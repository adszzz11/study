# 디자인 파트 요구사항 명세서

> 개발자 인터뷰 기반 디자인 파트 요구사항 통합 정리

---

## 필수 산출물 체크리스트

### 1순위 (필수)

| 산출물 | 형식 | 설명 | 요청 파트 |
|--------|------|------|----------|
| Figma 디자인 명세 | Figma | 컴포넌트 및 화면 디자인 | FE |
| 디자인시스템 | Figma | 토큰화된 시스템, 컴포넌트 정의 | iOS, FE |
| 프로토타입 | Figma | WireFrame 포함 | AOS |

### 2순위 (권장)

| 산출물 | 형식 | 설명 | 요청 파트 |
|--------|------|------|----------|
| 디자인 플로우 | Figma | Interaction이 많은 경우 필수 | FE |
| Animation Diagram | Figma | 인터랙션 정의 | AOS |
| 일관된 화면 디자인 | - | 모나지 않은 통일된 UI | AOS |

---

## 디자인시스템 구성 요소

### 필수 토큰
- [ ] **Primitive Token**: RGBA 값 (기본 색상)
- [ ] **Semantic Token**: 의미 기반 정의

### 컴포넌트 정의 (iOS)
- [ ] 텍스트박스 디자인
- [ ] 버튼 디자인
- [ ] 기타 UI 컴포넌트

---

## DO (했으면 좋겠는 것)

### 디자인시스템
- [ ] **디자인시스템 기반 구성을 먼저 진행** (iOS, FE)
- [ ] **Theme, Token을 일관성 있게 구성** (FE)
- [ ] **토큰화가 잘 되어있으면 좋겠다** (iOS)
- [ ] **컴포넌트 통일화** (iOS)
- [ ] 화면 제작 시 정의나 예시 기반으로 작업 (iOS)

### 구현 가능성 고려
- [ ] **일반적인 라이브러리에서 재현 가능한 수준의 디자인** (FE)
  - Material UI, Ant Design, shadcn/ui 등
- [ ] **구현 시간 고려한 대안 제시** (iOS)
- [ ] 인터랙티브 구현할 시간이 없을 때 대안 생각 (iOS)

### 플랫폼 고려
- [ ] **Android 특성 고려한 디자인** (AOS)
- [ ] **다양한 화면 크기 고려** - 넓은 화면 / 좁은 화면 (AOS)
- [ ] **폴더블(Fold) 기기 고려** (AOS)

### 일관성
- [ ] **디자인 통일성 유지** (AOS)

---

## DON'T (하지 않았으면 좋겠는 것)

### 과도한 요구
- [ ] ❌ **과도한 Animation 요구** (iOS)
- [ ] ❌ **구현 시간을 고려하지 않은 인터랙티브 요구** (iOS)
- [ ] ❌ 개발자의 구현 수준을 벗어나는 과도한 UX/UI 요구 (FE)

### 플랫폼 무시
- [ ] ❌ **iPhone 디자인만 보고 Android에 동일하게 요구** (AOS)
- [ ] ❌ **폴더블(Fold) 기기 미고려** (AOS)
  - 플립은 상대적으로 괜찮음
- [ ] ❌ 말도 안 되는 특수 기종 미고려 (AOS)

### 팀 목표 무시
- [ ] ❌ 팀 목표와 맞지 않는 디자인 요구 (FE)
  - 빠른 출시가 목표일 때: 과도한 디자인 지양
  - 예술성이 중요할 때: 디자인 강조 가능

---

## Android 개발자 호소 사항

> AOS 개발자가 특히 강조하는 부분

1. **Android 개발자라는 것을 인지해주세요**
2. **폴더블 기기를 고려해주세요**
3. **화면 크기 다양성을 고려해주세요**

### Android 기기 다양성

| 유형 | 고려 수준 |
|------|----------|
| 일반 스마트폰 | 기본 |
| 폴더블 (Fold) | 필수 고려 |
| 플립 | 상대적으로 여유 |
| 특수 해상도 기기 | 케이스별 협의 |

## 권장 UI 라이브러리 (FE 요청)

디자이너가 참고하면 협업이 수월해지는 라이브러리:

- Material UI
- Ant Design
- shadcn/ui
- App UI

---

## 추가 협업 방안

- 디자이너가 프론트 CSS를 직접 작성하는 경우도 긍정적으로 고려 (iOS)

---

## 파트별 주요 요청 요약

| 파트 | 핵심 요청 |
|------|----------|
| **FE** | 디자인시스템 기반, 라이브러리 재현 가능 수준, Token 일관성 |
| **iOS** | 토큰화, 컴포넌트 통일, 구현 시간 고려, Animation 자제 |
| **AOS** | Android 특성 고려, 폴더블 대응, iPhone 기준 디자인 금지 |
