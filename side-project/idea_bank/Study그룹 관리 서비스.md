# 발단
HOCO Object Study에서, Henry님의 개인 리소스를 사용하여 스터디그룹을 관리하고 있었음.
참가자는 주어진 과제를 하는 것 외의 리소스가 필요하지 않지만,  스터디를 관리하는 인원은 기존의 2배 3배 이상의 리소스를 요구해야 했음.

# 목표
스터디 관리자가 효율적으로 스터디를 관리할 수 있는 환경 제공
스터디의 일정관리, 목표 산정 등의 활동 제공
사용자에게 용기를 복돋아주기 위한 활동 제공

## 유즈케이스
#### 회원
- 사용자는 회원가입을 할 수 있다
- 사용자는 회원 비밀번호를 변경할 수 있다
- 사용자는 회원 탈퇴를 할 수 있다
- 사용자는 비회원인 상태로 스터디그룹을 이용할 수 있다
- 사용자는 스터디그룹을 생성할 수 있다
- 사용자는 여러 스터디그룹에 들어갈 수 있다
#### 그룹
- 스터디그룹은 사용자에 의해 생성할 수 있다
- 스터디그룹은 상태를 가진다(활성화, 종료, 임시 비활성화, etc)
- 스터디그룹은 여러 과제를 가질 수 있다
- 스터디그룹은 사용자에게 스터디 관련 일정관리를 제공한다
- 스터디그룹은 사용자의 목표달성을 위해 활동한다
- 스터디그룹은 사용자의 일정을 표기한다
